                    <div class="description">Smart Juggling Balls are Fun balls which can be used for juggling and as an educational platform to demonstrate physics in an interactive and interesting way.<br /><a href='https://github.com/arkadiraf/Juggling_Balls_Public' target="_blank">https://github.com/arkadiraf/Juggling_Balls_Public</a></div>

                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>This is my version for smart juggling balls. The idea is to create an open source platform which can be used (besides juggling) as an educational platform to teach physics using an interactive and fun way. The platform contains a 6 dof or a 9 dof sensor (Mpu 6050 or Mpu 9150) , micro controller, transmitter (NRF24L01) and some LED`s</p><p>Currently I have a working version of the balls, which can be used for juggling: </p><p><iframe width="500" height="281" src="//www.youtube.com/embed/MuF92wXwjuk" frameborder="0" allowfullscreen=""></iframe> </p><p>The files I share include PCB Circuit manufacture files (Gerber) STL models for the casing, BOM with links, assembling instruction, and codes (currently Test code). all are available at: <a href="https://github.com/arkadiraf/Juggling_Balls_Public" target="_blank">https://github.com/arkadiraf/Juggling_Balls_Public</a></p><p>It is possible to assemble the juggling balls using a prototype PCB such as: <a href="https://www.sparkfun.com/products/8810" target="_blank">https://www.sparkfun.com/products/8810</a> my first iterations were based on the same prototype board and later the PCB was design based on the physical dimensions of the board.</p><p>Future development:</p><p>As a first stage towards educational platform i am planning to improve the PCB design so it will be more user friendly and use it as a platform to teach soldering.</p><p>Later the kit can be used to teach coding and communication.</p><p>In parallel i am planing to start working on a lecture content for physics class.</p>
                                
                            </p>

                        <li>
                            <h2><a href="/project/5466/log/20296-eduball-demonstration">EduBall Demonstration</a></h2>
                            <small class="authorship">2 months ago  &bull;
                            <a class="gray-link" href="/project/5466/log/20296#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>I have done some tests with the platform and have plot results to show for (recorded in real time at 100 Hz).</p><p>First image demonstrates filter effects on the measured acceleration, for the test i have put the ball in a plastic bag and rotated it in the vertical plane resulting in the effect seen in the graph. As you can guess the purple line represents raw data and the yellow one, is the filtered data with a simple LPF filter at time constant of 1 second. (the graph scale is in g`s)</p><p><img src="https://cdn.hackaday.io/images/235821435746201516.png"></p><p>After that i have decided to play some throw and catch. (the graph scale is in g`s).</p><p><img src="https://cdn.hackaday.io/images/8910251435746524396.png"></p><p>And for the finish i have decided to test how precisely i can calculate the length of my plastic bag based on the readings of the gyro and the accelerometer.                             <h2><a href="/project/5466/log/20172-eduball-simulink-model">EduBall Simulink Model</a></h2>
                            <small class="authorship">2 months ago  &bull;
                            <a class="gray-link" href="/project/5466/log/20172#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>A few updates,</p><p>I have recently added a Matlab Simulink model to the GitHub including additional code, the model is also available on Matlab file exchange: <a href="http://www.mathworks.com/matlabcentral/fileexchange/51718-eduball-simulink">Simulink Model</a>. </p><p>The model is designed to check the feasibility of the project, in terms of what Physical phenomena i can actually demonstrate with the current hardware setup. The setup is quite simple, The juggling ball is used to measure the Acc/Gyro values and transmit it to a "dongle" (NRF_Receive) which is connected to the computer via USB. The Simulink model reads the values through a COM Port and performs some simple data manipulations. The update rate of the data is set to 100 Hz.</p><p>The simulink model is used to demonstrate Acc/Gyro Raw values, Acc/Gyro Scaled and biased values. Normal Acceleration and Rotation Rate. angle calculation based on Gyro integration and Acc gravity vector calculation. and the final is complementary filter demonstration with comparison between the different approaches.</p><p>See image as to how it all looks:</p><p><img src="https://cdn.hackaday.io/images/7343781435570808464.jpg"></p><p>Conclusions:</p><p>From the Acceleration reading it is very clear to distinguish Free fall and average g such as 1g when the ball is still. It is possible to see the high g`s when the ball is hit, but not at enough sample rate to perform any calculations based on the data such as energy conservation when the ball is thrown and caught at the same height)</p><p>The centrifugal forces are also very clear to demonstrate and to actually compare them to the gyro rotation rate, (as long as the gyro isn`t saturated (at 2000 deg/sec).</p><p>finally it seems that the hardware is good enough for most demonstrations, (still there is a lot of work ahead to actually build a class curriculum and run a pilot<span class="redactor-invisible-space">).</span></p><p><span class="redactor-invisible-space"><br></span></p><p><span class="redactor-invisible-space">The current stage of the project:</span></p><p><span class="redactor-invisible-space">- Find a big company which is interested in supporting the project</span></p><p><span class="redactor-invisible-space">- Test better hardware setup, to get higher sample rate (1 KHz), and with wider range of the accelerometer (200g).</span></p><p><span class="redactor-invisible-space">- Find someone interested in education to help with the curriculum<span class="redactor-invisible-space"></span></span></p>
