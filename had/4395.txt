                    <h5>Description</h5>
                    <div class="description">BeagleLogic realizes a logic analyzer on the BeagleBone Black using the Programmable Real-Time units and matching firmware and Linux kernel modules on the BeagleBone Black. Supports capturing into up to 300+ MB of memory (out of the 512 MB) of the BeagleBone Black.<br /><br />With the sigrok project, BeagleLogic gets support for software triggering and decoding a large variety of digital communication protocols. There's also a small web interface that allows plug-and-play debugging if you are not capturing tons of data once everything is up and running.<br /><br />I recently designed a cape to help interface BeagleLogic with external circuits and protect the BeagleBone from 5V circuits. You are seeing the photos of the cape.<br /><br />The possibilities with BeagleLogic are great, and this page should follow future development of BeagleLogic as it gradually evolves with more cool features.<br /><br />Visit BeagleLogic GitHub at <a href='https://github.com/abhishek-kakkar/BeagleLogic' target="_blank">https://github.com/abhishek-kakkar/BeagleLogic</a></div>

                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>BeagleLogic is a part of a vision of inexpensive Single Board Computers being repurposed as a useful debugging tool, a logic analyzer. It should be the first in a series of tools to make high quality, feature-rich and well-connected instruments accessible to more people over a familiar Linux environment. I see BeagleLogic as an educational tool that can help in understanding digital protocols as well, as one can use the same BeagleBone to send SPI and I2C signals which can be looped back into the inputs to BeagleLogic.</p><p>BeagleLogic makes use of the powerful <a href="http://sigrok.org" target="_blank">sigrok</a> library that provides a standardized interface to DMMs, scopes and logic analyzers. For logic analyzers, sigrok tools support decoding over <strong>50</strong><strong> </strong>different digital protocols <a href="http://sigrok.org/wiki/Protocol_decoders" target="_blank">(see complete list here)</a> for supported protocols. Since BeagleBone runs Linux, it can run the whole sigrok tool set by itself, but it is usually better to process data on a PC as it&apos;s faster.</p><p>There&apos;s a series of technical explanation walkthorughs on my blog that detail how BeagleLogic works:</p><ul><li>Part 1 : <a href="http://theembeddedkitchen.net/beaglelogic-building-a-logic-analyzer-with-the-prus-part-1/449">Building a Logic Analyzer using the PRUs</a>  - Explaining the functioning of the PRU side of things work</li></ul><p>Here&apos;s a system-level block diagram that explains how BeagleLogic works:<img src="https://cdn.hackaday.io/images/7233871424473941538.png"></p><p><strong>License Details</strong><br></p><p>Each component of the project as seen above is individually licensed as See <a href="https://github.com/abhishek-kakkar/BeagleLogic/wiki/sysfs-attributes-Reference" target="_blank">sysfs attributes reference</a> for a complete list of parameters that can be changed.</li><li>The label alongside each channel pin is the name to be used while using the &quot;sigrok-cli&quot; app. So if you wanted to capture channels 1, 2 and 3 you would use the names P8_45, P8_46 and P8_43 respectively and not refer to them as 1,3 and 5. </li><li>However if you process raw binary data, you must use the bit numbers ( channel number - 1) with sigrok to receive correctly.</li></ul>

Alternatively something like `xzcat &lt;image&gt;.img.xz | dd of=/dev/sdX` where X = ID of your SD card (b, c, d, ...)<br><br>Make sure that the correct drive is selected in this case.<br></p></div>

                                    <li>

                    <p>The 3.8.13 is as of now the only version of the Linux kernel on the  BeagleBone Black which allows dynamically loadable/unloadable capes using "device tree overlays" as of now. However the overlay thing was hacky and couldn't make it to the mainline then.</p><p>Developers from the BeagleBoard community are working on getting dynamic device tree loading/unloading into the upstream kernel, and there has been progress lately (<a href="http://www.spinics.net/lists/kernel/msg1927792.html" target="_blank">http://www.spinics.net/lists/kernel/msg1927792.html</a>)</p><p>The BeagleBone Black should technically be able to load the latest kernel and boot from it, but the drivers for PRU for the remoteproc subsystem (which is used by BeagleLogic) aren't there yet. It is planned to port the drivers and establish a framework which would make it to upstream (which is the reason I put it in the form in the first place), but there is no definite timeframe for this.
