                            <h2><a href="/project/4665/log/23281-documentation">Documentation</a></h2>
                            <small class="authorship">2 days ago  &bull;
                                <a class="gray-link" href="/project/4665/log/23281#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-23281"><p>I would usually make a separate external document for this part. But since Hackaday.io has all the documentation tools included, I will just collect everything in this project so things don&apos;t get duplicated.</p><h2>Brief Description</h2><p>Aleph is an NFC printer aimed to work with any existing POS system without the need of changing anything on the POS side. The POS will treat Aleph just like any other paper printer, Aleph software and hardware will manage the rest. As <a data-mention-user-id="7" href="/hacker/7">@Benchoff</a> puts it &quot;IT&#x2019;S LIKE APPLE PAY, BUT FOR RECEIPTS&quot; [<a href="http://hackaday.com/2015/07/06/hackaday-prize-entry-its-like-apple-pay-but-for-receipts/" target="_blank">link</a>]</p><h2>Milestones</h2><p>Based on a lot of feedback from this community, Aleph milestones have changed many times during the design process. I still expect it to change as the design continues. The image below describes the achieved and still to be achieved goals for Aleph.</p><p><img src="https://cdn.hackaday.io/images/3400411439801735051.png"></p><h2>Hardware Design</h2><p><img src="https://cdn.hackaday.io/images/6803471439803011576.png"></p>The hardware is responsible for transporting the JSON object generated by the software to the mobile app. It will receive an HTTP request parse it and send it using NFC to the smartphone.<p>The hardware design is done using KiCad. All the project files are open and available in the GIT repo. For those that do not have access to KiCad, the documents are also available in PDF and Gerber formats. For more information please visit the following <a href="https://bitbucket.org/HishamDaou/aleph-lan-dev/src/b89331b75f9365e26b0905dd69c82323182710a3/aleph-lan-hardware/Electronics/Design/?at=master" target="_blank">link</a>. </p><p>In order to make things easier here are the most important files in the repo : <a href="https://bitbucket.org/HishamDaou/aleph-lan-dev/raw/b89331b75f9365e26b0905dd69c82323182710a3/aleph-lan-hardware/Electronics/Design/aleph-lan-hardware.pdf" target="_blank">PDF Schematic</a> , <a href="https://bitbucket.org/HishamDaou/aleph-lan-dev/raw/b89331b75f9365e26b0905dd69c82323182710a3/aleph-lan-hardware/Electronics/Design/aleph-lan-hardware-gerber.pdf" target="_blank">PDF Gerber</a> , <a href="https://bitbucket.org/HishamDaou/aleph-lan-dev/raw/b89331b75f9365e26b0905dd69c82323182710a3/aleph-lan-hardware/Electronics/Design/aleph-lan-hardware.csv" target="_blank">CSV BOM</a> .</p><h2>Software Development</h2><p>The software side of Aleph is responsible of two main things. The first one is parsing the incoming print command and generating a JSON object from it. The second one is generating a PDF file from the print command to give an exact replica of the paper receipt. The JSON is then sent using LAN to the Aleph hardware. The PDF is uploaded to a secured cloud location for later access by the customer.</p><p>All the PC side software development was done using Java. This is to make it cross platform in the future. For the time being the PC software relies on two ready software that work only on windows, this will be changed in the future when all the hardware part is done. As with the hardware, the software is open source and available in the same repo. Here is a <a href="https://bitbucket.org/HishamDaou/aleph-lan-dev/src/b89331b75f9365e26b0905dd69c82323182710a3/aleph-lan-pc-print/?at=master" target="_blank">quick link</a> to the software folder. </p><h2>Firmware Development</h2><p>The firmware is built using Eclipse CDT and GCC on Ubuntu. But since the code is <a href="https://bitbucket.org/HishamDaou/aleph-lan-dev/src/b89331b75f9365e26b0905dd69c82323182710a3/aleph-lan-embedded/?at=master" target="_blank">published openly</a>, anyone can fork the repo and make his own version of Aleph firmware using any desired platform. </p><h2>Mobile App Development</h2><p>Currently, and since the transportation is done using NFC only, the mobile application is developed for Android. The mobile app has the minimal features to function and the UI was done quickly just to show the functionality. Later on the application should get a lift off and implement material design guidelines, also get a lot of more features. [<a href="https://bitbucket.org/HishamDaou/aleph-lan-dev/src/b89331b75f9365e26b0905dd69c82323182710a3/aleph-lan-mobile/?at=master" target="_blank">link</a>]</p><h2>Ultimate Goals</h2><p>The first goal is to make Aleph integrate inside already existing &apos;Tap to pay&apos; devices. This will allow the user to pay and get the receipt by a tapping on the same device. I am already working to achieve this, I will post more details when something solid comes up.</p><p>The next goal of Aleph is to allow users without access to smartphones to benefit from it. Again a course of action has been set, more details will be published once the work is proved.</p><h2>Licenses</h2><p>This version of Aleph is based on free software:</p><p>1- Redmon:  Redirection Port Monitor [<a href="https://bitbucket.org/HishamDaou/aleph-lan-dev/raw/b89331b75f9365e26b0905dd69c82323182710a3/Setup/redmon/LICENCE" target="_blank">GNU GPL3</a><span class="redactor-invisible-space">]</span></p><p>2- Ghostscript : A suite of software based on an interpreter for Adobe Systems&apos; PostScript and Portable Document Format (PDF). [<a href="http://www.artifex.com/page/licensing-information.html" target="_blank">LICENSE</a>]</p><p>The next version will be more robust and less dependent on 3rd party applications.</p></div>
