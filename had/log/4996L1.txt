                                <div id="post-body-15790"><p>I've been working on variations of this for just over a year now slowly in my spare time. Although the project is labelled complete, there are still lots of improvements and upgrades to carry out to make the Razbot more capable and reliable. <br><br>The 2nd generation of the printed circuit board has been designed and is currently being assembled and tested. This is the version is found in the documentation. It has a few corrections and a few more features than the initial 4 working prototypes. </p><p>Now that there is a reliable working hardware platform, we can begin adding features to both the hardware and software. A list of 'to-do' items is found at the end of the Build a Raspberry Robot.pdf document. Some examples include:</p><p>Mecanum wheels for omni-directional driving.</p><p>A web interface using ROSlibjs</p><p>Publishing the instructions and files to use the GAZEBO physics simulator and a virtual Razbot</p></div>

                            <h2><a href="/project/4996/log/16182-bringing-up-the-rpi2-with-razbot">Bringing up the RPi2 with Razbot</a></h2>
                            <small class="authorship">4 months ago  &bull;
                                <a class="gray-link" href="/project/4996/log/16182#discussion-list">
                                    1 comment
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-16182"><p>The Green Razbot now has an RPi 2B installed inside with one of the original MCU boards. </p><p>I updated the Razbot image to support the RP2 2 according to the following:<br><a href="http://thepihut.com/blogs/raspberry-pi-tutorials/16982376-updating-raspbian-on-your-microsd-for-the-raspberry-pi-2" target="_blank">http://thepihut.com/blogs/raspberry-pi-tutorials/16982376-updating-raspbian-on-your-microsd-for-the-raspberry-pi-2</a><br></p><p><a href="https://www.raspberrypi.org/documentation/raspbian/updating.md" target="_blank">https://www.raspberrypi.org/documentation/raspbian/updating.md</a><br></p><p>On the first shot it worked great. The robot started the MJPG-Streamer for the camera as well as the ROSCORE and required packages. The robot drove around and streamed video as normal. Using Putty to ssh into the Pi was much faster on the Pi2.</p><p>The catch: the Pi 2 appears to be much more sensitive to SD  card corruption when interrupting the power supply. (Powering off the Razbot by hitting the switch). This makes sense, if you kill the power while writing to the SD card, goodbye SD card. SSH-ing into the robot everytime you want to shut down sort of sucks</p><p>For now I'll have to lock down the boot partition as read only to ensure it doesn't corrupt. </p><p>In the long run it looks like we might wanted a larger capacitor on the MCU board (which has the 5V power supply). Using a GPIO pin to detect primary power loss, issue a sync command before losing secondary power (thanks Greg J for the suggestion) .<br><br>Hammering out all the minor details related to mech / elec / software and getting things to run smoothly and reliably is has got to be one of the biggest challenges in robotics. Before you can go implementing the latest sensors and SLAM algorithms you need a platform that works reliably and finding / building one is not always an easy task.<br><br></p></div>
                            </div>

                            <h2><a href="/project/4996/log/17009-simulate-it">Simulate it!</a></h2>
                            <small class="authorship">4 months ago  &bull;
                                <a class="gray-link" href="/project/4996/log/17009#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-17009"><p>Our friend Paul B has created a simulation model of the Razbot. You can take it for a test drive before you build one.<br><br>To run the simulation, install ROS, Gazebo and the Gazebo ROS packages on your Ubuntu system.</p><p>Clone the following repository into your catkin workspace and build the workspace</p><p><a href="https://github.com/Waterfox/razbot_tutorials" target="_blank">https://github.com/Waterfox/razbot_tutorials</a></p><p>There may be a few more dependencies to install. When finished, you can launch the simulation with: </p><pre>roslaunch gazebo_razbot simulator.launch</pre><p>To control the robot, run rosrun rqt_robot_steering rqt_robot_steering in a separate terminal. Set the output topic</p><p>to /razbot_diff_drive_controller/cmd_vel, and use the sliders to drive the robot. If you have a joystick, you can also use the teleop_twist_joy package. Just be sure to remap the cmd_vel topic to /razbot_diff_drive_controller/cmd_vel</p><p><img src="https://cdn.hackaday.io/images/6064071430100095596.png" alt="" style="display: block; margin: auto;"><br></p><p>The simulator software is actually ahead of the software being run on the Razbot itself. One of the next steps is to re-write a razbot control node to subscribe to cmd_vel and output the motor control topic that the MCU is listening for. <br><br>I'll add some detailed instructions into the manual on running the simulator the next time it is updated.<br><br></p></div>
