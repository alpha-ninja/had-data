                            <div style="position:relative">
                                <div id="post-body-20682"><p>Mandatory Feature Set:</p><p>1) Ability to designate a thermostat as the master, from which the system polls the process temperature.</p><p>2) Use an already owned Linux set top box as the private server for Sparkfun's Data platform.</p><p>3) Run off DC wall warts. </p><p>4) Fail Safe.</p><p>5) Ascetically pleasing, i.e. wife approved. </p><p>Desired Long Term Features:</p><p>1) Create a webpage on the Linux box allowing the HVAC system to be configurable by mobile devices.</p><p>2) use a SDR to capture the frequency and protocol for the wireless fan remotes in the den and master bedroom. Implement the ability to control the fan via the thermostat.</p><p>3) Create an outdoor weather station that communicates with the base station.</p><p>4) Make the downstairs thermostat run off the 24VAC supplied for the HVAC control signals. </p><p>5) Add Z-wave or X10 window sensor to determine if windows are open. </p><p>6) Control the existing attic fan.  </p><p>7) Integrate in the Goggle calendar API so the system can poll a calendar to determine a schedule.</p><p>8) Add to the web interface, plot the temperature and HVAC usage.</p><p>9) Find an API or code chunk to ping our mobile devices to determine if they are on home Wi-Fi to help determine when we are away.  </p></div>
                            </div>
                        </li>

                                <div id="post-body-20684"><p><a href="https://hackaday.io/project/6456-esp8266-din-8-channel-wifi-relay-controller">https://hackaday.io/project/6456-esp8266-din-8-channel-wifi-relay-controller</a></p><p>and it really got me focused on using an ESP8266 as the brains of the operation. The touch screen module I want to use can communicate via SPI and I'm sure I can find a temperature / humidity sensor that can output SPI. That would significantly decrease the I/O pin count I would need especially if for the downstairs unit, I could use an SPI GPIO expander to drive the relays.</p></div>
