                            <small class="authorship">3 months ago  &bull;
                                <a class="gray-link" href="/project/4108/log/17719#discussion-list">
                                    2 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-17719"><p>So, my last log was about implementing a compiler for my project. Today it was<a class="vnyapcfh" href="" title="Click to Continue &gt; by PriceMinus"> time<img src="https://cdncache-a.akamaihd.net/items/it/img/arrow-10x10.png"></a> to prepare a stage for my compiler, meaning making some sort of text editor. </p><p>I am making a single line text editor . First I will compile each line on the fly, which has a bad side to it -&gt; you can't really go back and fix something in your code. You make a mistake, you start all over again. Later, when all commands will be implemented, I will change that. </p><p>Today I had a couple of hours of thinking and searching to do, before finnaly figuring out how to make a backspace functionality for my LCD display.   But it is working now. I also<a class="vnyapcfh" href="" title="Click to Continue &gt; by PriceMinus"> started<img src="https://cdncache-a.akamaihd.net/items/it/img/arrow-10x10.png"></a> outlining the finished prototype, since the compiler (or better to say op code) is what connects all the parts of my computer together. So the plan is to have a two state computer. </p><p>STATES:</p><p>1.<a class="vnyapcfh" href="" title="Click to Continue &gt; by PriceMinus"> programming<img src="https://cdncache-a.akamaihd.net/items/it/img/arrow-10x10.png"></a> (editor, compiler, compiler output target)</p><p>2. Execution (not executing code directly from a target, but executing some built in functions, among which is a a method to call an assembled code from ROM).</p><p>Please, any thoughts on that? </p><p>I am pretty lost, so I'm just trying to make it as simple as possible, so to have something to build on. The main problem is that I don't have an exact idea of what the end thing will look like and work like. </p><p>Thanks.<br></p></div>
