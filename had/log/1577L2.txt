                            <h2><a href="/project/1577/log/9133-hardware-thoughts">Hardware Thoughts</a></h2>
                            <small class="authorship">a year ago  &bull;
                                <a class="gray-link" href="/project/1577/log/9133#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-9133"><p>I just had a chat with a friend who has some more experience with dealing with SD cards. He told me some unpleasant things about the potential write delays. It looks like I am going to need considerably more RAM as a result. Luckily the chip has a built-in memory controller with an SRAM mode that can deal with an 8 bit wide memory. I may have to switch to a different version of the uC to get enough pins, but that's not too bad. The physical size of the readily available non-BGA SRAMs is annoying, however.</p></div>
                            </div>

                            <small class="authorship">4 months ago  &bull;
                                <a class="gray-link" href="/project/1577/log/17417#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-17417"><p>I haven't been making all the much progress on this because I've been concentrating on the Arachnio.</p><p>However, I've decided I'm going to strip down even more on the development hardware to get the software up to the demo stage. Here's what the new stack is going to look like, since I largely have the pieces:</p><p>1 x Arduino Due</p><p>1 x Arduino Mega Protoshield (for mounting stuff)</p><p>1 x MAX11300 dev board</p><p>1 x CC3000 shield</p><p>1 x Memoire shield</p><p>I think I have all the parts in hand, except an extra Protoshield. All I definitely have to write is a driver for the MAX11300 and the glue code to talk to the host. Then I have to work on the host stuff. <br></p></div>
