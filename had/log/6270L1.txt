                            <h2><a href="/project/6270/log/19394-johanson-balun">Johanson Balun</a></h2>
                            <small class="authorship">2 months ago  &bull;
                                <a class="gray-link" href="/project/6270/log/19394#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-19394"><p>I had originally designed the dev board using the datasheet schematic. After doing that, I went to DigiKey and found all the parts. I learned something; RF capacitors and inductors are really expensive. In singles at least. I mean really expensive, upwards of $1.50 for a single capacitor.<span></span></p><p>Looking around, I saw that a single <a href="http://www.johansontechnology.com/datasheets/baluns/Balun_Filter_Matched_2450BM14A0002_v2.pdf">Johanson balun</a> costs under $1.50, uses less parts, and is just all around a simpler solution.</p><p>Looking into it more, it wasn't quite that easy. It turns out that to work with the <a href="http://www.johansontechnology.com/datasheets/baluns/Johanson%20nRF24E01_QFN24_Matching.pdf">LE1 QFN24 version</a>, you need 2x 1nH Inductors and a small capacitor.</p></div>
                            </div>
                        </li>
                    
                        <li>
                            <h2><a href="/project/6270/log/19395-reg711-33">REG711-33</a></h2>
                            <small class="authorship">2 months ago  &bull;
                                <a class="gray-link" href="/project/6270/log/19395#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-19395"><p>I have chosen to power the board through multiple sources; battery, LiPo, and USB. To simplify turning the various inputs to a clean 3.3 volts, I'm using the Texas Instruments <a href="http://www.ti.com/lit/ds/symlink/reg711-3.pdf">REG711</a> 3.3v regulator. I debated using the 1.8 volt regulator, but decided against it to make it simpler to use other sensors. <br></p><p>It remains to be seen if this will allow a battery to be used longer. That's the idea though. </p></div>
                            </div>
                        </li>
                    
                        <li>
                            <h2><a href="/project/6270/log/19397-i2c-sensors">I2C Sensors</a></h2>
                            <small class="authorship">2 months ago  &bull;
                                <a class="gray-link" href="/project/6270/log/19397#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-19397"><p>The le1 is a space-limited device, only 16kb. So I am designing the dev board to interface with I2C sensors. Most nodes will take some measurement, temperature say, send it along and go back to sleep. Making a couple calls to the I2C bus takes up very little space, giving plenty of room for the network protocol and other housekeeping functions. </p><p>Only using I2C means that the larger versions of the le1 aren't really needed. No need to have a bunch of unused GPIO pins when you are only using SDA and SCL. </p><p>I haven't figured out how to deal with most sensor breakout boards coming with pullups already.</p></div>
                            </div>
                        </li>
                    
                        <li>
                            <h2><a href="/project/6270/log/19400-other-breakout-modules">Other breakout modules</a></h2>
                            <small class="authorship">2 months ago  &bull;
                                <a class="gray-link" href="/project/6270/log/19400#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-19400"><p>You can buy a breakout module on aliexpress for around $5.00. I considered just using those, but I soon ran into a spaghetti tangle of wires connecting power, sensors, programming pins and whatever else. I also found, to my annoyance, that each company sells a different breakout. I have multiple versions of le1's.  Each one has different pinouts. It is impossible to reliably replicate things. One supplier might disappear and I'll be left  figuring out some new board. And, they all come with 1.27mm headers which is very inconvienent to work with. </p><p>Also, I want to design a board from start to finish. Just to do it and see it work. <br></p></div>
                            </div>

                            <h2><a href="/project/6270/log/19403-programmer-details">Programmer details</a></h2>
                            <small class="authorship">2 months ago  &bull;
                                <a class="gray-link" href="/project/6270/log/19403#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-19403"><p>I currently have a mess of wires from a le1 hookup board to an Arduino Uno. This works, but barely. The le1 is not 5v tolerant per the datasheet, but it is tolerant 98% of the time, the other 2% will kill it. Not to mention the wires and the extra board. </p><p>To make things better, I am going to design a daughterboard to an Arduino Leonardo Micro. I picked a Leonardo board because it is small, cheap, and has USB built-in. The daughterboard will include a level shifter to get down to 3.3v and female headers to accept the dev module I'm also designing. </p><p>The Leonardo will interface with the already working PC programming software. </p></div>
                            </div>
                        </li>
                    
                        <li>
                            <h2><a href="/project/6270/log/19405-pc-development-environment">PC development environment</a></h2>
                            <small class="authorship">2 months ago  &bull;
                                <a class="gray-link" href="/project/6270/log/19405#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-19405"><p>The PC side of this is already done. I just need to package it up nicely. </p><h2>Compiler:</h2><p>When it comes to compilers, there is only free option, <a href="http://sdcc.sourceforge.net/">SDCC</a>. The other option, Kiel, costs an outrageous amount of money. SDCC works in both Linux and Windows.</p><h2>SDK:</h2><p>Likewise, there is only one option here. Someone named <a href="http://blog.diyembedded.com/">Brennen</a> did a huge amount of work and compiled an <a href="https://github.com/justind000/nRF24LE1_SDK">SDK</a> that makes use of nearly the entire chip's functions and released it to everyone under an open source license. Combine this with my <a href="https://hackaday.io/project/5794-nrf24le1-wiring-library">Wiring</a> library and you get a familiar and portable environment. </p><h2>IDE:</h2><p>I considered integrating this with the Arduino IDE, but then I remembered how awful that is in comparison to just about everything else. Right now I am using <a href="http://www.sublimetext.com/">Sublime</a> and a make file. It works , although it could use some work to automate things. It works on Linux and Windows. build-essential is required on Linux for make. You can try installing CygWin on windows for make, but I have just been using a bat file called make.bat. Sublime doesn't know the difference. </p><h2>Programmer:</h2><p>I have <a href="https://github.com/justind000/nRF24LE1_Programmer">forked</a> <a href="https://github.com/DeanCording/nRF24LE1_Programmer">Dean Cording</a>'s perl script and Arduino sketch to program the le1. </p></div>
                            </div>
                        </li>

                        <li>
                            <h2><a href="/project/6270/log/19658-software-package">Software Package</a></h2>
                            <small class="authorship">2 months ago  &bull;
                                <a class="gray-link" href="/project/6270/log/19658#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-19658"><p>I have heavily borrowed from <a href="https://packagecontrol.io/packages/AVR">SublimeAVR</a>, a Sublime Text plugin that lets you use the AVR toolchain, and made the equivalent for the nrf24le1. There will be a Windows and Linux version. </p><p>The Windows version will include GNU Make and the programmer (which is a perl script) 'compiled' so that the end-user doesn't need to bother setting up a perl environment. It also puts the required include and lib directories in the project and a Makefile so that everything just works out of the box. </p><p>The Linux version is mostly the same, with a 'compiled' perl script that includes the dependencies so there won't be any fuss installing extras. Make won't be included since it's likely to be installed anyway. </p><p>That should mean that SDCC and Sublime Text will be the only required installs. And neither will require any tinkering. </p><p>After everything else is ready, I will put this on <a href="https://packagecontrol.io/">Package Control</a> so it will be easy to install. Until then, check it out at my <a href="https://github.com/justind000/ionode_sublime_plugin">repo</a>. </p></div>
