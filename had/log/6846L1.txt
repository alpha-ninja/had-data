                                <div id="post-body-22647"><p>A new function has been added to the concept of &#x201C;Top Navette&#x201D; which Allows exchanging (with other &quot;Top Navette&quot; users) informations about route conditions, this functions is named &#x201C;FC9&#x201D;.</p><p>The characterization of this function indicates that &#x201C;Personal navigation data is absolutely preserved, the information exchanged are those concerning road conditions, such as : length, fluidity (depending on date), caps, &#x2026;&#x201D; </p><p>This function will allow the &#x201C;Top Navette&#x201D; system to be interconnected to real road informations that can help its user to improve predictions about shuttling time.</p><p>The use of the &#x201C;FC9&#x201D; function is totally free, but the &#x201C;Top Navette&#x201D; user may have a Smart Phone or a PC equipped with Bluetooth devise in order to enjoy this feature.</p><p>The Smart Phone freely downloads and uploads roads data form the dedicated website when is connected to internet; then it synchronizes data with &#x201C;Top Navette&#x201D; when it is inside the vehicle.</p></div>

                            <h2><a href="/project/6846/log/23285-top-navette-use-case">Top Navette Use Case</a></h2>
                            <small class="authorship">2 days ago  &bull;
                                <a class="gray-link" href="/project/6846/log/23285#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-23285"><p><img src="https://cdn.hackaday.io/images/9151331439804961264.png"></p><p><br>This is a briefs descriptions of use case for TOP NAVETTE </p><p><strong>/*********************                              Actor: User                                *******************/</strong></p>  <p><strong>Use Case:</strong> Choose the destination</p>  <p><strong>Description: </strong>By default, the system will choose a destination among his daily planning. User can change the proposed destination or just reject it.</p>    <p><strong>Use Case:</strong> Regenerate the result of road planning</p>  <p><strong>Description: </strong>By default, the system give a result of road planning among the mode (mode of road planning) chosen by user. He can change quickly it for this shuttle using NEXT button.</p>    <p><strong>Use Case:</strong> Choose another shuttle time</p>  <p><strong>Description:</strong> When the user reach his destination, the system will give him the ideal time for his next shuttle. He can choose another shuttle time and the system will give him an ideal time corresponding to the time chosen.</p>    <p><strong>Use Case:</strong> Schedule daily shuttle</p>  <p><strong>Description: </strong>The user can put daily shuttles with corresponding time to start and the system will give ideal time to go for each shuttle.</p>  <p><strong> </strong></p>  <p><strong>Use Case:</strong> Change mode of road planning </p>  <p><strong>Description: </strong>The result of road planning calculated among the mode selected. There is four modes: </p>  <ul><li>The shortest way</li><li>The most rapid route</li><li>The most economic route</li><li>All, (33% for shortest way, 33% rapid route, 33% economic route)</li><li> Elapsed time, the time elapsed since the start-up of his shuttle. </li><li>Offset time, the difference between time of getting in the car and ideal time for his shuttle.</li><li>If there is abnormal traffic in the road, because accident or anything else</li><li>If there is another road much perfect then road he used now.</li></ul>          <p><strong>Use Case:</strong> Manage localization point</p>  <p><strong>Description: </strong>the user can add localization point, edit, or remove it.</p>    <p><strong>Use Case:</strong> Change time setting </p>  <p><strong>Description: </strong>The user configure time setting among the GMT and summer hour.</p>    <p><strong>Use Case:</strong> Change display setting</p>  <p><strong>Description: </strong>The user can activate or deactivate:</p>        <p><strong>Use Case:</strong> Manage Bluetooth devices</p>  <p style="margin-left: 20px;"><strong>Description: </strong>The user can add his Bluetooth device by searching it and adding it. He can remove it from the list.</p><p style="margin-left: 20px;">/************                 <strong>Actor: Bluetooth device application</strong> *********************/<br></p>    <p><strong>Use Case:</strong> Bluetooth device authentication</p>  <p><strong>Description: </strong>When the system detect Bluetooth device connection, it authenticate it to allow it to use few functions between them.</p>    <p><strong>Use Case:</strong> Synchronize the daily street weight data</p>  <p><strong>Description: </strong>This use case contain few steps</p>  <ul><li>The system give the application new coordinate data (Longitude, latitude coordinate of GPS device)</li><li>The application save it. It identify the countries and cities. When the device connected to Ethernet (Wi-Fi, Home, job or another Ethernet connection), it upload the street weight data and save into it.</li><li>The application send new street weight data saved into it to the system (TOP NAVETTE)</li></ul>        <p><strong> </strong></p>  <p><strong> </strong></p>  <p><strong>Use Case:</strong> Synchronize the system configuration</p>  <p><strong>Description: </strong>The system read the date of the last modification of system configuration in Application. If the date newer then that the system have, then it will uploaded it. Else, it will send the last modification to the system.</p>  <p><strong> </strong></p>  <p><strong>Use Case:</strong> Notify that it&#x2019;s online</p>  <p><strong>Description: </strong>This is a special case, if the Bluetooth device connected to 3G and it is able to connect to the server when the user in the car, then the application will to notify that&#x2019;s is online</p>    <p><strong>Use Case:</strong> Send instantly weight data</p>  <p><strong>Description: </strong>The application send, automatically and instantly to system, notifications about :</p><ul><li>If there is abnormal traffic in the road, because accident or anything else</li><li>If there is another road much perfect then road he used now.</li></ul><p><img src="https://cdn.hackaday.io/images/308401439805215822.png"></p><p>  Briefs descriptions of use case of <strong>Bluetooth Device Application For TOP NAVETTE</strong></p><p><strong>/*********************                          Actor: User                        ***********************/</strong></p>  <p><strong> Use Case:</strong> Change the system configuration</p>  <p><strong>Description: </strong>User can see the system configuration before the last synchronization and change it. The system configuration contain: </p>  <ul><li>Mode of road planning</li><li>Manage localization point</li><li>Change time setting</li><li>Change display setting</li><li>Wi-Fi</li><li>3G</li><li>&#x2026;, etc.</li></ul>          <p><strong>Use Case:</strong> Allow the application to use Internet</p>  <p><strong>Description: </strong>User can allow the application to use Internet and specify what types of Internet connection the application can automatically use it</p>          <p><strong>Use Case:</strong> Consult the navigation history</p>  <p><strong>Description: </strong>User can see his history of navigation of few months ago and clear it from the device if he want.</p>    <p><strong>Use Case:</strong> Schedule daily shuttle</p>  <p><strong>Description: </strong>The user can put daily shuttles with corresponding time to start and the system will give ideal time to go for each shuttle.</p>  <p>.</p>  <p><strong>Use Case:</strong> Get notification about ideal time</p>  <p><strong>Description: </strong>When the ideal time comes for next shuttle, the application send notification to remind the user.</p>    <p><strong>Use Case:</strong> Change time, destination or the road for the next shuttle</p>  <p><strong>Description: </strong>User can change the destination or the time for the next shuttle or even, he can choose the road and regenerate the result of road planning.</p>    <p><strong>/**********************                      Actor: TOP NAVETTE                          *****************/</strong></p>    <p><strong>Use Case:</strong> Bluetooth device authentication</p>  <p><strong>Description: </strong>When the system detect Bluetooth device connection, it authenticate it to allow it to use few functions between them.</p>    <p><strong>Use Case:</strong> Synchronize the daily street weight data</p>  <p><strong>Description: </strong>This use case contain few steps</p>  <ul><li>The system give the application new coordinate data (Longitude, latitude coordinate of GPS device)</li><li>The application save it. It identify the countries and cities. When the device connected to Ethernet (Wi-Fi, Home, job or another Ethernet connection), it upload the street weight data and save into it.</li><li>The application send new street weight data saved into it to the system (TOP NAVETTE)</li></ul>        <p><strong> </strong></p>  <p><strong> </strong></p>  <p><strong>Use Case:</strong> Synchronize the system configuration</p>  <p><strong>Description: </strong>The system read the date of the last modification of system configuration in Application. If the date newer then that the system have, then it will uploaded it. Else, it will send the last modification to the system.</p>  <p><strong> </strong></p>    <p><strong>Use Case:</strong> Send instantly weight data</p>  <p><strong>Description: </strong>The application send, automatically and instantly to system, notifications about:</p>  <ul><li>If there is abnormal traffic in the road, because accident or anything else</li><li>If there is another road much perfect then road he used now.</li></ul>    <p><strong> </strong></p>  <p><strong>/*******************                      Actor: VAN                          ************************/</strong></p>  <p><strong> </strong></p>  <p><strong>Use Case:</strong> Update the daily street weight data</p>  <p><strong>Description: </strong>If the device connected to VAN, it will update from VAN the daily street weight data and send the personal street weight data to VAN.</p>    <p><strong>Use Case:</strong> Notify that it&#x2019;s online</p>  <p><strong>Description: </strong>This is a special case, if the Bluetooth device connected to 3G and it is able to connect to the server when the user in the car, then the application will to notify that&#x2019;s is online</p><p><strong></strong></p></div>

                            <h2><a href="/project/6846/log/23581-general-description-of-top-navette">General Description Of Top Navette</a></h2>
                            <small class="authorship">a day ago  &bull;
                                <a class="gray-link" href="/project/6846/log/23581#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-23581"><strong></strong><p><img src="https://cdn.hackaday.io/images/9866821439841500947.png"></p><p><strong>Why we  use Top Navette?</strong></p><p><strong></strong>When you want to know the most efficient road from your Home to your JOB without needing to buy a expensive product that it have more features useless in your case. Then, Top Navette is the best solution. It can give your the best roads and it will guide you  <span class="redactor-invisible-space"><span class="redactor-invisible-space"></span></span><span class="redactor-invisible-space"><span class="redactor-invisible-space"></span></span><strong><span class="redactor-invisible-space"><span class="redactor-invisible-space"></span></span></strong>by names of cities that you Now.<br><strong></strong></p><p>Top Navette store all the regular shuttle in SD card and when the user need to know what is the best road to his regular shuttle, Top Navette will generate from his personal database what is the best roads for a specific time.</p><p><strong>The features of  Top Navette </strong></p><p><span class="redactor-invisible-space"></span>Top Navette is an <span class="short_text"><span class="hps">autonomy</span></span> system but  for more performance and options you can connect your Smart Phone or your PC to it. Every time, it authenticate with your device it will synchronize with a specific application of Top Navette installed in the device.</p><p>So here, your personal data is saved in your device when you want to go in another shuttle with another car have Top Navette. All you need to do just connect your device to Top Navette and it will update the data to it.</p><p>When the communities of Top Navette become much bigger by using it in many cars, you would like to update daily your Top Navette  to become more efficient. This is done by allow the application to synchronize with the server (VAN) and then your device will send the last update to your Top Navette when they connecting by Bluetooth.</p><p> If your device is a Smart Phone and you have an access to Internet by 3G, you can get the most perfect roads by synchronizing with the server and get the instant data about road traffic.</p><p>You can use Top Navette or the application in your device to schedule your daily planning and get in every shuttle the ideal time and the best road.</p><p><strong>How we keep your shuttle privacy ?</strong></p><p><strong></strong>So this is almost all the data structure in Top Navette and Smart Phone</p><p> <img src="https://cdn.hackaday.io/images/883631439842992730.png"></p><p>So when your phone synchronize with the server, it will send only &quot;Street&quot; &amp; &quot;City&quot;. The &quot;UserStreet&quot; and &quot;UserCity&quot;  describe how you using &quot;Street&quot; and &quot;City&quot;, so they will stay in Top Navette or in your device application without sending them to the server.</p><p>The server all it need is the weight of the street.</p><p><strong>What is weight of the street ?</strong></p><p><span class="redactor-invisible-space">It&apos;s an argument that helps us to calculate the <span class="short_text"><span class="hps">expected time</span> to reach your destination and this argument is exit into &quot;Street&quot;.</span></span></p><p><span class="redactor-invisible-space"><span class="redactor-invisible-space">So the server collect all the weight of the same street from all the user and calculate the average weight in interval of time (like from 7.30 to 7.45) <br><br></span></span></p></div>
                            </div>
