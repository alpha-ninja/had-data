                            <h2><a href="/project/6592/log/23705-dipsy-testing">DIPSY Testing</a></h2>
                            <small class="authorship">13 hours ago  &bull;
                                <a class="gray-link" href="/project/6592/log/23705#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-23705"><p>Easiest way is to use the ICEd board as DIPSY Programmer, then it would be recognized by Lattice Programming tools directly on USB Port 1 (FTDI Channel B). </p><p>Step by Step:</p><p>0: Assume ICEd board is preflashed with &quot;DIPSY Programmer&quot; FPGA code (it is available in github ..\THP2015..)</p><p>1: Insert DIPSY into the socket</p><p>2: Connect micro USB Cable to ICEd<span class="redactor-invisible-space"><br></span></p><p><img src="https://cdn.hackaday.io/images/800071439924120833.jpg"></p>Correct alignment for DIPSY, and settings for the DIP switch and jumpers. Green LED on ICEd should be ON (goes off when rst button pressed, then back on when HX1K is configured from Flash)<p>3: Start Lattice Diamond Programmer 3.5<br></p><p>4: Click &quot;Detect Cable&quot;</p><p><img src="https://cdn.hackaday.io/images/6452721439922976650.png"></p><p>This shows that FTDI on ICEd is detected and selected as USB Programmer (not in the screenshot, some scan result is displayed from old programmer project, with fresh new one there would be empty screen),</p><p>5: Select FTUSB-1 as &quot;Port&quot; this would direct programming operations to DIPSY and not the HX1K on the ICEd</p><p><img src="https://cdn.hackaday.io/images/1362451439923341860.png"></p><p>FTDI Channel B (FTUSB-1) selected as USB port</p><p>6: Click on &quot;scan&quot;</p><p><img src="https://cdn.hackaday.io/images/3590171439923403156.png"></p><p>7: You should see this</p><p><img src="https://cdn.hackaday.io/images/5769921439923527326.png"></p><p>8: Click on &quot;...&quot; to select Programming file, can for test choose file from the DIPSY gihub repo  .. /THP2015</p><p><a href="https://github.com/AnttiLukats/DIPSY/tree/master/THP2015/AVR_SoC">https://github.com/AnttiLukats/DIPSY/tree/master/THP2015/AVR_SoC</a><br></p><p>there is &quot;blink.bin&quot;</p><p>9: Click on the green arrow to start programming</p><p>10: RGB LED should blink on DIPSY (not on ICEd)</p><p>Congratulations:</p><p>You have configured DIPSY with AVR RISC Based custom System on Chip &quot;binary code&quot; that did configure the LUTS and FLIP FLOPs to &quot;emulate&quot; the AVR processor HARDWARE, and this processor is now running AVR Object code.</p></div>
                            </div>
