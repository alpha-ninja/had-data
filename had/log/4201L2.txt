                            <small class="authorship">6 months ago  &bull;
                                <a class="gray-link" href="/project/4201/log/14604#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-14604"><p><img src="https://cdn.hackaday.io/images/1686111425208731841.JPG"></p><p>As I already presented, This tiny computer can transfer picture (jpeg) and BASIC program made on the console with keyboard to another computer (PC or Mac or iPad, wifi-capable any computer), but if we want to copy files to this computer, I have to pulled out FlashAir and should copy to it. It takes some time..</p><p>Today I confirmed that, Special function of FlashAir enables direct upload to FlashAir storage area, where TinyBasic can directly load by "load" command.</p><p>Step 1: modify "/SD_WLAN/CONFIG" file</p><p><img src="https://cdn.hackaday.io/images/380431425210912613.jpg"></p><p>On the SD card, add one line "UPLOAD=1" to "CONFIG" file in invisible directory "SD_WLAN". This directory is invisible so will need some way (example,  from terminal, cd /Volumes/SD_CARD_NAME and vim CONFIG in case of OSX).</p><p>Insert FlashAir to tinyComputer and Wi-Fi access from another computer/tablet. The address is</p><p><img src="https://cdn.hackaday.io/images/5364341425211121436.jpg"></p><p>Please note that if you are using special DNS, 8.8.8.8 or something, please de-select and set it up as DNS is given by server (otherwise it will not appear)</p><p><img src="https://cdn.hackaday.io/images/3424801425211211561.jpg"></p><p>On browser, you will see cgi screen and push "Choose File" and select the file you want to upload, and push "submit", that is all. Now the file is uploaded to FlashAir, naturally can be loaded by tinyBasic or Bitlash, or anything running on it!</p><p>Have fun!</p></div>

                            <h2><a href="/project/4201/log/14635-further-explore-of-flashair-by-tiny-computer">Further explore of FlashAir by Tiny Computer</a></h2>
                            <small class="authorship">6 months ago  &bull;
                                <a class="gray-link" href="/project/4201/log/14635#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-14635"><p>As I presented, FlashAir by TOSHIBA has a tremendous potential for IoT device realisation. I would show several information, how to utilize the real value of FlashAir by standalone ATMega1284P. These information would be also useful for Arduino Mega users.</p><p>*** I have no relation with TOSHIBA, just a personal user. I will not get any benefit from TOSHIBA (unfortunately) even if my information helps its sales..</p><h2>How to explore and utilise FlashAir not just only as SD Card sized Web server and wireless AP</h2><p>Step 1: Modify "CONFIG" file in the invisible directory on FlashAir</p><p><img src="https://cdn.hackaday.io/images/3165221425302401594.jpg"></p><p>"IFMODE=1" should be added on CONFIG. By adding this line, GPIO control of FlashAir is activated. Please note that GPIO control is activated <strong>IF</strong> FlashAir is <strong>NOT</strong> initialized by host. (In other words, if you insert some computer and if you see the picture inside, GPIO mode is deactivated. But GPIO mode is automatically activated if it is connected to some interface with iSDio procedure described below..  )</p><h2>Step 2: Access official website to get further info.</h2><p>Read through</p><p><a href="https://flashair-developers.com/en/documents/tutorials/arduino/1/">Controlling FlashAir with Arduino</a> and <a href="https://flashair-developers.com/en/documents/tutorials/arduino/2/">Creating the iSDIO extension command</a>.<br></p><p>In the end of <a href="https://flashair-developers.com/en/documents/tutorials/arduino/2/">Creating the iSDIO extension command</a>, you can get the modified SD card library for Arduino IDE 1.0.5. Download and install your local library...</p><h2>Step 3: Additional zip package download from Japanese written file.</h2><p>I don't know why but "tutorial 3" of FlashAir is released by only Japanese version. Download "arduino_tutorial_03.zip"  of the bottom of the page of <a href="https://flashair-developers.com/ja/documents/tutorials/arduino/3/">page.</a></p><p>Step 4: Modify several files in iSdio in your installed Arduino library.</p><p><img src="https://cdn.hackaday.io/images/7127771425303135039.jpg"></p>You can see which file should be modified from the time stamp. The files modified "today" should be modified if you don't use Arduno UNO. Indeed, you don't touch "Sd2Card.h" if you are using hardware SPI of ATMega, if not, please read the file and find the point, but "Sd2PinMap.h" is absolutely required to modify not with UNO.<p><img src="https://cdn.hackaday.io/images/2241931425303253773.jpg"></p>It is similar to the case of original SD card library of Arduino, but this library does not know how to handle "ATMega1284P", so the notation, "AVR_ATmega1284P" should be added as above. Now the mod of library is done....<p><img src="https://cdn.hackaday.io/images/1245161425303492203.png"></p><p>CS in sample file "arduino_tutorial_03" should be modify to fit your configuration. and</p><p><img src="https://cdn.hackaday.io/images/4847921425303572216.jpg"></p>Comment out "while (!Serial){} part and need "delay(5000)" to see the full information of flashAir.....<p>Lastly the result...</p><p><img src="https://cdn.hackaday.io/images/5661621425303705053.jpg"></p>With all the preparation above, you can access all the internal information of FlashAir. Now you are in the entrance of "wonderful FlashAir world". You can find a lot of report on youtube by keyword "GPIO FlashAir", have fun!</div>

As one example, I implemented twitter message compose/sending function. One additional command to TinyBasicPlus, "twit" is added and simply twit "the message you want to send" will send on Twitter!</p><p>The corresponding arduino sketch is uploaded on <a href="https://github.com/kodera2t/TinyBasicPlus_OLED_support/blob/master/Ada_ATMega1284_LCD_Twit_R/Ada_ATMega1284_LCD_Twit_R.ino">github!</a> containing "twit" command and "netinfo" command, showing IP address obtained through DHCP.</p><p>This sending function is realised as a "command" so for example,<br></p><pre class="hljs cpp"><span class="hljs-number">10</span> <span class="hljs-keyword">for</span> i=<span class="hljs-number">1</span> to <span class="hljs-number">1000</span>
