                        <li>
                            <h2><a href="/project/6062/log/22490-first-prototypes-assembled-tests">First prototypes assembled + tests</a></h2>
                            <small class="authorship">8 days ago  &bull;
                                <a class="gray-link" href="/project/6062/log/22490#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-22490"><p>Hey there,</p><p>As promised, I came back from holidays last Sunday afternoon, and as soon as I got home I took my soldering iron and started assembling one proto. I had some issues cause I only had 1.2mm solder that slowed me down a lot (it took me 4 hours to get it done), but when I connected it to the Atmel ICE it was promptly recognized. Good.</p><p>Programming the bootloader worked too, but then JACK won&apos;t enumerate.</p><p>Monday morning I had to work, so I took my first proto with me along with the Atmel ICE debugger; I fiddled with the Arduino Zero bootloader first, to switch the bootloader pin to anything diffferent than PA15 which is used on my board as input for the 12.288 MHz crystal, I modded the makefile to point to correct directories, and TADA, I now have a JACK bootloader; I added a JACK board definition in the boards.txt file of the SAMD Arduino devices, opened Arduino, flashed the bootloader and and the device DID enumerate, as an &quot;Arduino Zero bootloader&quot; device (just like what you&apos;d except when connecting a Zero&apos;s native USB port, not the EDBG one). I could also flash a Blink sketch, but it won&apos;t start automatically and randomly if I touched some pins it would eventually start. I thought that this issue surely is connected to the clock not being the same as the Zero.</p><p>Monday afternoon I went home early to start assembly of three others boards (the judges&apos; boards); before that I bought a 0.5mm solder that helped me A LOT during production, it took me in fact 4 hours to make THREE boards (that&apos;s 300% faster than Sunday). I had some issues on two boards for the USB transient suppressor chip, on one I messed up 2 of 6 pins of that IC :/ In fact, these boards are still recognized by the Atmel ICE, but one of them won&apos;t show up at all as a USB device, the other would report &quot;Unknown device&quot;. I&apos;ll try to reflow these chips using an hot air gun, if that does not work, I&apos;ll take off some parts and solder them on two spare boards</p><p>Photos here:</p><p><img src="https://cdn.hackaday.io/images/6050491439284973611.jpg">So far so good.</p><p><img src="https://cdn.hackaday.io/images/8338731439285026114.jpg"></p><p>Almost there...</p><p><img src="https://cdn.hackaday.io/images/2081351439285642348.jpg"></p><p>A pair of finished boards, one connected to the Atmel ICE.</p><p>You can also find a video of the device in action (connection to Atmel ICE, bootloader flashing, enumeration as Arduino Zero, programming of Blink sketch, LED blinking):</p><iframe style="width: 500px; height: 281px;" width="500" height="281" src="//www.youtube.com/embed/Me_bltUciJQ" frameborder="0" allowfullscreen=""></iframe><p>To solve the issue that I had yesterday I indeed had to change the Clock Generators; the Zero takes the main oscillator from an external 32.768 KHz crystal while I&apos;m using a 12.288 MHz crystal; these crystals have separate pins. I then reconfigured the Zero startup.c file to use XOSC rather than XOSC32K as main source and you can see in the video that as soon as I program the device with the Blink sketch, it loads it. </p><p>I still have a software issue, that is when I flash JACK with a sketch it won&apos;t enumerate anymore as an &quot;Arduino Zero bootloader&quot; COM port so I cannot program it again. I also enabled an option that if you tap twice the RESET button the bootloader is forced to load again, but it won&apos;t show up either.</p><p>See you soon for other updates.</p><p>Cheers,</p><p>Mick</p></div>

I had a small issue on the LED that was not working, but I just had to add some solder on its pads and on the 68 Ohm resistor pads and it blinked correctly.</p><p>Now, for the guides, please take a look at the details of the project, which I&apos;ll update in a few minutes.</p><p>Cheers,</p><p>Mick</p><h2></h2><p style="margin-left: 40px;"><strong></strong></p></div>

Let&apos;s start with what works:</p><h2>PWM and Analog inputs:</h2><h2><iframe width="500" height="281" src="//www.youtube.com/embed/ujpMQGawmno" frameborder="0" allowfullscreen=""></iframe>I2C<iframe width="500" height="281" src="//www.youtube.com/embed/wUb7It0z8H4" frameborder="0" allowfullscreen=""></iframe>SAM-BA bootloader (double tap RESET function to run bootloader mode)</h2><p><iframe width="500" height="281" src="//www.youtube.com/embed/lm4ly-gQs4Q" frameborder="0" allowfullscreen=""></iframe></p><p>I tried SPI with the Sparkfun Micro OLED module and its companion library but it gave me some compiler errors, I tried a quick-and-dirty fix for that but even after that the SPI bus seemed like plain dead, no signal on no pin (MOSI, SCLK, CS).
