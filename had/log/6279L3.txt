                        <li>
                            <h2><a href="/project/6279/log/20068-server-side-and-web-frontend">Server side and WEB frontend</a></h2>
                            <small class="authorship">2 months ago  &bull;
                                <a class="gray-link" href="/project/6279/log/20068#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-20068"><blockquote>Update Tue Jun 30 00:31:40 UTC 2015</blockquote><p><img src="https://cdn.hackaday.io/images/7900991435427015329.png"></p><p>I am working at the server side technologies, above there is my last conceptual work, some details could change between today and tomorrow but we are almost there. The technologies involved are:</p><ul><li><strong>WAMP on websockets</strong> </li></ul><p style="margin-left: 40px;"> crossbar.io at the early stage of the deployment, after will be more probable switch to something in erlang but still WAMP. It also offers tokens for the authorization before the websocket handshake</p><p style="margin-left: 40px;"><a target="_blank" href="http://wamp.ws">The WAMP architecture</a><br> </p><ul><li><strong>no cookies to keep the session at the browser side</strong></li></ul><p style="margin-left: 40px;">I want to implement something new, the WAMP will provides tools to track the user and his session</p><ul><li><strong>REST API to push telemetry via HTTPS from the IOT sensors or robots</strong></li></ul><p style="margin-left: 40px;">This will be in pure python: falcon or wheezy.web, after I will switch to something in erlang probably. The REST API front end offer a solid PUSH only new data interface and very fast. I calculated to support at least 30K requests for second.<br></p><ul><li><strong>Postgresql </strong></li></ul><p style="margin-left: 40px;">Main DB for the data of sensors, rock solid, proven and becoming very versatile these days, I still have to decide few things but I will play with NOSQL schemaless datastore at first, the data will be stored probably in JSONB. I still have to deeply verify if we can pass JSONB to the javascript in browser. </p><ul><li><strong>Redis as auth. token DB store, it will keep the tokens until the ws is handshaked</strong></li></ul><p style="margin-left: 40px;">This is a security extra layer, before establish the websocket handshake, the browser needs to send a token received at the first contact with the server, REDIS DB will keep temporary the tokens until they get used with the handshake.</p><ul><strong></strong><li><strong>in browser HTML rendering of the templates</strong></li></ul><p style="margin-left: 40px;">This is my challenge, it is more than one year I see new javascript framework to process HTML templates browser side, this means they are not generated server side, the server is just a data streaming backend. It should e useful to keep an impressive amount of requests on.</p><ul><li><strong>Bootstrap responsive theme with HTML5 and CSS/JS</strong></li></ul><p style="margin-left: 40px;">I am evaluating several admin panels based on bootstrap, at the moment the more interesting are:</p><p style="margin-left: 40px;">Ultimo admin <a target="_blank" href="http://canvaskite.com/marketplace/ultimo/index.html">link</a></p><p style="margin-left: 40px;">AdminLTE <a target="_blank" href="https://almsaeedstudio.com/themes/AdminLTE/index2.html">link</a></p><p style="margin-left: 40px;">further templates could be added tomorrow.</p><h1><br></h1><h1>WAMP, it's time to play..</h1><p>I wanted to see how the things work, so after installing <a target="_blank" href="http://crossbar.io">crossbar.io</a>, I began to look at the code of the example project and read the <a target="_blank" href="http://crossbar.io/docs/">online documentation</a>, I changed configuration parameters and code to understand better how it works. There is not to much to see about underneath protocol and WAMP because everything is raised to  an higher level of abstraction and it just works but some hacking is still good...</p><p><img src="https://cdn.hackaday.io/images/3653101435623563463.png"></p><p><a target="_blank" href="http://msgpack.org/"></a></p><p><img src="https://cdn.hackaday.io/images/7479741435623890426.png"></p><p>Now my concern is about the security I could build on top of this system so I am interested to the authorization and authentication features provided by crossbar.io</p><p>The current documentation on github explain about the <a target="_blank" href="http://crossbar.io/docs/WAMP-CRA-Authentication/">CRA authentication method</a>, it is interesting, I could write an authenticator in python able to retrieve the secret from a DB to let crossbar.io to perform the authentication BUT some clever guy on github suggest to implement <a target="_blank" href="https://github.com/tavendo/WAMP/issues/135">WAMP-SCRAM</a> instead because more secure.</p><p>I am exactly there...no documentation yet...reading  the source code to understand how to use it...sure will be a long night again :)....see you tomorrow...cheers<br></p></div>
                            </div>
