                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">The most recent implementation of the nRF51822 BLE Flex Module is based on the Molex 3dB antenna.  It includes 7 analog input/GPIO and 13 GPIO castellation pins, 3 GPIO driving an optional RGB LED, a reset button, and a 32k crystal to reduce power consumption.  The circuit is shielded, has separate ground planes, and has a ferric bead filter on VCC.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>This is our main communication <a href="http://hackaday.io/project/2236-Flex-Modules" target="_blank">Flex Module</a>.&nbsp; Bluetooth BLE will be the primary way to interact with all <a href="http://hackaday.io/project/2236-Flex-Modules" target="_blank">Flex Modules</a>.</p><p>These boards are alpha quality, they will likely not work yet.&nbsp; The design is in <a href="http://www.diptrace.com/">DipTrace</a>.&nbsp; The design will require some tuning.&nbsp; This module is an nRF51822 with balun and chip antenna.&nbsp; I wanted to work with normal board thicknesses instead of more common module designs around a PCB trace antenna which requires a much thinner board.&nbsp; I am testing with 1db, 2db, and 3dB chip antennas in a variety of layouts.</p><p>Programming is supported through SWD utilizing the 6-Pin Tag-Connect <a href="http://www.tag-connect.com/catalog/6">TC2030-IDC</a>.</p><hr><p><strong>Design Notes:</strong></p><p><br><strong></strong></p><p><strong>Ground Stitching</strong></p><p>2.45Ghz frequency has a wavelength of 12.08cm.<br>Recommended ground stitching distance is wavelength/20 or 6.04mm</p><p><strong>Antenna Feedline</strong></p><p><a href="http://johansontechnology.com/" target="_blank">Johanson Technology</a> recommends a coplanar waveguide<br>(Calculated using <a href="http://www.awrcorp.com/products/optional-products/tx-line-transmission-line-calculator" target="_blank">TXLine</a> and OSHPark board specs):</p><p>4layer: dielectric constant3.665, loss tang 0.012, copper, ~51ohm, 2.45ghz, height 6.7mil, thickness1.4mil, gap 6mil = <strong>11.2756</strong>mil</p><p>2layer: dielectric constant 4.6,loss tang 0.016, ~51Ohm, 2.45ghz, height 1.6mm, thickness 1,4mil, gap 6mil = <strong>29.0102</strong>mil</p><p>The solder mask will lower the impedance slightly, so instead of 50Ohm, I calculated with 51Ohm.<br><br>I original attempted a microstrip design, but the width was around 110mil for 2 layer boards.&nbsp; Surrounding the trace with ground plane is much more sensible.&nbsp; The distance to the Antenna pad from the trace line is copied from the reference design.&nbsp; The balun should do most of the matching.&nbsp; I would recommend adding a microstrip for tuning, but until I have a chance to test with a Network Analyzer, it is pointless.</p>
                                
                            </p>
                        </div>

                            <h2><a href="/project/2239/log/13186-new-revision">New revision</a></h2>
                            <small class="authorship">7 months ago  &bull;
                            <a class="gray-link" href="/project/2239/log/13186#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>I have finally been able to update this board.  The BiPower OTG and Qi/USB Charger modules are mostly done.  So I know what the pin outs will look like (which held this board back).  So I should be sending a new design with the 3dB antenna and shield to OSHPark this weekend.  The design just fits the Tag Connect pogo programmer and antenna just outside the shield.  I also added optional components - a reset button, an RGB LED (in shield), and a 32k crystal.<br></p>
                                
                            </p>
                        </li>
                    
                        <li>
                            <h2><a href="/project/2239/log/11735-design-updates">Design updates</a></h2>
                            <small class="authorship">8 months ago  &bull;
                            <a class="gray-link" href="/project/2239/log/11735#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>As with all the other Flex Modules, I will be improving the castellation design shortly.  In addition, I may try to add more GPIO pins.  As the shield and the 3dB Molex antenna are similar heights, I can get away with closer trace routes with that antenna.  So in addition to it being probably the highest gain chip antenna (especially for the price), it also improves the design.  I'll also likely move to a 4 layer stack up, so I need to recalculate the co-planar waveguide.  We should be able to start selling these when ready in the next month or two as a Flex Module is a "subassembly" not requiring certification.  We will try to get it certified when it becomes popular.</p>
                                
                            </p>
                        </li>
                    
                        <li>
                            <h2><a href="/project/2239/log/7081-prefab-shielding">Prefab shielding</a></h2>
                            <small class="authorship">a year ago  &bull;
                            <a class="gray-link" href="/project/2239/log/7081#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>Hi, we have been comparing footprints and placement.&nbsp; At this point some of the smaller prefab almost fit, but not quite.&nbsp; The big issue is the 2.54mm sized castellations.&nbsp; They are too close for even the smallest shields.&nbsp; Between that and antenna placement.&nbsp; We may need to adjust the overall board layout.&nbsp; Almost all common nRF51822 modules have shields, but they also have really poor antenna placement (according to manufacturer recommendations).&nbsp; For now, we will not include component shielding.</p>
