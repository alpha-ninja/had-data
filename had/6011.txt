                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">This project is to design an open source embroidery machine. Current embroidery machines are very expensive and the average consumer can only afford a cheap version that really a modified home sewing machine. <br /><br />3d printers have show that a community of people can develop a low cost and easy to build version of very expensive commercial machines. An embroidery machine is very similar to these commercial 3d printers. They have a moving XY bed, and needle assembly. This is not very different from 3d printers with an XY bed and extruder. <br /><br />My goal is to develop an open source mechanical design, electronics and software so that people can build an embroidery machine in the same manner that people are building 3d printers.</div>
                </div>

                            <h2><a href="/project/6011/log/19111-research">Research</a></h2>
                            <small class="authorship">2 months ago  &bull;
                            <a class="gray-link" href="/project/6011/log/19111#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>I have been doing some research to try and figure out the necessary parts. Their is a really good description on the SmoothieWare website with a lot of details.</p><p><a href="http://smoothieware.org/embroidery" target="_blank">smoothieware embroidery</a><br></p><p>The part that I am struggling with right now is to figure out the needle assembly and how to either build my own or finding a suitable replacement part that I can hack. Below is link that show the mechanism while the needle plunges.</p><p><a href="http://sewingschool.org/2010/11/04/how-a-bobbin-works/" target="_blank">How a Bobbin Works</a><br></p><p>I have been told that they call the needle assembly an Embroidery Foot. It seems like it should be reasonable to find a replacement foot for an embroidery machine. There are quite a few industrial companies out there so the trick would be to find one that will work for this project. So I am currently looking into finding a suitable source for these parts. </p><p>Also it seems like there are really two styles of embroidery machines. The home user one style and the commercial style. The main difference as far as I can tell is that the home user one is basically an upgraded sewing machine and can only ever do 1 color of thread at a time. I think if you were to try to build this style you could just get a cheap sewing machine and hack it together. </p><p><img src="https://cdn.hackaday.io/images/6395111433610272856.jpg"></p><p>The commercial versions look a little different. I think this is the design that makes more sense to try and build as an open source design. It can support many more needles at the same time and if you could develop an open source design you could make it modular and scalable. You can start with a single needle and over time add more. So this is the design I am gonna work toward building.</p><p><img src="https://cdn.hackaday.io/images/3416871433610488085.jpg"></p>Software:<p>I also learned a little bit about the software. Almost all embroidery machines accept the DST file format. So the software should be able to accept this format.</p><p>I also did some reading and the design should allow for a cap attachment, so you can embroider on round objects. Shouldnt be too hard If we keep that in mind from the beginning.</p><p>Random Links to Useful bits of information</p><p><a href="https://grabcad.com/library/arm-for-sewing-machine-juki--1">Sewing Machine Arm</a><br></p><p><a href="http://www.alibaba.com/product-detail/Embroidery-Machine-head-parts_1411840432.html">http://www.alibaba.com/product-detail/Embroidery-Machine-head-parts_1411840432.html</a><br></p><p><a href="http://www.alibaba.com/product-detail/Chinese-Embroidery-Machine-Parts_51047854.html" target="_blank">http://www.alibaba.com/product-detail/Chinese-Embroidery-Machine-Parts_51047854.html</a><br></p><p><a href="http://www.emblibrary.com/el/help.aspx?page=WhichSize">List of Machine Field Sizes</a><br></p><p>These two links have a lot of good details. </p><p><a href="http://homeappliance.manualsonline.com/manuals/mfg/brother/pr650.html" target="_blank">Brother PR-650 Operation Manual</a><br></p><p><a href="http://homeappliance.manualsonline.com/manuals/mfg/janome/mb4.html" target="_blank">Janome MB-4 Operational Manual</a><br></p>

                        <li>
                            <h2><a href="/project/6011/log/18627-in-the-beginning">In the beginning...</a></h2>
                            <small class="authorship">3 months ago  &bull;
                            <a class="gray-link" href="/project/6011/log/18627#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>This is the beginning of the project. I am currently investigating the details and develop the specs of what this machine will need to do. I am not exactly sure how embroidery machines work beyond having seen one or two running. But I see this project as being split into 3 main areas.</p><ul><li>Mechanical Design<ul><li>The mechanical design will consist of a frame, xy table, and needle assembly. I believe the needle assembly will be the most complicated mechanical assembly. While building one from scratch would be ideal, it might be possible to find replacement parts for an off the shelf machine to start with.</li></ul></li><li>Electrical<br><ul><li>The electrical should be fairly similar to a cnc mill or 3d printer. A microcontroller to run everything, some stepper motor drivers, possibly a dc motor for the needle, some encoders on the needle, and perhaps a few other bits. Options include using an arduino mega with RAMPs board, Smoothie board, Beagle Bone Black, or some other board. It is probably quite easy to use any of these and make it work.</li></ul></li><li>Software</li><ul><li>There will need to be some digitizing software to take an image file and convert it into the standard format for embroidery machines. I would like to be compatible with current standards for embroidery machines. The software for embroidery machines is quite expensive. If the machine firmware can read the standard format, then you could generate the code using any method you like. This allows the user to generate the code in whatever method you like. If someone has a old embroidery machine but doesn't have the software then they could use the open-source software to run their machine. It just makes sense to follow the standard format.</li></ul></ul>
                                
                            </p>
                        </li>
                    
                    </ul>
