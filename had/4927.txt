                    <h5>Description</h5>
                    <div class="description">This project solves multiple problems facing the new x86 console gaming initiatives by Valve, Nvdiia, and Alienware.  Most obviously,  it would require 3 boxes to cover all the gaming options that x86 gaming promises. Some other solutions have been presented such as streaming from another machine, or dual booting. These options are viable for some, but not all. <br />For x86 gaming to effectively compete with current consoles, the same plug & play experience should be offered by a single box that TRULY rules the livingroom.  This runs not just 3, but as many OS's as you care to add - up to a ridiculous 512. Mouse, keyboard, headphones, & USB all from the controller. Get your exercise elsewhere, since you don't need to leave the couch. Streaming of your game to the controller, additional game screens AT the controller.  <br /><br /><br /><br /><br /><br /><br /><br /><br /><br /></div>

                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>The Multiheaded Hydra ran 6 seats of SteamOS (or whatever other x86 OS) but it wasn't a living room friendly size. So the Hydra Gaming Console is 15" x 12" by 2 ½".It's built around an Asrock H77M mini ITX with a Nvidia Quadro FX3800 desktop GPU. Right now it's running SteamOS, Windows7, and Android x86 KitKat. I've also been intending to switch back to VM control via an Android app. </p><p>And it hit me that an Android cell phone would probably make a pretty good Game Controller. It has:</p><p> Touchpad is your mouse </p><p>Keyboard for logging in, etc</p><p>WiFi</p><p>Bluetooth</p><p>Haptic feedback/Vibrator</p><p>Accelerometer/Tilt</p><p>Cameras</p><p>USB on the go</p><p>Headphone jack</p><p>Display</p><p>Easy dev platform</p><p>Speakers</p><p>Game Integration<br></p><p>Etc</p><p>And they're CHEAP. My biggest problem now is that there are SO many ways to skin the cat here, designing a controller around a phone is going to take some time. I think the USB OTG offers the best option, sort of like a "spread" controller that clamps the phone and plugs in the end. I have used a USB hub to connect both mouse &amp; keyboard and sent them to the VM. Also a gravis gamepad. I was thinking something along the lines of this failed kickstarter:</p><p><img src="https://cdn.hackaday.io/images/4500671427305962146.jpg"></p><p> Depending on the BT stack on the phone, the phone can be a HID device. I currently have the wrong phone, so am unable to test. </p><p>There are also many ways to emulate a controller on the phone. </p><p>There's also the Moga controller, which may be hackable, but I wish it was USB. It certainly works in the Android VM. </p><p>So what I have done for now is build a cell phone mount on an Xbox 360 controller, with the Xbox PC Wireless Adapter plugged into a passed thru USB on the host. The cell phone runs a menu that controls the host, selects which VM to run, which one comes up at boot, manage snapshots, and install a new VM from scratch. </p><p> Like this:</p><p><img src="https://cdn.hackaday.io/images/8186841427306138667.JPG"></p><p> And to make a proper Game Console, I have to display the Menu system on the TV. So, the Quadro goes in port 1 of an HDMI switch inside the Host. A Microsoft Miracast device is in port 2. The HDMI switch is controlled by an Arduino that also controls the Hosts power switch. The cell phone screen is casted to the Miracast, and switching is controlled over bluetooth. I added a 1k resistor from the hot plug detect to 5v (pin 18 &amp; 19) so the Miracast always thinks it's connected .</p><p><br><img src="https://cdn.hackaday.io/images/9203031427306366306.jpg"> So you power on the Hydra Game Console from the Controller App. The cell screen is displayed on the TV, with a progress bar while the hypervisor boots. When port 5901 is active on the Hydra, we VNC in and can access the bios of the VM, or monitor the txt mode booting (Just in case we end up up in Startup Repair or some other boot prob) . We are also monitoring the Vmkernel.log for the event indicating the Nvidia GPU is active, and then the HDMI is switched to the Quadro. </p><p> Play Games, shut down, and the HDMI switches back to the menu presenting other VM's to run + options. Use the cell as a pointer &amp; keyboard. Or plug real mouse &amp; keyboard in the USB OTG. Use the Xbox controller in games. </p><p>Future development will allow streaming games from the Hydra Game Console to the phone, Mute the TV and use headphones on the phone, and using the phone as an auxiliary display in games. Also fully utilizing the accelerometer, cameras, vibrator, etc</p><p>I also need to buy (or most likely make) some custom cables to bring the designated ports out the back neatly. </p><p>Here is a video, demonstrating current functionality.  I boot the machine, and switch between the 3 OS's. I play a little of a game in each. All of this entirely from the controller:<iframe style="width: 500px; height: 281px;" src="//www.youtube.com/embed/7cov4LHAduo" allowfullscreen="" frameborder="0" height="281" width="500"></iframe></p><br>

                    <div class="section section-instructions">
                        <h5>Build instructions</h5>
                        
                        
                            <ul class="post-content section-instructions-list">
                                
                                    <li>
                                        <span class="instruction-number">1</span>
                                        <div class="instruction-list-item"><p><strong>1) Gather the parts</strong></p><p> Start with just enough to enable &amp; test PCI Passthrough. </p><p>So you need :</p><p><strong>VTD capable system board</strong> (Mini-ITX for the Game Console version) </p><p>I am currently using an Asrock H77M-ITX, which is hard to find now. The H81M-ITX/WiFi should work, as I have verified in the manual.  I have been sticking with Asrock, they seem to reliably support VTD. The option is under "Chipset Configuration" in the bios, according to the manual here: </p><p><a target="_blank"></a><a target="_blank"></a><a target="_blank"></a><a target="_blank"></a><a target="_blank"><a target="_blank">ftp://66.226.78.21/manual/H81M-ITXWiFi.pdf</a></a></p><p>The other thing to verify is that the GB Lan is supported (or has a workaround) by ESXi</p><p>The H81M-ITX/WiFi uses the RTL8111 which is NOT officially supported, but there is a workaround. If you intend to use another board, check the Lan interface and Google like this:</p><p>RTL8111 ESXi Vib</p><p>to verify a viable workaround, if it's not natively supported. VMware seems to adjust the list of natively supported NIC's to include a smaller set with each revision. So, you can use the driver from 5.0 in 5.5, etc.  Intel NIC's are the most commonly supported, since this is a Datacenter product. <br></p><p><strong>VTD capable Processor</strong> </p><p>If you use the Asrock board, check the "CPU Support List" and pick at least an i5. </p><p>Like the i5-4440 I have listed, just put i5-4440 in Google and pick the Intel ARK link:</p><p><a href="http://ark.intel.com/products/75038/Intel-Core-i5-4440-Processor-6M-Cache-up-to-3_30-GHz">Intel ARK i5-4440</a></p><p>And you see "<span class="tooltippable">Intel® Virtualization Technology for Directed I/O (VT-d) "  under "Advanced Technologies" and "yes" in the right column. </span></p><p><strong>Video Card </strong> - You can use an ATI Radeon as low as an HD-4650 for testing. I'm using an Nvidia Quadro FX3800, but I do have some HD 4650's and 6450's since I know they work. I was initially unsure of the Quadro. It helps to have a known working Mobo/Proc combo and known good GPU.  </p><p>Obviously, some RAM (8GB),  PSU, etc. Have another system nearby to run the VSphere client. A hard drive of at least 250GB for VM storage. You can install the Hypervisor to a USB stick of 2GB+, but it will boot slow from that. I am using a 2GB SSD. You don't need this, but it allows you to switch to bigger storage without re-installing the Hypervisor. </p><p>Download the VMware VSphere Hypervisor. I am using 6.0, which is still BETA. </p><p>I am using this because 5.1 and 5.5 have issues with USB passthrough &amp; autostarting VM's. I would also like to be using the latest, rather than dropping back to 5.0. </p><p>6.0 does not have these issues, but you may not be able to get it yet unless you participate in the BETA. And I can't provide it here. SO if you can't get 6.0, get 5.0. You will be limited to 32GB of RAM with 5.0........</p><p>The Download will be an ISO, burn a cd from the bootable image. Have some extra blank cd's just in case.....</p><p>So you should have a VTD capable Mobo/Proc/GPU, a destination for your VM's &amp; Hypervisor, and VSphere Hypervisor on CD and ISO. </p><p>You also should have a cd drive of some kind. A USB CDROM is what I used<br></p></div>
                                    </li>
                                
                                    <li>
                                        <span class="instruction-number">2</span>
                                        <div class="instruction-list-item"><p><strong>2) Install VSphere Hypervisor</strong><span></span></p><p>You may run into some trouble here......So you need to use a tool to inject the RTL8111 (or whatever) driver into your ISO. Get the "ESX Customizer" here:</p><p><a href="http://vibsdepot.v-front.de/tools/ESXi-Customizer-v2.7.2.exe" target="_blank">http://vibsdepot.v-front.de/tools/ESXi-Customizer-v2.7.2.exe</a></p><p>And get the Vib for your NIC such as:</p><p><a href="https://tinkertry.com/files/VMware_bootbank_net-r8168_8.013.00-3vmw.510.0.0.799733.vib" target="_blank">https://tinkertry.com/files/VMware_bootbank_net-r8168_8.013.00-3vmw.510.0.0.799733.vib</a></p><p><a href="http://www.createinnova.com/Shares/net51-drivers-1.0.0-1vft.510.0.0.799733.x86_64.vib" target="_blank">http://www.createinnova.com/Shares/net51-drivers-1.0.0-1vft.510.0.0.799733.x86_64.vib</a><br><a href="https://tinkertry.com/files/VMware_bootbank_net-r8168_8.013.00-3vmw.510.0.0.799733.vib"></a></p><p>and run the customizer:</p><p><img src="https://cdn.hackaday.io/images/7189761428080339559.JPG"></p><p>And your new ISO with the NIC driver will be in the working directory you set. </p>Burn the CD and boot from it. Aim the install at the 2GB SSD (if you have one, if not the HDD is ok). The install is pretty self explanatory, just follow the steps and reboot like it says when you're done. It will require you set a password, don't go crazy here since it's a GAME machine after all. Install takes maybe 15 min. <p>When it comes back up, press f2 and set a STATIC  IP on your local NET (like 192.168.1.xx). </p></div>
                                    </li>
                                
                                    <li>
                                        <span class="instruction-number">3</span>
                                        <div class="instruction-list-item"><p><strong>3) Set UP PCI-Passthrough</strong></p><p>Reboot to the BIOS and set these items:</p><p>Onboard GPU must be OFF so disable:</p><p>IGPU Multi-Monitor</p><p>Render Standby</p><p>Deep Render Standby</p><p>For some reason the HDAUDIO must also be disabled</p><p>Set primary GPU to PCIE</p><p>Enable VTD</p><p>On your other PC go to the IP of your ESXi box in a browser and you will get this:</p><p>If you are using ESXi 5.5 or above, you will need Win7 or higher. There is a workaround for XP, but it's beyond the scope of these instructions. <br></p><p><img src="https://cdn.hackaday.io/images/5105441428087077255.JPG"></p><p>Install and aim it at your static IP, login with root &amp; the password you set. </p></div>
                                    </li>
