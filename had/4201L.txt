                            <small class="authorship">6 months ago  &bull;
                                <a class="gray-link" href="/project/4201/log/14000#discussion-list">

                                </small>
                            <div style="position:relative">
                                <div id="post-body-14000"><p><img src="https://cdn.hackaday.io/images/444591423741460677.JPG"></p><p>I found yet another TinyBasic implementation for Arduino. It is "Toyoshiki TinyBasic", released a bit earlier to TinyBasicPlus. All the source is released on github:<br><a href="https://github.com/rutles/ttbasic_arduino">TOYOSHIKI Tiny BASIC for Arduino</a><br><br>This implementation is more strict to original <a href="http://en.wikipedia.org/wiki/Li-Chen_Wang#Palo_Alto_Tiny_BASIC">PALO ALTO TinyBASIC by Li-Chen Wang,</a> means zero extended command related to Arduino hardware (like dwrite and read), but code size size and its memory usage should be minimum. C-code expression is completely different but it may be interesting to compare with TinyBasicPlus implementation.<br></p><p>I quickly (indeed just ten minutes was enough, since the code has quite clear expression!) add my OLED code to TOYOSHIKI TinyBasic and released in the sub-folder of Github</p><p><a href="https://github.com/kodera2t/TinyBasicPlus_OLED_support/tree/master/toyoshiki-tinybasic" target="_blank">https://github.com/kodera2t/TinyBasicPlus_OLED_support/tree/master/toyoshiki-tinybasic</a><br></p><p>Please enjoy it!</p></div>

                            <small class="authorship">6 months ago  &bull;
                                <a class="gray-link" href="/project/4201/log/14228#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-14228"><p><img src="https://cdn.hackaday.io/images/5387261424264950259.jpg"></p><p><span></span>I made one additional variant for my small computer series. I made it with DIP package version of ATMega1284P and,,,, this is big advance, FTDI chip is added on the back of PCB! This version is made of single side ( pattern is just below PCB, not topside) PCB and much more easy to make. And one more big advance is,,, you never need AVR programmer or another arduino UNO to upload boot loader of ATMega1284P now! I put 4-jumper pins and when you need upload boot loader to brand-new (just bought, bare) ATMega1284, those pins should be shorted and just avrdude on command line make it as arduino-based-ATMega1284.<br></p><iframe style="width: 500px; height: 281px;" width="500" height="281" src="//www.youtube.com/embed/8m-W7x0qa0E" frameborder="0" allowfullscreen=""></iframe><p>All the information is uploaded on github:</p><p><a href="https://github.com/kodera2t/DIP_version_ATMega1284_computer-" target="_blank">https://github.com/kodera2t/DIP_version_ATMega1284_computer-</a><br></p><p>Have fun!!</p></div>

                            <h2><a href="/project/4201/log/14266-tinybasic-on-atmega328colorlcdspi">TinyBasic on ATMega328+ColorLCD(SPI)</a></h2>
                            <small class="authorship">6 months ago  &bull;
                                <a class="gray-link" href="/project/4201/log/14266#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-14266"><p><img src="https://cdn.hackaday.io/images/6447781424348250052.JPG"></p><p>Initially I made a LCD watch (real objective was to make some smart watch). It is made of ATMega328P and,,</p><p><img src="https://cdn.hackaday.io/images/3202921424348338106.JPG"></p><p>pressure sensor and RTC with backup battery is on backside. But its sketch size is close to ATMega328P limit,</p><p><img src="https://cdn.hackaday.io/images/6195411424348567524.jpeg"></p><p>and I gave up to complete the watch to add wireless function for real smart watch. </p><p>But now I installed TinyBasic for that...</p><p><img src="https://cdn.hackaday.io/images/4753631424348654935.JPG"></p><p>Indeed, small character is readable !( of course OLED is much better, though..) The most defect is its free memory space. With TinyBasicPlus + LCD driver and frame buffer occupies most of SRAM (poorly just 2 kB) of ATMega328! As you can see, it has just "296 bytes free memory!!!</p><p>This is not the binary size problem but the working memory shortage (SRAM),</p><p><img src="https://cdn.hackaday.io/images/94321424348887576.jpg"></p><p>actually still 4 kB is free for sketch...</p><p>But! The code is working with several fundamental function ( dwrite, aread,...) related to Arduino hardware and working with ATMega328P!</p><p>So if you have</p><p><strong>Arduino UNO</strong> and <strong>Adafruit 1.8' </strong><strong>SPI LCD (and its compatibles)</strong></p><p>, you can enjoy Basic on minimum configuration without PC monitor....</p><p>( Input is through serial console or modified Xbox360 chatpad)</p><p><a href="https://github.com/kodera2t/TinyBasicPlus_OLED_support/tree/master/ATMega328P%2BColorLCD">Arduino sketch is uploaded here!</a> Enjoy!</p><iframe style="width: 500px; height: 281px;" width="500" height="281" src="//www.youtube.com/embed/QxdI-itwXt0" frameborder="0" allowfullscreen=""></iframe></div>
