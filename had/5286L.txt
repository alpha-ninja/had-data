                            <h2><a href="/project/5286/log/16789-design-functional-considerations">Design &amp; functional considerations</a></h2>
                            <small class="authorship">4 months ago  &bull;
                                <a class="gray-link" href="/project/5286/log/16789#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-16789"><p>Mentioned below are key factors and constraints that will impact product components and overall design:</p><ul><li>board should have <strong>dimensions</strong> of 24x32mm and be <strong>compatible</strong> with the cheap, sub $10, eBay sourced nRF51822 board (it is a variation of <em>Core51822 </em>with additional quartz). This board provides BLE capability, ARM Cortex M0 CPU and some storage.</li><li>the <strong>footprint</strong> of 9DOF board, nRF51822<span class="redactor-invisible-space"> and battery all stacked together should not exceed nRF51822<span class="redactor-invisible-space">'s dimensions</span></span></li><li>it should be <strong>super power efficient</strong>. Hence, every sourced component must be selected against this criterion</li><li>Important part of the device is its storage capability. <strong>FLASH</strong> chip should take care of permanent storage. Minimum 16MB</li><li>In addition to flash, <strong>FRAM</strong> chip will be used. This is a very fast nonvolatile memory and does not have limit of repeated writes, thus will ideally suit as a counter storage or buffer memory</li><li><span class="redactor-invisible-space">always-on <strong>accelerometer</strong> chip for low-precision measurements</span></li><li><span class="redactor-invisible-space"><strong>9DOF multi chip module</strong> for precise motion detection</span></li><li><span class="redactor-invisible-space">3.7V Li-Po <strong>battery</strong> with small footprint</span></li></ul><p>Nice to have:<br></p><ul><li><span class="redactor-invisible-space"><strong>SPI expansion</strong> socket to include other sensors (ex. barometer )</span></li><li><span class="redactor-invisible-space">for visual feedback a single-colour <strong>LED</strong></span></li><li><span class="redactor-invisible-space">for tactile feedback a <strong>vibration</strong> <strong>motor</strong></span></li><li><span class="redactor-invisible-space"><strong>button</strong> will be used for input</span></li></ul><p>so .... <strong>Phase 1</strong> - DONE</p></div>
                            </div>
                        </li>
                    
                        <li>
                            <h2><a href="/project/5286/log/16835-component-selection">Component selection</a></h2>
                            <small class="authorship">4 months ago  &bull;
                                <a class="gray-link" href="/project/5286/log/16835#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-16835"><p>Based on the criteria specified previously we chose:</p><ul><li>Microchip <a href="http://ww1.microchip.com/downloads/en/DeviceDoc/20005158C.pdf" target="_blank">MCP1710</a> 3.0V current LDO regulator</li><li>Analog Devices <a href="http://www.analog.com/media/en/technical-documentation/data-sheets/ADXL362.pdf" target="_blank">ADXL362</a> ultralow power 3-axis accelerometer</li><li>Invensense <a href="http://www.invensense.com/mems/gyro/documents/PS-MPU-9250A-01.pdf" target="_blank">MPU-9250</a> 9-axis motion tracking multi-chip module</li><li>Winbond <a href="http://www.winbond-usa.com/resource-files/w25q256fv_revg1_120214_qpi_website_rev_g.pdf" target="_blank">W25Q256FV</a> 3V 256M-bit SPI Flash memory</li><li>Cypress Perform <a href="http://www.cypress.com/?docID=47959" target="_blank">FM25CL64B</a> 64k-bit SPI Fram memory</li><li>additionally we plan to use <a href="http://www.meas-spec.com/product/pressure/MS5611-01BA03.aspx" target="_blank">MS5611</a>, <a href="http://www.meas-spec.com/product/Pressure/MS5803-01BA.aspx" target="_blank">MS5803</a> and <a href="http://www.bosch-sensortec.com/en/homepage/products_3/environmental_sensors_1/bmp280/bmp280" target="_blank">BMP280</a> pressure sensors</li></ul><p>Smaller parts include:</p><ul><li>0603 smd components for optimal size vs. soldering difficulty</li><li>2.0 pitch 2x9 connector sockets for NRF51822 connection</li><li>6 pin smd socket for external SPI expansion (ex. pressure sensors)</li><li>21x32mm LiPo battery of around 120mAh</li><li>Tactile smt switch</li><li>cheap Vibration motor 3-4.5V 0.06A</li></ul><img src="https://cdn.hackaday.io/images/4065911429795051191.png"></div>
                            </div>

                        <li>
                            <h2><a href="/project/5286/log/19333-its-alive">it&#39;s alive !</a></h2>
                            <small class="authorship">2 months ago  &bull;
                                <a class="gray-link" href="/project/5286/log/19333#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-19333"><p>The components are responding! We used a logic analyser with <a href="https://www.saleae.com/downloads" target="_blank">Saleae software</a> to check for I/O digital signals and decode them on SPI.</p><p><em><a href="http://www.cypress.com/?docID=47959" target="_blank">FM25CL64B</a> Fram tests:</em><br><img src="https://cdn.hackaday.io/images/9134071434017288916.png" style="width: 513px; height: 314.77459749552776px;"></p><p><em><a href="http://www.meas-spec.com/product/Pressure/MS5803-01BA.aspx" target="_blank">MS5803</a> pressure sensor tests</em></p><p><img src="https://cdn.hackaday.io/images/1701151434017370289.png" style="width: 510px; height: 312.9778672032193px;"></p><iframe style="width: 500px; height: 281px;" width="500" height="281" src="//www.youtube.com/embed/v50u4TmCZvU" frameborder="0" allowfullscreen=""></iframe><p>But there were problems of course. Main ones we had to overcome:<br></p><ul><li>Bypass connectors helped in fixing the wrongly designed layout for <a href="http://www.winbond-usa.com/resource-files/w25q256fv_revg1_120214_qpi_website_rev_g.pdf" target="_blank">W25Q256FV</a> flash memory module.</li><li>Apparently pins P0.26 &amp; P0.27 for the nRF51822 are by default used for the 32kHz crystal. These we used for SPI communications, so again we had to solder bypass connectors. Pins P0.28 &amp; P0.29 worked great instead.</li><li>We did not connect VCCIO from the <a href="http://www.analog.com/media/en/technical-documentation/data-sheets/ADXL362.pdf" target="_blank">ADXL362</a> accelerometer near C6 capacitor, hence another bypass was required.</li><li></li></ul><p><strong>So looks like it's all working!</strong> Phase 6 &amp; phase 8 are <strong>DONE</strong>.<br></p></div>
                            </div>
