                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">Introducing the typhone: A DIY smartphone anyone can make. The typhone uses the Raspberry Pi for processing and has a 3.5in touchscreen. An Adafruit Fona is used to give the typhone the capability to make calls, send sms messages, and get the time using the built in RTC. It also has a 5mp camera module giving it the power to take HD photos and send them to Dropbox over WiFi. Speaking of Wifi, the typhone has a USB wifi adapter so it can communicate with the internet along with cellular networks. All this technology is enclosed in a 3D printed case. </div>
                </div>

                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><h2><iframe width="500" height="281" src="//www.youtube.com/embed/H2AY7ciEJvo" frameborder="0" allowfullscreen=""></iframe></h2><p>Note that in the video TYOS 0.1.0 is used. Many, many new features have been added since then.</p><h2>Hardware</h2><p>The typhone is made up of a few basic components:</p><ul><li>The Raspberry Pi</li><li>The cellular module (FONA)</li><li>The 3.5in TFT</li><li>3D Printed case.</li></ul><p>The Raspberry Pi handles all the processing, and connects everything together. The TFT talks to the Raspberry Pi over SPI, and the FONA talks to it over UART. Everything is powered with a 1200mah battery connected to the FONA. The FONA has a charging circuit perfect for use with the phone. Since the battery is only 3.7v, a boost converter is used to boost the voltage to 5v for the Raspberry Pi and TFT. A metal speaker and microphone is connected to the FONA for audio, along with an antenna. The FONA also has a real time clock built in, so the Raspberry Pi can keep time even when off wifi connection. The Raspberry Pi also has a USB Wifi Adapter connected to it giving it internet access. Right now all that it is used for is to upload camera photos to dropbox, but more features will be implemented in the future. The Raspberry Pi camera module is used for pictures and video.                             <h2><a href="/project/5083/log/23085-tyos-051">TYOS 0.5.1 </a></h2>
                            <small class="authorship">a day ago  &bull;
                            <a class="gray-link" href="/project/5083/log/23085#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>In previous releases of TYOS, the software depended on <a href="https://github.com/adafruit/adafruit-pi-cam">adafruit-pi-cam</a>, a point and shoot camera project from Adafruit. As you can see in the <a href="https://youtu.be/H2AY7ciEJvo?t=1m40s">original demonstration video</a>, the software is slow, pictures are low resolution, and only half of the display is used. All that has changed in the new release. </p><p><img src="https://cdn.hackaday.io/images/99691439748844116.JPG"></p><p><img src="https://cdn.hackaday.io/images/9605271439749005962.JPG"></p><p>The new camera app takes full advantage of the entire screen, loads pictures almost instantly, and instead of an endless &quot;Loading...&quot; prompt, a subprocess is created to handle the uploading to Dropbox to reduce waiting times by over half.</p>
                                
                            </p>

                        <li>
                            <h2><a href="/project/5083/log/19356-tyos-049">TYOS 0.4.9</a></h2>
                            <small class="authorship">2 months ago  &bull;
                            <a class="gray-link" href="/project/5083/log/19356#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>A couple days ago I released <a href="https://github.com/spadgenske/TYOS/releases/tag/0.4.9" target="_blank">TYOS 0.4.9</a>, below is a list and some pictures of the new features. Note that some of the features were actually released in TYOS 0.3.2.</p><h2>New Features</h2><ul><li>Error logs. When an error occurs it is stored in <code>tyos/logs/tyos.log</code></li><li>Audio config file. Switch your audio mode from built in (1) to external (0) in the config file<code>tyos/configure/audio.conf</code></li><li>Added debug mode to framebuffer.</li><li>More than four apps can be installed. New app list</li></ul><p><img src="https://cdn.hackaday.io/images/2769601434055233353.JPG"></p><ul><li>Settings app. Delete SMS messages and toggle on and off the headset for phones without built in speaker and microphone.</li></ul><p><img src="https://cdn.hackaday.io/images/9894551434055379792.JPG"></p><p>Along with that, over 15 bugs were squashed with some help from Todd Riesz. Thanks Todd!</p><h2>How to Update</h2><ol><li>Remove old TYOS build with <code>sudo rm -rf tyos</code></li><li>Download new edition with <code>wget <a href="https://github.com/spadgenske/tyos/archive/0.4.9.zip">https://github.com/spadgenske/tyos/archive/0.4.9.zip</a></code></li><li>Unzip package with <code>unzip 0.4.9</code></li><li>Rename with <code>mv TYOS-0.4.9 tyos</code></li></ol>
