                            <div style="position:relative">
                                <div id="post-body-20066"><p><img src="https://cdn.hackaday.io/images/9696981435426256492.jpg"></p><p><img src="https://cdn.hackaday.io/images/3939571435426319518.jpg"></p><p><img src="https://cdn.hackaday.io/images/6990281435426341726.jpg"></p><p><img src="https://cdn.hackaday.io/images/6839861435426363039.jpg"></p><p><img src="https://cdn.hackaday.io/images/6279821435426393431.jpg"></p><p><img src="https://cdn.hackaday.io/images/9464721435426422986.jpg"></p><p><img src="https://cdn.hackaday.io/images/9337521435426465581.jpg"></p><p><img src="https://cdn.hackaday.io/images/1731811435426483607.jpg"></p><p>To power the Arduino I choose Li Ion battery (18650).If you look at the Periodic Table, you will find that Lithium is on the far left in the first column, where all the most reactive elements live.So during handling care should be taken.</p><p><strong>Caution :</strong></p><p>You must take certain precautions when dealing with Lithium Ion Batteries. In order to maintain a very precise voltage when charging. The 3.7V batteries we're using in this guide need to have a charging voltage of 4.2V.A volt high or a volt low can mean an out of control chemical reaction which can lead to danger.</p><p>Don't worry a suitable Li Ion battery charge controller will solve the above problem.</p><p><strong>Circuit Connection :</strong><br></p><p>Solder the Input positive terminal of boost converter(red wire) and battery holder positive terminal(red wire) to the charging board's BAT + .</p><p>Solder the Input negative terminal of boost converter(black wire) and battery holder negative terminal(black wire) to the charging board's BAT-.</p><p>The boost converter output is a USB terminal. For powering a bread board circuit we need two wires for connection.So we have to modify something according our requirement.</p><p>The usb have four terminals (5V,D-,D+and GND).</p><p>Solder the red and black wire to the + and - respectively as shown in the back side of the boost converter.</p><p>Note : The boost converter do not have any marking.So use my picture during soldering.</p></div>

                            <h2><a href="/project/6477/log/20098-power-optimization-by-using-sleep-mode">POWER OPTIMIZATION BY USING SLEEP MODE</a></h2>
                            <small class="authorship">2 months ago  &bull;
                                <a class="gray-link" href="/project/6477/log/20098#discussion-list">

                            <div style="position:relative">
                                <div id="post-body-20098"><p><a id="photoset-link-FAUEWS2I7BQFB1G-0" class="photoset-link" href="http://cdn.instructables.com/F94/NWFR/HZQTF21P/F94NWFRHZQTF21P.LARGE.jpg"><img alt="Picture of POWER OPTIMIZATION BY USING SLEEP MODE" src="http://cdn.instructables.com/F94/NWFR/HZQTF21P/F94NWFRHZQTF21P.MEDIUM.jpg"></a></p><p><a id="photoset-link-F9NBEAHI7BQFB1H-1" class="photoset-link" href="http://cdn.instructables.com/FJY/2LJD/HZQTF1V4/FJY2LJDHZQTF1V4.LARGE.jpg"><img alt="temp_-758114554.jpg" src="http://cdn.instructables.com/FJY/2LJD/HZQTF1V4/FJY2LJDHZQTF1V4.MEDIUM.jpg"></a></p><a id="photoset-link-F0A1KQOI7BQFB1I-2" class="photoset-link" href="http://cdn.instructables.com/FB2/81UF/HZQTF2L2/FB281UFHZQTF2L2.LARGE.jpg"><p><img alt="temp_185183326.jpg" src="http://cdn.instructables.com/FB2/81UF/HZQTF2L2/FB281UFHZQTF2L2.MEDIUM.jpg"></p></a><p><a id="photoset-link-F3FDR0BI7BQFB1J-3" class="photoset-link" href="http://cdn.instructables.com/FWT/Q4DC/HZQTF2BM/FWTQ4DCHZQTF2BM.LARGE.jpg"><img alt="temp_1442858346.jpg" src="http://cdn.instructables.com/FWT/Q4DC/HZQTF2BM/FWTQ4DCHZQTF2BM.MEDIUM.jpg"></a></p><a id="photoset-link-FOS7ZE3I7BQFB1K-4" class="photoset-link" href="http://cdn.instructables.com/FEI/T88Q/HZS902PM/FEIT88QHZS902PM.LARGE.jpg"><p><img alt="temp_-595250227.jpg" src="http://cdn.instructables.com/FEI/T88Q/HZS902PM/FEIT88QHZS902PM.MEDIUM.jpg"></p></a><p>The weather data does not change frequently.So we can take reading at an interval of 5mins.As we are taking readings at regular intervals, it is a fantastic way to save lots of power. A system with appropriate sleep schedules can run for several months on just two AA batteries.We are so lucky that Arduino has several sleep modes that can be used to reduce power consumption.<br><br>This is most useful for any sensor networks.You can use this tricks in any of your stand alone sensor project.<br><br>After searching through the internet for using the sleep modes, I found a simple but powerful library by Rocket Scream has a Lightweight Low Power library supports all AVR power down modes. Each mode has an associated library method that lets you control sleep duration using the watchdog timer.For a novice programmer like me it is very simple and easy to use.<br><br><strong>How to use LowPower Library :</strong><br><br>1. Download the library from <a href="https://github.com/rocketscream/Low-Power">GitHub</a><br>2.Extract the zip file to the arduino library in your computer.<br>3.Import the library in your code. <br>4.Write the following one line code for power saving.<br>"LowPower.powerDown(SLEEP_1S, ADC_OFF, BOD_OFF) ; "<br><br>You can also pass different arguments to shut off individual peripherals. For different argument and sleep time refer the table provided by <a href="http://www.rocketscream.com/blog/2011/07/04/lightweight-low-power-arduino-library/">Lightweight Low Power Arduino Library</a>.<br><br><br><strong>example code :<br></strong><br></p><pre class="hljs coffeescript"><span class="hljs-comment">#include "LowPower.h"</span>
<span class="hljs-reserved">void</span> setup()
{ 
<span class="hljs-regexp">//</span> No setup <span class="hljs-keyword">is</span> required <span class="hljs-keyword">for</span> <span class="hljs-keyword">this</span> library 
} 
<span class="hljs-reserved">void</span> <span class="hljs-keyword">loop</span>() 
{ 
<span class="hljs-regexp">//</span> Sleep <span class="hljs-keyword">for</span> <span class="hljs-number">8</span> s <span class="hljs-reserved">with</span> ADC <span class="hljs-built_in">module</span> <span class="hljs-keyword">and</span> BOD <span class="hljs-built_in">module</span> <span class="hljs-literal">off</span> 
LowPower.powerDown(SLEEP_8S, ADC_OFF, BOD_OFF); 
<span class="hljs-regexp">//</span> Do something here 
<span class="hljs-regexp">//</span> <span class="hljs-attribute">Example</span>: read sensor, log data, transmit data 
}</pre><p><br><br>Lets use it in the blink code of arduino IDE example</p><p><br><br><strong>Apply "LowPower library" in Blink code</strong></p><pre class="hljs cpp"><span class="hljs-preprocessor">#<span class="hljs-keyword">include</span> "LowPower.h" <span class="hljs-comment">// import the lowpoer library</span></span>
<span class="hljs-keyword">int</span> led = <span class="hljs-number">13</span>; 
<span class="hljs-function"><span class="hljs-keyword">void</span> <span class="hljs-title">setup</span><span class="hljs-params">()</span>
</span>{ 
pinMode(led, OUTPUT); 
}
<span class="hljs-function"><span class="hljs-keyword">void</span> <span class="hljs-title">loop</span><span class="hljs-params">()</span> 
</span>{ 
digitalWrite(led, HIGH); 
LowPower.powerDown(SLEEP_1S, ADC_OFF, BOD_OFF); <span class="hljs-comment">// instead of delay(1000) ; </span>
digitalWrite(led, LOW); 
LowPower.powerDown(SLEEP_1S, ADC_OFF, BOD_OFF); <span class="hljs-comment">// instead of delay(1000) ; </span>
}</pre><br><strong>Before using the Lowpower library current taken by arduino</strong><p>51.7mA when led is ON<br></p><p>47mA when led is OFF</p><p><strong>After using the Lowpower library current taken by arduino</strong></p><p>34.93mA when led is ON<br>31.73mA when led is OFF</p><p>Are you happy to reduce 32.43 % power ?? Hey there is still room to reduce the power consumption.<br>Your arduino board have different power sucking components like power led,voltage regulator and USB interface chip which takes most of the power even when it is idle.</p></div>

                            <small class="authorship">2 months ago  &bull;

                                </a>

                            <div style="position:relative">
                                <div id="post-body-20101"><p><a id="photoset-link-FJC9RW6I7BQFB1U-0" class="photoset-link" href="http://cdn.instructables.com/FZP/VPY0/HZV3OM8F/FZPVPY0HZV3OM8F.LARGE.jpg"><img alt="Picture of Battery Life Estimation " src="http://cdn.instructables.com/FZP/VPY0/HZV3OM8F/FZPVPY0HZV3OM8F.MEDIUM.jpg"></a></p><p><a id="photoset-link-F56WONRI7BQFB1V-1" class="photoset-link" href="http://cdn.instructables.com/FKI/MTT0/HZV3OMBJ/FKIMTT0HZV3OMBJ.LARGE.jpg"><img alt="temp_977081451.jpg" src="http://cdn.instructables.com/FKI/MTT0/HZV3OMBJ/FKIMTT0HZV3OMBJ.MEDIUM.jpg"></a></p><a id="photoset-link-FEOVDBMI7BQFB1W-2" class="photoset-link selected" href="http://cdn.instructables.com/FAV/9FNO/HZV3ONGG/FAV9FNOHZV3ONGG.LARGE.jpg"><p><img alt="temp_-509662963.jpg" src="http://cdn.instructables.com/FAV/9FNO/HZV3ONGG/FAV9FNOHZV3ONGG.MEDIUM.jpg"></p></a><p>The battery life can be calculated by determining the average current for the circuit.</p><p>Use the following general equation to calculate average current.</p><p><strong><br></strong></p><p><strong>Iavg = (Ton*Ion + Tsleep*Isleep ) / (Ton +Tsleep)</strong><strong></strong></p><p>Ton (arduino is active ) = 250 ms =0.25s and Ion = 16mA</p><p>Tsleep = 5min =300s and Isleep = 200 uA (approx)</p><p>It is very difficult measure the current during the sleep period .Some time I got zero reading.</p><p>Iavg = 0.205 mA</p><p>Operating Voltage =5 V</p><p>Pavg=VxIavg =5x.205=1.026 mW</p><p>Li Ion battery capacity =3000 mAh</p><p>Battery voltage =3.7V</p><p>Power =3.7x3000=11100 mWh</p><p>Battery life = 11100/1.026 =10,818.7 hours = <strong>15 months approximately</strong></p><p>From the above calculation it is clear that theoretically by using a fully charged 3000 mAh Li Ion battery we can run the arduino for 15 months.In practical due to battery self discharge this figure may be different.</p><p>As the system is equipped with solar charging system ,we can run for few years without any interruption.</p></div>
