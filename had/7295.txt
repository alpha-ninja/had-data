                    <h5>Description</h5>
                    <div class="description">This project will attempt to build a prototype hardware/software solution using consumer 3D-scanning and printing technologies to produce individually designed and fitted nasal CPAP cannula for suitable for preterm infants in Neonatal Intensive Care.  We intend to use 'Processing 2' open source programming language (2003-2014 Ben Fry and Casey Reas), some Arduino bits and bobs and NinjaFlex flexible filament.<br /><br />The initial design and proof of concept will be undertaken using a resuscitation manikin as a proxy for a human infant.  We intend to provide a solution that provides an improved seal to the nose to improve ventilation and reduce the potential for nasal septal injury.  We hope that this prototype may pave the way for a product that has the potential to be better tolerated by the baby, should minimise handling by nursing staff and provide more consistent therapy. <br /><br />Oh yeah - and we have zero experience in coding and electronics so this should be easy right?<br /><br /></div>
                </div>

                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>Why do we need this?</p><p>Most preterm kids require ventilation support during a NICU stay - these kids may weigh as little as 500g at birth and require months of ventilation support.  Ill fitting nasal cannula can reduce the effectiveness of the therapy and in some cases can cause significant injury and disfigurement to the nasal septum of these kids.</p><p>Surely we can do better?</p><p>In true hackaday spirit this will be attempted by myself (Ben) and Dave - who work in NICU but have no programming or electronics experience.  What could possibly go wrong?</p><p>On that note - advice and input are welcome.<br></p>
                                
                            </p>

                            <p><p>The basic template with customisable dimensions works from a software perspective and prints (just).  You may notice when the object rotates on the screen I need to write the code to cut out the internal surface of the main cylinder where the two nares tubes intersect it.  Found the math - just not written the code yet.</p><p>I am currently optimising the printing.  So trying different profiles and wall thicknesses for the cylinder tube.  I am finding an eliptical (in profile) cylinder prints well as does a cylinder with a circular outer surface and eliptical inner surface. Bridging the top of the cylinder is obviously the difficult part of the print.</p><p>Next step will be to import a 3D facial scan into a Processing sketch as a PShape object and work out how to extract the data I need to match the basic template to the profile of the infants face.  Got some ideas on how to write a loop to run through the points that define each face in the PShape and extract a plane (through y axis).  Given this is the first programme I have ever written it is a little daunting.</p><p>Currently the Processing sketch draws the cylinder by drawing the circles each end of the cylinder (inner and outer) and then essentially joining them up with a series of triangular surfaces.  I figure if I can get a profile of the face, directly under the nose, and then slice that line into a number of segments I can then draw a whole bunch of connecting cylinders that follow the profile of the face.</p><p>That&apos;s the plan anyways.  At the very least I am pretty happy with my progress for a complete novice.</p><p>I plan that this will be the first prototype - design elements may change once the proof of concept is complete.  A few interesting ideas have cropped up while printing these test versions.  We do think it might be interesting to look at the internal structure of the main cylinder.  Creating turbulent air flows within this form may improve ventilation.  This could well be an interesting application of the rapid prototyping concept.  Particularly as the glaring obvious issue this product will have is that there is currently no FDA (or TAG in Australia etc) approved flexible 3D printing filament.  Partly the point of this Hackaday Prize entry is to demonstrate the potential uses of such a filament.<br></p>
                                
                            </p>

The filament path needs to be constrained so it doesn&apos;t buckle when being driven into the extruder.</p><p>I found that this <a href="http://www.thingiverse.com/thing:53125">extruder upgrade</a> is necessary for my Makerbot Replicator 2 - but easily printable in PLA.  The only changes I made to the standard Makerbot print settings were to slow the printing speed to 30mm/sec (230C print temp).</p><p>I also have a Printrbot Simple Metal - Ninjaflex not printing as well on this yet but will post details if I have any success.
