                            <h2><a href="/project/4510/log/14796-lcd-pinout">LCD pinout</a></h2>
                            <small class="authorship">5 months ago  &bull;
                                <a class="gray-link" href="/project/4510/log/14796#discussion-list">
                                    1 comment
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-14796"><p>Today I had some time to check LCD pinout and write a simple app that test weloop tommy LCD.<br><br>First thing first, you should read documentation for Sharp memory lcd <a href="http://www.pebble-smartwatch.de/wp-content/uploads/2012/04/LS013B7DH01_Application_Info.pdf" target="_blank">LS013B7DH01</a>. Next step is to read <a href="http://www.sharpmemorylcd.com/resources/programming_memory_lcd_app_note.pdf" target="_blank">Programming memory LCD application note</a> so you will know how to communicate with the screen.</p><p>Communication is really easy, it consists of:</p><ul><li>write only SPI line: SCLK, SI</li><li>chip selection pin: SCS</li><li>screen enable pin: DISP</li></ul><p>The only problem with this is to know watch pinout. It's as follows:</p><table><tbody><tr><td><strong>LCD Pin</strong></td><td><strong>SPI name</strong></td><td><strong>NRF WLCSP ball<br></strong></td><td><strong>NRF Name<span class="redactor-invisible-space"></span></strong><strong></strong></td></tr><tr><td>SCLK</td><td>SCLK</td><td>E8</td><td>P0.31</td></tr><tr><td>SI</td><td>MOSI</td><td>B7</td><td>P0.28</td></tr><tr><td>SCS</td><td>SS</td><td>D8</td><td>P0.30</td></tr><tr><td>EXTCOMIN</td><td>-</td><td>F8</td><td>P0.04</td></tr><tr><td>DISP</td><td>-</td><td>G8</td><td>P0.06</td></tr></tbody></table><p>EXTCOMIN pin can be skipped because EXTMODE pin is grounded on board.</p><p>SPI MISO line is not used so can be set as NC in code.</p><p>This LCD requires 5V power supply, <strong>by default 5V voltage regulator is disconnected so LCD does not work</strong>, use <strong>P0.13</strong> to enable voltage regulator!</p><p>The most important SPI commands are</p><ul><li>override single line</li><li>override multiple lines</li><li>clear whole screen</li></ul><p>You cannot change just one pixel, the smallest accepted data portion is one line. Full description of a command structure and list of all commands can be found in  <a href="http://www.sharpmemorylcd.com/resources/programming_memory_lcd_app_note.pdf" target="_blank">Programming memory LCD application note</a> </p><p>I've created a <a href="http://developer.mbed.org/users/sl0w0rm/code/mbed_weloop_screen_demo/" target="_blank">simple app</a> that shows how to communicate with screen. It can be imported and compiled on mbed site.</p><p><img src="https://cdn.hackaday.io/images/5113131425733231990.jpg"></p><p>In a few days I will publish pinout for:</p><ul><li>watch buttons</li><li>backlight</li><li>vibration motor</li></ul><p>To write an accelerometer demo I still need a new watch. Thank you all for the donations, $30 more and I will have it. If you want to help please donate <a href="https://www.paypal.com/cgi-bin/webscr?cmd=_donations&amp;business=V6KL4EH8ZCQX2&amp;lc=US&amp;item_name=Open%20source%20sport%20watch&amp;currency_code=USD&amp;bn=PP%2dDonationsBF%3abtn_donate_SM%2egif%3aNonHosted" target="_blank">HERE</a>. Thanks!</p></div>

                            <h2><a href="/project/4510/log/15224-external-memory-pinout">External memory pinout</a></h2>
                            <small class="authorship">5 months ago  &bull;
                                <a class="gray-link" href="/project/4510/log/15224#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-15224"><p>Weloop tommy has an external FLASH and RAM. According to <a href="http://welooptommy.com/weloop-tommy-smartwatch-technical-specifications/" target="_blank">product specification</a> it's 16kB RAM and 512kB FLASH. </p><h2>RAM pinout</h2><p>It's easier to describe pinout than indicate a specific chip that's used. RAM looks like a microchip 23K640. Pinout is correct but available size is lower than expected. But that may be an error in specification. It looks like internal RAM size and external RAM size were swapped. Documentation for microchip 23K640 can be found <a href="http://ww1.microchip.com/downloads/en/DeviceDoc/22126D.pdf">HERE</a>.</p><p><img src="https://cdn.hackaday.io/images/4669101426447920455.jpg"></p><p>Chip pinout:</p><table><tbody><tr><td><strong>RAM pin</strong></td><td><strong>NRF WLCSP pin</strong></td><td><strong>NRF pin name</strong></td></tr><tr><td>SCK</td><td>J6</td><td>P0.11</td></tr><tr><td>SI</td><td>J5</td><td>P0.12</td></tr><tr><td>SO</td><td>J7</td><td>P0.09</td></tr><tr><td>CS</td><td>B6</td><td>P0.23</td></tr></tbody></table><h2><br></h2><h2>FLASH pinout</h2><p>It's even harder to indicate flash chip model. If you know what chip it is please add a comment or write a message.</p><p><strong>Update:</strong><br>It's W25Q40CL. Documentation can be found <a href="https://www.winbond.com/resource-files/da00-w25q40cle1.pdf" target="_blank">HERE</a>.</p><p><img src="https://cdn.hackaday.io/images/3970551426447996936.jpg"></p><p>Chip pinout:</p><table><tbody><tr><td><strong>FLASH pin</strong></td><td><strong>NRF WLCSP pin</strong></td><td><strong>NRF pin name</strong></td></tr><tr><td>SCK</td><td>J6</td><td>P0.11</td></tr><tr><td>SI</td><td>J5</td><td>P0.12</td></tr><tr><td>SO</td><td>J7</td><td>P0.09</td></tr><tr><td>CS</td><td>J8</td><td>P0.08</td></tr><tr><td>WP</td><td>H6</td><td>P0.10</td></tr></tbody></table><br></div>

                            <h2><a href="/project/4510/log/15505-sky-is-the-limit">Sky is the limit!</a></h2>
                            <small class="authorship">5 months ago  &bull;
                                <a class="gray-link" href="/project/4510/log/15505#discussion-list">
                                    2 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-15505"><p>Today I managed to write an app for mac OSX that's able to upload a custom firmware using firmware update feature of the watch. It doesn't require to enter an OTA mode so there's a catch:</p><p><strong>You can upload a custom firmware to any watch that's in the bluetooth range and is not connected to any phone! No physical access needed!</strong></p><p>Because of that issue I decided not to publish source code. But it doesn't stop you from uploading your custom firmware in OTA mode. To enter OTA mode just connect a charging cable, press all 3 buttons on the right side of the watch and release top and bottom keeping middle button pressed few seconds more. How to upload a new firmware in that mode? Just use a <a href="https://play.google.com/store/apps/details?id=no.nordicsemi.android.mcp&amp;hl=en" target="_blank">nRF Master Control Panel</a> on an android phone. No need to calculate and provide firmware checksum (.dat file), watch bootloader is old and doesn't require to do that.</p><h2>Firmware upgrade constrains</h2><p>There are two ways to modify the watch:</p><ul><li>Update just the firmware</li><li>Update firmware, soft device and the bootloader</li></ul><p><strong>The first option</strong> is quite easy. There's no way you can break anything (unless you play with internal flash from the code ;) ) because there's always a possibility to switch watch to an OTA mode and override existing/broken firmware. But it has also the biggest limitations:</p><ul><li>we have to use existing softdevice which is a S110 6.0.0. This is very old and obsolete softdevice and it's not possible to write code in mbed (unless you use an old/buggy version)</li><li>we're limited to being a peripheral device so no way to connect to an external sensor without a phone</li></ul><p>To write a firmware that way you need to:</p><ul><li>Set firmware base address to 0x14000 (that's the address of the first instruction in the firmware)</li><li>Your firmware cannot be bigger than 147456 bytes (0x24000 bytes) because than you will override a bootloader (it starts at 0x38000)</li><li>Use old NRF SDK 5.2.0 with S110 6.0.0 soft device</li></ul><p><strong>The second option</strong> is to override not only firmware but also the bootloader and the softdevice. That way we can use other softdevices like S120 (for central device) or S130 (both central and peripheral) and write anything we want!</p><p>I will try to upload S130 softdevice and a sample app in the following days. Stay tuned!</p></div>

                            <h2><a href="/project/4510/log/15688-almost-there">Almost there</a></h2>
                            <small class="authorship">5 months ago  &bull;
                                <a class="gray-link" href="/project/4510/log/15688#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-15688"><p><strong>The bed news </strong>is that it's not possible to update softdevice/bootloader with standard weloop bootloader. When you try to update the firmware/application it works but when you try to update bootloader/softdevice it just restarts. </p><p><strong>The good news </strong>is that we can write a custom application that will do this. Now I'm working on an app that will fetch new bootloader/softdevice using nordic UART over BLE and save it in external flash. When everything is uploaded then it will copy data from external flash to internal flash and we will have a new softdevice/bootloader. And I have to write it using nrf 5.2.0 SDK which is really annoying ;)</p><p>There was only one problem... I didn't know what external flash is in this watch. After some research I found out that there is a standard spi command that works with almost every flash chip. It's 90h and returns chip version but that version may be shared between many variants of the same chip. Then I found a newer command: 9Fh and now I'm 100% sure what chip is there.<br></p><p><img src="https://cdn.hackaday.io/images/5272331427403991143.jpg"></p><p>Manufacturer Id: 0xEF</p><p>Device Id: 0x4013</p><p>So it's definitely <a href="https://www.winbond.com/resource-files/da00-w25q40cle1.pdf" target="_blank">THIS</a> chip :)</p></div>

                            <h2><a href="/project/4510/log/16334-changing-softdevice-and-bootloader-relocation">Changing softdevice and bootloader relocation</a></h2>
                            <small class="authorship">4 months ago  &bull;

                                <div id="post-body-16334">Today I finally managed to upload a new bootloader/softdevice to a watch! It wasn't easy because bootloader that is originally installed on weloop does not allow to upload a new softdevice or bootloader. There's also no support in softdevice to run command on device start (using MBR) that will copy part of firmware from one place to another (it was added in 7.1.0 softdevice and is required to change bootloader or softdevice). So the biggest issues were:<br><ul><li>no bootloader/sofdevice upload feature in original bootloader</li><li>no way to use a newer bootloader code because it requires features that are not available in S110 6.0.0 softdevice.</li><li><strong>UICR cannot be updated! </strong>That's the biggest issue because there is a value 0x14000 set at 0x10001000 address so there's no way to use newer softdevice with this value there. There's also 0x38000 set at 0x10001014 and I wanted to make bootloader smaller and use 0x3C000 as a start address.</li></ul><p>So how I manage to do this? I will describe it tomorrow because it's time to go sleep ;) If you want to check it yourself take a look at <a href="https://github.com/ossw/ossw-installer">THIS CODE</a>. A compiled firmware that replaces bootloader/softdevice can be found <a href="https://github.com/ossw/ossw-installer/releases/tag/0.1" target="_blank">HERE</a>. But be careful with it, it can brick your watch!<br><br>So from now on my weloop can connect directly to other devices without a mobile phone! Now's time to write an awesome firmware ;)<br><br></p></div>
