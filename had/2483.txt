                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">This project was build based on idea on how to make a simple and cheap online surveillance camera robot that everyone can build. Not only that the robot also have to be easy to use and maintain. Using the raspberry pi, few cheap components, a robot chassis/RC car, and  webiopi tutorial on the MagPi magazine this project was successfully build. For launching this robot to be accessible online, ngrok service from www.ngrok.com is used. Without any hectic and difficult port forwarding configuration this robot now can be controlled from anyone in the world by just using firefox or chrome browser.<br />Here is a demo video: <a href='http://www.youtube.com/watch?v=9osFoNgC9cw' target="_blank">http://www.youtube.com/watch?v=9osFoNgC9cw</a></div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>Hardware:</p><p>The Blok Diagram for the whole system:</p><p><img src="https://cdn.hackaday.io/images/2901691408263467108.jpg"></p><p>No soldering is required to build this robot because I use jumper wire to connect all necessary connection between the GPIO to the L298n motor driver module, and then i connect the two dc motor to the mounting hole on the L298n module.</p><p><img src="https://cdn.hackaday.io/images/2046041408261066848.jpg"></p><p> you can arrange all the parts to the main body all the way you like and then make it attach really strong using a few nuts and spacer.</p><p><img src="https://cdn.hackaday.io/images/1424601408261468459.jpg"></p><p>the main thing you have to concern is arranging the cables neatly so the robot can move easily.</p><p>Software:</p><p>This robot run on raspbian using the webiopi framework.&nbsp;</p><p>If you don't know how to install raspbian you can check:</p><p><a href="http://www.raspberrypi.org/documentation/installation/installing-images/README.md">http://www.raspberrypi.org/documentation/installation/installing-images/README.md</a></p><p>you can install webiopi using this tutorial:</p><p><a href="https://learn.adafruit.com/raspberry-pi-garage-door-opener/web-io-pi">https://learn.adafruit.com/raspberry-pi-garage-door-opener/web-io-pi</a></p><p>for the camera stream I use mjpg-streamer which give a very good streaming live video for the robot. this is how you can easily install it using this tutorial:</p><p><a href="http://www.linuxcircle.com/2013/02/06/faster-video-streaming-on-raspberry-media-server-with-mjpg-streamer/">http://www.linuxcircle.com/2013/02/06/faster-video-streaming-on-raspberry-media-server-with-mjpg-streamer/</a></p><p>after installing the software needed you can grab my code from github:</p><p><a href="https://github.com/chalei/surocam">https://github.com/chalei/surocam</a></p><p>make sure the raspberry pi is connected to your network.</p><p>launch the python program using on your raspberry pi using:</p><p>$ sudo python surocam.py</p><p>launch your browser then open:</p><p>&lt;raspberrypiipaddress&gt;:8000</p><p>and voila now you can control the robot by only using the web browser.</p><p>The configuration above is for using the robot offline which means you can only control it in a local network.</p><p>To make it online its a bit tricky because it need a static IP and whole lot other configuration needed. Most of it cannot be succeeded and its hard to find what went wrong for me</p><p>For using the robot online the simple way I use ngrok service. the information on how to use and download it, you can find it here:</p><p><a href="https://ngrok.com/">https://ngrok.com/</a></p><p>make sure you sign up and get a token key, because you can choose the subdomain the way you want.</p><p>after you download the ngrok first you have to run the python program in the background using this command:</p><p>$ sudo python surocam.py &amp;</p><p>then launch the ngrok software with the network port that was used and the subdomain you desired, for example:</p><p>$ ngrok -subdomain=surocam 8000</p><p>Open the web browser again but now open the DNS that you've choose before:</p><p><a href="https://surocam.ngrok.com">https://surocam.ngrok.com</a></p><p>Now you can easily access the robot from anywhere in the world just by using a web browser</p>

                            <p><p>In order to drive the motor I don't need to use another microntrollers board because the Raspberry Pi capable to run Motor driver motor directly.</p><p> For the motor driver I choose the L298n motor driver module because it can give 2 A to the motors and has 4 mounting hole that can be attached with nut or spacer and also it is quite cheap ($3 at amazon)</p><p>The specs told me that it need 5v logical input and raspberry pi have 3,3v output from the GPIO. But on the first try it can take the 3,3 V input from the raspberry pi. and it also run well. the robot move smoothly no matter what kind of chassis it used.</p><p>it also have onboard heatsink that can make L298 IC little bit "cooler"</p>

                            <h2><a href="/project/2483/log/7529-going-online-the-easy-way">Going Online The easy way</a></h2>
                            <small class="authorship">a year ago  &bull;
                            <a class="gray-link" href="/project/2483/log/7529#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>After successfully make this robot run on a local network, the next step is to connect this robot to the world (via internet). After a few google hunt, they say I need to do a port forward and ip public is needed. If i dont have ip public I can use dynamic DNS service, but most of them does not come for free. Because It need a little bit hectic configuration and I also don't have any international money account (difficult country regulation), I need find the another option.</p><p>Then I found pagekite from&nbsp;<a href="http://pagekite.net/">http://pagekite.net/,</a> it is a service that based on python. I just download a few files and sign up free. After that just launch the pagekite app and the robot now can be access online anywhere. But there is still problem with this, because it only have one month trial and to continue the service I have to pay, the service is really cheap actually but I dont have any account to make the payment.</p><p>After another google hunt I found about tunneling, and there are a lot of tunneling service. Then I choose ngrok from&nbsp;<a href="https://ngrok.com/">https://ngrok.com/</a> because of its simplycity. After signing up and get a token key, I insert the token to the raspberry pi and then launch the ngrok application. because I have already sign up I can choose any subdomain I want as long as it is not been used by other user. And the very great thing about this service is, it is free forever and does not require any difficult configuration.</p>
