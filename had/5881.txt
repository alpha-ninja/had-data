                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">This is an arduino based oscilloscope that should be capable of ~150k samples per second capture. It consists of three parts, the arduino firmware, a shield and PC software written in c++ to display the signal.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>small-scope is composed of three components:</p><ul><li><a href="https://github.com/marvin-sinister/small-scope-electronics">arduino shield</a></li><li><a target="_blank" href="https://github.com/marvin-sinister/small-scope">firmware</a></li><li><a href="https://github.com/marvin-sinister/small-scope-qt">software</a></li></ul><p>The shield shapes the signal. The arduino itself captures the data and sends it to PC, which then displays the data in software.</p><p>To test the firmware and software, the shield can be replaced with a few wires and a pot (or even nothing at all).<br></p>
                                
                            </p>
                        </div>

                            <h2><a href="/project/5881/log/20878-new-pcb-design-finished">New PCB design finished</a></h2>
                            <small class="authorship">a month ago  &bull;
                            <a class="gray-link" href="/project/5881/log/20878#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>Today the new PCB design is finished. I'm quite satisfied. It's probably not my finest work and the PCB area is larger than necessary, but it's single sided with only two short leads on top layer of the board.</p><p>I haven't made and tested the board yet, and that will probably have to wait at least until next week.</p><p>For the moment, this is it, and if the board works as intended, I'll probably call this project finished and start something new.<br></p>
                                
                            </p>
                        </li>
                    
                        <li>
                            <h2><a href="/project/5881/log/20819-current-status-2015-07-11">Current status 2015-07-11</a></h2>
                            <small class="authorship">a month ago  &bull;
                            <a class="gray-link" href="/project/5881/log/20819#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>So today the schematics got an update. This should be more or less final form for the moment, no more feature creep.</p><p>Next thing on the schedule is designing PCB which will take a bit of time since I want to stick with single layer design friendly to home PCB manufacture.</p><p>The features now include attenuator, AC coupling with offset, DC offset, some safety, two LEDs for status...</p><p>As before, this is still in development and hasn't been tested, so I don't know whether it <strong>actually</strong> works, but to best of my knowledge, it should.<br></p>
