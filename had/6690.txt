                    <div class="description">A few weeks ago we started a Star Wars rol with some collegemates, we though that it would be nice surprise for our group if we were to bring 'live' to an NPC who comes always with us. <br />This is a surprise so don't tell anyone from our group!                             Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p>The control is made over UDP packets. They're not very reliable, but as we well always be in sight of it, it shouldn't be a problem. Aside from moving, it has features like the song of Cantina and of course the main theme from Star Wars. Also a fully functional radio. A servo controls its head and it features an amazing totally normal blue LED. No holo projector, sorry. ;( Had to keep it in budget. 
                                
                            </p>

                        <li>
                            <h2><a href="/project/6690/log/21026-controller">Controller</a></h2>
                            <small class="authorship">a month ago  &bull;
                            <a class="gray-link" href="/project/6690/log/21026#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>We picked the ESP8266 thinking on it&apos;s WiFi and all the possibilities that it could bring to our project. </p><p>We have chosen Android as the controller, though it would be easily portable to iOS, Windows, Mac, GNU/Linux, etc, as it&apos;s as simple as UDP packets. </p><p>It isn&apos;t very reliable, but as we have visual on R2D0 we can press the button again. In some cases, as the LEDs on the head, it even replies back and the button switches to red or green, depending of the state. </p><p>There is a slider which goes from 1 to 100, it controls the speed. It has problems starting the motor under a certain value which depends on the voltage, but we aren&apos;t thinking on fixing that yet, as this way we can start it on 50% and then switch to 10%, that way it can walk slower. </p><p>In the gallery we have an screenshot of the controller. </p><p>We can easily add more buttons, delete or hide them, as we try to make it an universal remote for our robots with ESP8266. So we can copy &amp; paste all the controls and the code and only modify a few options. For example, in a another robot we wouldn&apos;t need the radio options or the servo, but we could activate the line follower mode if that robot would have it.</p><p>Link to the app in the Play Store: <span class="component-description"><a href="https://play.google.com/store/apps/details?id=udpcontroller.nomal" target="_blank">https://play.google.com/store/apps/details?id=udpcontroller.nomal</a></span><br></p>
                                
                            </p>
                        </li>
