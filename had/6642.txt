                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">With a display inspired by Daniel Rojas' "Micro Word Clock" (previously seen on Hackaday), this project is a digital compass that uses minimal hardware.<br /><br />Nifty features include:<br /><br />o- calibration measures and compensates for hard- and soft-iron interference and sensor orientation<br /><br />o- calibration data stored in EEPROM with wear-leveling<br /><br />o- easy recalibration; displayed prompts guide the user<br /><br />o- automatic temperature compensation<br /><br />o- detects and warns about (some) interference, and about excessive sensor tilt (relative to the plane of rotation established during calibration)<br /><br />o- under 5KB of program space; 100% C code (no C++ or assembly); 100% Open Source<br /><br />o- lightweight 8x8 LED matrix library I wrote for this project<br /><br />o- original HMC5883L driver, incl. self test<br /><br />o- original fixed-point math library</div>
                </div>

There's a series of <a href="http://mythopoeic.org/tech/avr-compass/">eight articles</a> I wrote about all the steps (and mis-steps) and building blocks that went into it.</p><p>One article in particular I want to call attention to is <a href="http://mythopoeic.org/magnetometer/">the one that explains the math</a> behind the transformations I use to turn the 3-axis magnetometer reading into a compass heading. <br></p>
