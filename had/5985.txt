                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">I always wanted to do nice video of me flying my RC plane, but I can't fly and film at the same time. I decided to develop an autonomous way of filming my planes using arduinos and my father HD sony camera.<br /><br />It started with image treatment with OPEN CV and a webcam to detect motion, but after some weeks of development it was obvious that I was good enought in image treatment to keep track of a planes with different shape / size / color in a reliable way.<br /><br />I briefly switched to RSSI tracking as I allready have in most of my plane 433mhz telemetry, I am keeping this one as backup.<br /><br />Right now it is using a beacon with GPS, needed in the moving object.<br /><br />This project will be done in 3 phase :<br />- Found and validate the way to track the object using a V1 hardware<br />- Develop a better hardware (V2 with CAD + 3D printed body)<br />- Software tunning<br /><br />It is entering the final phase right now.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>The principle is simple, compare GPS coordinate between 2 elements to determine where to aim a pan and tilt to film it.</p><p>However, there is some limitation, cheap GPS are not very precise so it will not work with a distance below 15m. Also altitude measurement with GPS is not accurate enough to use it. I decided to go with pressure sensors (MS5611) and use a simple pilote rule that 100Pa = 2 feets.</p><p>The first version was using a direct orientation to North pole for absolute positioning. I quickly understood that is was a strong contrain. The new version wait for the start of the tracking to take the current position as 0° forward (meaning you have to point the beacon while starting tracking). It is then using the course_to function from the tinygps++ lib to compute an angle. I use the tinygps++ beaucse it can output the coordinate with more precision.</p><p>There is a button on the beacon to command the base for start and stop tracking. At the same time the IR while send to the sony camera to order to start recording.</p><p>On the base you have 3 button :</p><p>-1 used for altitude compensation, it will zero the offset between the pressure of the beacon and the one of the base. </p><p>-1 used for command to start or stop the tracking</p><p>-1 is the reset of the Arduino Mega</p><p>There is also 2 switch</p><p>-1 for 2 preset of rotation speed (usefull for development and later for preset according to the scene you are filming)</p><p>-1 for 2 preset for zoom behavior (max zoom value)</p><p>I am using 2 Arduino on the base to avoir having jitter on the servo as interrupt present problem for the PWM generation. The 2 Arduino are link by a serial comm using my own basic protocol to transmit only position and speed.</p><p>The beacon is using a moteino as it is usefull for me to have the µ and the RFM69HW directly on the same PCB and at 3.3V (no level shifters). The GPS is working either in 3.3v or 5v.</p><p>Software details :</p><p>- Tinygps++ libraries for NEMA translation and course computation</p><p>- Servo libraries with speed included</p><p>- IRremote libraries</p><p>- RFM69HW library (currently set to minimal power)</p><p>- GPS set to 5Hz</p><p>- Loop time target 100ms</p><p>- Simple low pass filtering on GPS coordonate</p><p>To do :</p><p>- Tune speed parameters to smooth the rotation</p><p>- Tune zoom control in flight</p><p>Nice to have :</p><p>- Kallman filter</p><p>Finaly you may have noticed that you already saw something like my device, you are right, it is called Soloshot and available on amazon, build by surfers but working with almost every hobbies !</p><p>But the fun is to do it by ourselves isn't it ?</p>
                                
                            </p>
