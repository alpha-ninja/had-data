                            <small class="authorship">a month ago  &bull;
                                <a class="gray-link" href="/project/6568/log/21093#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-21093"><p>Well in  the comments there was the idea that it would be useful to daisy chain this display.  Theoretical the only thing which is needed is the shift output of the the first register an connect it to the input of the next. But then you would need al lot of wire additional to connect the supply voltage, the clock and the latch pin. So I came up with this (rendering from OSH-Park):</p><p><img src="https://cdn.hackaday.io/images/2029501437227616748.png"></p><p>As you can the there are now additional PAds on both sides. These are for soldering two boards together. Only on the first board the ISP header is to be mounted. So multi segment display can be created. </p><p>Thanks to <a data-mention-user-id="39031" href="/hacker/39031">@Stefan Lochbrunner</a> and <a data-mention-user-id="64089" href="/hacker/64089">@Dmm6137</a> for their comments. <br></p></div>
                            </div>

I Hope this will work:</p><p><img style="width: 536px; height: 731px;" src="https://cdn.hackaday.io/images/9856311437313631115.png" height="731" width="536"></p><p>Normally the shipping and handling is slow.                             <h2><a href="/project/6568/log/21328-arduino-library-updated">Arduino Library updated</a></h2>
                            <small class="authorship">a month ago  &bull;
                                <a class="gray-link" href="/project/6568/log/21328#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-21328"><p>So the library (see bitbucket) does now support all three kinds of mounting the display. Just use different argument at the .begin function like: </p><pre class="hljs cpp"> myTiny7.begin(<span class="hljs-number">3</span>);<span class="hljs-comment">
//if your display is mounted normal use no argument or 0 else:</span>
<span class="hljs-comment">// 0: normal 		(default)</span>
<span class="hljs-comment">// 1: mirror 		mounted on the other side of the PCB over the IC</span>
<span class="hljs-comment">// 2: rotated		mounted on normal side (over the QR-Code) but upside down</span>
<span class="hljs-comment">// 3: mirror + rotated 	mounted on other side upside down </span></pre>Also Attiny85/456 based boards like the Trinket should be supported now. Then the default pin out is: <br>1: SDO serial data out<br>2: Clock<br>3: latch<br>Change this as additional parameter in the .begin function, if needed, like this:<pre class="hljs cpp">myTiny7.<span class="hljs-keyword">begin</span>(<span class="hljs-number">3</span>, <span class="hljs-constant">SDO_PIN</span>, <span class="hljs-constant">CLK_PIN</span>, <span class="hljs-constant">LATCH_PIN</span>);</pre></div>
