                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">JACK is a development kit powered by an Atmel SAM D21; it's pretty barebone, that is it has just the bare minimum to make it work while keeping an eye on reliability. </div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><h2>Features:</h2><ul><li>a<strong> USB device por</strong>t with cable shield filtering and transient suppression </li><li>a <strong>3.3V LDO</strong> with bypass caps and filters to separate analog VDD from digital VDD </li><li><strong>Schottky diodes</strong> to protect the possible power inputs </li><li>a <strong>12.288 MHz crystal</strong> for the master clock (pretty useful in audio applications) </li><li>a <strong>reset pushbutton</strong> with debounce cap and pullup resistor </li><li>a <strong>blue LED</strong> on <strong>pin 32</strong> for the classic Blinky test </li><li><strong>two breakout headers</strong> (all pins broken out except 2 pins for crystal and 2 for USB) </li><li>the <strong>ATSAMD21G18</strong> in TQFP package (max specs for the D21, easier to solder than a QFN)</li><li><span></span><strong>Plus: </strong><strong>Full compatibility with Arduino Zero firmware (without debugging options because there&apos;s no programmer/debugger chip on the board)</strong></li></ul><p><strong><br></strong></p><hr><h2>Licences</h2><p><strong>JACK hardware</strong> is <strong>OSHW, </strong>licensed under <strong>CERN OHL 1.2</strong><span class="redactor-invisible-space"></span></p><p><strong>JACK software</strong> is <strong>LGPL 2.1 </strong>(because it&apos;s <strong>based on the Arduino Core</strong>)</p><hr><h1>Installation and usage guides</h1><p style="margin-left: 20px;"><strong></strong></p><h2>Arduino IDE setup for JACK</h2><ol><li>Donwload the <strong>Arduino</strong> <strong>IDE</strong> <strong>1.6.5</strong></li><li>Go to <strong>Tools</strong>-&gt;<strong>Board</strong>-&gt;<strong>Boards</strong> <strong>Manager...</strong></li><li>Install <strong>Arduino SAMD Boards (32-bit ARM Cortex-M0+) 1.6.1</strong></li><li><span class="redactor-invisible-space"><span class="redactor-invisible-space">Download the Arduino Core modified for JACK compatibility: <a href="https://github.com/MickMad/ArduinoCore-samd/"></a></span></span><a href="https://github.com/MickMad/ArduinoCore-samd/archive/master.zip">https://github.com/MickMad/ArduinoCore-samd/archive/master.zip</a></li><li>Move the following files/folders to <strong>C:\Users\mperla.CORP\AppData\Roaming\Arduino15\packages\arduino\hardware\samd\1.6.1 </strong><span class="redactor-invisible-space">(or whatever your Arduino SAMD package path is)</span>:<ol><li><strong>bootloaders\jack </strong></li><li><strong>cores\jack </strong></li><li><strong>variants\jack</strong></li><li><strong>boards.txt</strong></li></ol></li></ol><p>A JACK board flashed with the modded Arduino core will show as an &quot;Arduino Zero bootloader (COMn)&quot; port on your PC; if you can&apos;t find the board, but it shows as &quot;Unknown hardware&quot; you shall update its drivers with the Arduino Zero ones (you can find them in ..\your_arduino_path\arduino-1.6.5-r2\drivers)</p><hr><h2>Using JACK with the Arduino IDE</h2><ul><li><strong>Connect JACK to an USB</strong> port on your PC, if it has never been programmed you should see an <strong>Arduino Zero bootloader</strong> serial port device</li><li>Open your Arduino IDE and <strong>write a sketch</strong> (e.g.: Blink LED on pin 32/PIN_WIRE_SCL)</li><li>Select <strong>Arduino Zero (JACK) </strong><span class="redactor-invisible-space">from <strong>Tools</strong><span class="redactor-invisible-space">-&gt;<strong>Board</strong></span></span></li><li><span class="redactor-invisible-space"><span class="redactor-invisible-space"><span class="redactor-invisible-space">Select the <strong>Arduino Zero (Native USB Port) </strong><span class="redactor-invisible-space">under <strong>Tools</strong><span class="redactor-invisible-space">-&gt;<strong>Port </strong><span class="redactor-invisible-space">(or just select the correct port associated to Arduino Zero bootloader in your device manager)</span></span></span></span></span></span></li><li><span class="redactor-invisible-space"><span class="redactor-invisible-space"><span class="redactor-invisible-space"><span class="redactor-invisible-space"><span class="redactor-invisible-space"><span class="redactor-invisible-space"><span class="redactor-invisible-space">Click <strong>Upload</strong><span class="redactor-invisible-space">, when finished the device will reset and start the sketch</span></span></span></span></span></span></span></span></li></ul><p><strong>NOTE: </strong><strong>as of now the USB CDC Arduino code is not working properly therefore when you upload a sketch on JACK, then detach and reattach it to the USB port, you will see an <em>Unknown device. </em><span class="redactor-invisible-space">Follow the below guide to reprogram it.</span></strong><br></p><hr><h2>Reset JACK to run the bootloader</h2><p>The<strong> SAM-BA </strong><strong>bootloader </strong><span class="redactor-invisible-space">running on the Zero has a pretty nice feature that you can enable via code, called <strong>double tap reset</strong><span class="redactor-invisible-space">; this feature allows an user to force bootloader startup if there&apos;s any issue on the user code. Here&apos;s how it works:</span></span></p><ul><li>Attach JACK to the USB port (or detach-attach)</li><li><strong>As soon as you attach it</strong>, click the RESET button <strong>twice</strong><span class="redactor-invisible-space"> in a small frame of time</span></li><li><span class="redactor-invisible-space">If everything was fine, you shall know see again the JACK serial port on your PC</span></li></ul><p>The time frame you have to double tap the reset button is not so big (you gotta do that before the USB Host tries to enumerate JACK, so it&apos;s less than half a second) but it&apos;s enough. This feature is pretty cool because it is helping me a lot in this phase where the USB CDC class of the Arduino Core is not yet working.</p><p>Video on how to start the bootloader here:</p><p><iframe width="500" height="281" src="//www.youtube.com/embed/lm4ly-gQs4Q" frameborder="0" allowfullscreen=""></iframe><br></p><hr><h1>System diagrams</h1><h2>Board diagram</h2><p><img src="https://cdn.hackaday.io/images/3359331439740124889.png"></p><h2>Software diagram</h2><p><img src="https://cdn.hackaday.io/images/561911439740140081.png"></p><p><strong></strong></p>

                            <small class="authorship">7 days ago  &bull;

                            <p><p>Hey there,</p><p>As promised, I came back from holidays last Sunday afternoon, and as soon as I got home I took my soldering iron and started assembling one proto. I had some issues cause I only had 1.2mm solder that slowed me down a lot (it took me 4 hours to get it done), but when I connected it to the Atmel ICE it was promptly recognized. Good.</p><p>Programming the bootloader worked too, but then JACK won&apos;t enumerate.</p><p>Monday morning I had to work, so I took my first proto with me along with the Atmel ICE debugger; I fiddled with the Arduino Zero bootloader first, to switch the bootloader pin to anything diffferent than PA15 which is used on my board as input for the 12.288 MHz crystal, I modded the makefile to point to correct directories, and TADA, I now have a JACK bootloader; I added a JACK board definition in the boards.txt file of the SAMD Arduino devices, opened Arduino, flashed the bootloader and and the device DID enumerate, as an &quot;Arduino Zero bootloader&quot; device (just like what you&apos;d except when connecting a Zero&apos;s native USB port, not the EDBG one). I could also flash a Blink sketch, but it won&apos;t start automatically and randomly if I touched some pins it would eventually start. I thought that this issue surely is connected to the clock not being the same as the Zero.</p><p>Monday afternoon I went home early to start assembly of three others boards (the judges&apos; boards); before that I bought a 0.5mm solder that helped me A LOT during production, it took me in fact 4 hours to make THREE boards (that&apos;s 300% faster than Sunday). I had some issues on two boards for the USB transient suppressor chip, on one I messed up 2 of 6 pins of that IC :/ In fact, these boards are still recognized by the Atmel ICE, but one of them won&apos;t show up at all as a USB device, the other would report &quot;Unknown device&quot;. I&apos;ll try to reflow these chips using an hot air gun, if that does not work, I&apos;ll take off some parts and solder them on two spare boards</p><p>Photos here:</p><p><img src="https://cdn.hackaday.io/images/6050491439284973611.jpg">So far so good.</p><p><img src="https://cdn.hackaday.io/images/8338731439285026114.jpg"></p><p>Almost there...</p><p><img src="https://cdn.hackaday.io/images/2081351439285642348.jpg"></p><p>A pair of finished boards, one connected to the Atmel ICE.</p><p>You can also find a video of the device in action (connection to Atmel ICE, bootloader flashing, enumeration as Arduino Zero, programming of Blink sketch, LED blinking):</p><iframe style="width: 500px; height: 281px;" width="500" height="281" src="//www.youtube.com/embed/Me_bltUciJQ" frameborder="0" allowfullscreen=""></iframe><p>To solve the issue that I had yesterday I indeed had to change the Clock Generators; the Zero takes the main oscillator from an external 32.768 KHz crystal while I&apos;m using a 12.288 MHz crystal; these crystals have separate pins. I then reconfigured the Zero startup.c file to use XOSC rather than XOSC32K as main source and you can see in the video that as soon as I program the device with the Blink sketch, it loads it. </p><p>I still have a software issue, that is when I flash JACK with a sketch it won&apos;t enumerate anymore as an &quot;Arduino Zero bootloader&quot; COM port so I cannot program it again. I also enabled an option that if you tap twice the RESET button the bootloader is forced to load again, but it won&apos;t show up either.</p><p>See you soon for other updates.</p><p>Cheers,</p><p>Mick</p>
