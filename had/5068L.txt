                            <h2><a href="/project/5068/log/17627-supercapacitor-will-it-work">Supercapacitor, will it work?</a></h2>
                            <small class="authorship">3 months ago  &bull;
                                <a class="gray-link" href="/project/5068/log/17627#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-17627"><p>My idea is to use a supercapacitor for energy, when there is no sun or wind. I decided to do some tests to make sure the energy stored is enough. </p>
<p>First of all, what would we theoretically expect? The BL600 will on average draw a pretty constant current, independent on the voltage it gets, and there is a linear relationship between current drawn and the voltage drop:</p><p><img alt="\color{White} \large V(t) = \frac{It}{C}" title="\color{White} \large V(t) = \frac{It}{C}" src="/api/latex?tex=%5Ccolor%7BWhite%7D%20%5Clarge%20V(t)%20%3D%20%5Cfrac%7BIt%7D%7BC%7D">This means we can easily calculate the time we can survive on the capacitor:</p><p><img alt="\color{White} \large t =  \frac{C \Delta V }{I}" title="\color{White} \large t =  \frac{C \Delta V }{I}" src="/api/latex?tex=%5Ccolor%7BWhite%7D%20%5Clarge%20t%20%3D%20%20%5Cfrac%7BC%20%5CDelta%20V%20%7D%7BI%7D">If we, let's say, have an average current consumption of 100 µA, the time will be:</p><p><img alt="\color{White} \large V_{0} = 3.3 V" title="\color{White} \large V_{0} = 3.3 V" src="/api/latex?tex=%5Ccolor%7BWhite%7D%20%5Clarge%20V_%7B0%7D%20%3D%203.3%20V"><img alt="\color{White} \large V_{min} = 1.6 V" title="\color{White} \large V_{min} = 1.6 V" src="/api/latex?tex=%5Ccolor%7BWhite%7D%20%5Clarge%20V_%7Bmin%7D%20%3D%201.6%20V"><img alt="\color{White} \large I_{avg} = 100 \mu A" title="\color{White} \large I_{avg} = 100 \mu A" src="/api/latex?tex=%5Ccolor%7BWhite%7D%20%5Clarge%20I_%7Bavg%7D%20%3D%20100%20%5Cmu%20A"><img alt="\color{White} \large C = 2.5 F" title="\color{White} \large C = 2.5 F" src="/api/latex?tex=%5Ccolor%7BWhite%7D%20%5Clarge%20C%20%3D%202.5%20F"><img alt="\color{White} \large t = 11.81 h" title="\color{White} \large t = 11.81 h" src="/api/latex?tex=%5Ccolor%7BWhite%7D%20%5Clarge%20t%20%3D%2011.81%20h">Looks promising, assuming that we only have to survive until the sun comes up, but it's hard to know how close to reality this is. Therefore, I made a couple of tests.<br></p><p>I have the following capacitors at my disposal:</p><ol><li>2.5 F, Bussman ($ 11.50)</li><li>15 F, Vishay ($ 8.93)</li><li>90 F, Vishay ($ 19.16)</li></ol><p>These are the general conditions for the tests:</p><ul><li>Sending power data over bluetooth to a computer with <a href="http://openhab.org">openhab</a>, using the BL600 internal voltage sensor, which in turn saves the data in a mysql database</li><li>Acquiring data every minute</li><li>Using smartBASIC code intended to minimize current consumption, but still work smoothly. The exact code can be found <a href="https://github.com/ulfwin/weatherstation/releases/tag/v0.0.0">here.</a></li></ul><p>Below you can see the results:</p><p><img style="display: block; margin: auto;" alt="2.5 F, Bussman, 5 hours" src="https://cdn.hackaday.io/images/3163351431204008781.png"></p><p>(2.5 F, Bussman, 5 hours)</p><p><img style="display: block; margin: auto;" alt="90 F, Vishay, 72  hours" src="https://cdn.hackaday.io/images/5987651431204061521.png"></p><p>(90 F, Vishay, 72 hours)</p><p>The 15 F capacitor did not even last 5 minutes, so not much result to show there. My conclusion is that both Vishay capacitors are so unreliable that I don't want to use them (even though the 90F capacitor lasted an amazing 3 full days). The bussman capacitor has a beautiful linear behavior, as expected. 5 hours is not enough, but it turns out you can get an affordable 5 F capacitor from the same manufacturer for $ 10.50 (it's cheaper because it's rated at 5 V instead of 5.6 V). Furthermore, for these tests I have only been able to charge the capacitor to 3.3 V. If I use a DC-DC converter I can charge it all the way up to 5 V.</p><p>We can also see that 100 µA was quite an underestimation. Using the experiments we can get a more accurate value:</p><p><img alt="\color{White} \large t = 5 h = 18000 s" title="\color{White} \large t = 5 h = 18000 s" src="/api/latex?tex=%5Ccolor%7BWhite%7D%20%5Clarge%20t%20%3D%205%20h%20%3D%2018000%20s"><img alt="\color{White} \large I_{avg} = \frac{C \Delta V }{ t}" title="\color{White} \large I_{avg} = \frac{C \Delta V }{ t}" src="/api/latex?tex=%5Ccolor%7BWhite%7D%20%5Clarge%20I_%7Bavg%7D%20%3D%20%5Cfrac%7BC%20%5CDelta%20V%20%7D%7B%20t%7D"><img alt="\color{White} \large I_{avg} = 236 \mu A" title="\color{White} \large I_{avg} = 236 \mu A" src="/api/latex?tex=%5Ccolor%7BWhite%7D%20%5Clarge%20I_%7Bavg%7D%20%3D%20236%20%5Cmu%20A">The <a href="http://www.lairdtech.com/brandworld/library/BL600%20Hardware%20Integration%20Guide.pdf">BL600 Hardware Integration Guide</a> mentions a possible average current of 4 µA, so a lot of improvement should be possible (although data transfer is not taken into account in that number, only keeping a connection). The solar panel I intend to use charges at up to 50 mA, so a lot of tolerance there as well.<br></p>In other words, it looks promising! Next, I want to decide all components to use, and make the PCB layout.</div>
                            </div>

                            <h2><a href="/project/5068/log/19275-slow-vcc-rise-time">Slow VCC Rise Time</a></h2>
                            <small class="authorship">2 months ago  &bull;
                                <a class="gray-link" href="/project/5068/log/19275#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-19275"><p>When the weather station is powered by the sun and wind, the increase of VCC will be quite slow, while the capacitor is charged. This might cause the microprocessor to not start up properly, so I decided to test it. </p>
<p>There is reset circuitry built in to the BL600 to take care of such problems, but looking at the <a href="http://www.lairdtech.com/brandworld/library/BL600%20Hardware%20Integration%20Guide.pdf">data sheet</a>, it is not enough. A minimum rise time of 60 ms from 0 to 1.8 V is mentioned. Anyway I set out to test how bad it actually is by charging and discharging the capacitor 10 times, while observing if connection is made with my host computer. I have made <a href="https://github.com/ulfwin/weatherstation/blob/master/client_script/ble2openhab.py">a script</a><a href="https://github.com/ulfwin/weatherstation/blob/master/client_script/ble2openhab.py"></a> to continuously try to make a connection, and I uploaded <a href="https://github.com/ulfwin/weatherstation/tree/slowVCCRiseTimeTest">smartBASIC code</a> I new worked well.</p>

At least for the first prototype that should be good enough.</div>

                            <h2><a href="/project/5068/log/20740-bl600-communication-board">BL600 Communication Board</a></h2>
                            <small class="authorship">a month ago  &bull;

                            <div style="position:relative">
                                <div id="post-body-20740"><p>I will need to design a PCB for the weather station, where I will place the BL600 and the super capacitor, amongst other things, but I will also need something to communicate to the BL600. Since the weather station should be small and as cheap as possible, I want to push the  RS-232 to UART conversion, power supply and JTAG circuitry out from the main board. Therefore, I've designed a separate board for that (see pictures below).</p><p><img style="width: 392px; height: 250px;" src="https://cdn.hackaday.io/images/8929021436475699008.png" height="250" width="392"></p><p><img style="width: 345px; height: 356px;" src="https://cdn.hackaday.io/images/209201436475732845.png" height="356" width="345"></p><p>The source files can be found <a href="https://github.com/ulfwin/WeatherStationPCBs">here</a>. Next post will be about the design of the main board.<br></p></div>

                            <h2><a href="/project/5068/log/22521-wind-vane-pcb">Wind Vane PCB</a></h2>
                            <small class="authorship">7 days ago  &bull;
                                <a class="gray-link" href="/project/5068/log/22521#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-22521"><p><img src="https://cdn.hackaday.io/images/7838111439313527134.png"></p><p>To measure the wind direction I need a wind vane. A simple way to do that is to use hall sensors and a small magnet, which rotates with respect to the sensors (the hall sensor triggers an output when the magnet is close). My solution is to use four such sensors, one for each cardinal direction (north, south, west, east).</p><p>Note that this does not mean I&apos;m restricted to measuring only four directions; by placing the magnet at the correct distance, I will get two sensor readouts when it&apos;s in between directions. In other words I get eight directions (see table).</p><table><tbody><tr><td><strong>Direction</strong></td><td><strong>Readout (N,W,S,E)</strong></td></tr><tr><td>North</td><td>1,0,0,0</td></tr><tr><td>North-East</td><td>1,1,0,0</td></tr><tr><td>East</td><td>0,1,0,0</td></tr><tr><td>South-East</td><td>0,1,1,0</td></tr><tr><td>South</td><td>0,0,1,0</td></tr><tr><td>South-West</td><td>0,0,1,1</td></tr><tr><td>West</td><td>0,0,0,1</td></tr><tr><td>Noth-West</td><td>1,0,0,1</td></tr></tbody></table><p>The most straight forward way to connect this to the BL600 processor is to use in total six wires; four to BL600 inputs and two for GND and supply. That&apos;s a bit of waste of inputs that can be used for other peripheral devices, and I need the vane to be quite far from the main PCB so I&apos;d also like to reduce the amount of wires. Therefore I decided to combine the four digital signals into one analog.</p><p>It&apos;s quite common to combine several buttons into one analog signal (see example <a href="https://arduino-info.wikispaces.com/Keyboards-MultipleButtons">here</a>). The problem, though is that they most of them assume that you&apos;re only going to press one button at a time. For example, in the picture below, if I press &quot;RIGHT&quot;, it doesn&apos;t matter what other button is pressed, since &quot;AD0&quot; is already grounded.</p><p><img style="width: 216px; height: 359px;" src="https://cdn.hackaday.io/images/4066661439311885284.jpg" height="359" width="216"></p>Instead I used this approach:<p><img src="https://cdn.hackaday.io/images/8076901439312032588.png"></p> By choosing different values of the resistors N, E, S and W, I will get different voltage outputs for combinations of presses. I made a spreadsheet to fiddle with resistor values (see <a>here</a>) and after a while I found a configuration with maximu<a href="https://docs.google.com/spreadsheets/d/1oHtwEPnT-i8HeHX16upyjqc0DSVR5-lAH4XNxVrh8Qw/pubhtml?gid=367533195&amp;single=true"></a>m 70 mV difference between the states I care about. The worst case ADC resolution is about one tenth of that, so that should be fine.<br><p>Considering that I need quite accurate placement of the sensors, I decided to make a PCB. Below you can see the schematic and layout.</p><p><img style="width: 299px; height: 309px;" src="https://cdn.hackaday.io/images/6092481439310075947.png" height="309" width="299"></p><p><img style="width: 314px; height: 302px;" src="https://cdn.hackaday.io/images/1638191439313075555.png" height="302" width="314"></p><p>As usual, all the source files can be found through the project links.</p></div>
