                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">The WatchDuino project started as an attempt to build a smartwatch at the reach of everyone: inexpensive and open. So that anyone can afford one, or even build it from scratch. And at the same time, so that anyone can built their custom applications for it.<br /><br />WatchDuino2 is an improved redesign from scratch over the previous version (featured on the Hackaday blog a year ago! <a href='http://hackaday.com/2014/05/06/tell-time-and-blink-an-led-on-your-wrist-with-watchduino/' target="_blank">http://hackaday.com/2014/05/06/tell-time-and-blink-an-led-on-your-wrist-with-watchduino/</a> ) . It's main feature is including a BLE component so that it can communicate with an Android phone, and have a featureset very similar to that of commercial smartwatches.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p><strong>New bluetooth-based apps</strong><br></p><p>The main improvement over the previous version is the BLE component, that allows it communicate with an Android phone. This is incredibly powerful, as effectively makes the phone become a &quot;second brain&quot; which is connected to the Internet, and can provide the watch with many useful data that was previously out of it&apos;s reach.<br><br>Thanks to the bluetooth pairing, we were able to build the following applications in the watch:<br><br>- time synchronization<br>- display phone notifications in the watch</p><p>- send battery status from watch to the phone, to make graphs<br>- watch twitter app (will show you the latest tweets from your timeline)<br>- watch weather app (will retrieve the weather in your location from <a href="http://forecast.io" target="_blank">http://forecast.io</a>)<br>- watch TFL app (will tell you when the next buses/tubes will arrive based on your location in London).<br><br>The trick is that all of these apps also require an Android component in the WatchDuino Android companion app, doing all the heavy lifting and simply passing WatchDuino the results to display, via Bluetooth.</p><p><strong>Notifications from the phone</strong></p><p>The best feature in WatchDuino is the capability of sending notifications from your Android phone (just like all the good smartwatches do). By using the Notification Listener Service we can redirect <strong>any </strong> notification to the watch via Bluetooth automagically (SMS, Emails, Whatsapp, Facebook, Google Now, calendars, etc). Then, once the notification arrives to the watch, you can just press a button to dismiss that notification in your phone too.</p><p><img src="https://cdn.hackaday.io/images/9156441439722327683.jpg"></p><p><strong>New UI</strong></p><p><br>Since we are using a different screen, with a better resolution (128x64), we were able to build a more flexible UI.<br><br>The top of the screen is showing a status bar, similar to what smartphones do. The 4 corners of the screen show contextual symbols that indicate what the purpose of the buttons is in every occasion. </p><p>For instance, in the main menu you can move left and right, enter or exit.</p><p><img style="width: 342px; height: 258px;" src="https://cdn.hackaday.io/images/748801439677462674.jpg" height="258" width="342"></p><p><br>On an app, the functionality of the buttons can change. For instance, on the Twitter app you can request a reload, or navigate between tweets.</p><p><img style="width: 333px; height: 249px;" src="https://cdn.hackaday.io/images/796621439677497299.jpg" height="249" width="333"></p><strong></strong><p><strong>Next Challenges</strong><strong></strong></p><p>There is still plenty to do! We have a working prototype with some cool apps, but there&apos;s much polishing until we can reach a product that actually useful and nice to use in your everyday life:</p><p><br>- We need to polish the code, and specially the app-building framework. The purpose is to make it easy for anyone to write and integrate new apps.<br>- We need to improve battery life. Currently WatchDuino 2 is able to run for 18 hours after a 20 minute charge. This is not to bad if you compare it to commercial fully fledged smartwatches, but for a device that is using an arduino processor and a really non-fancy screen, we expect that the battery life can be drastically improved. On the original WatchDuino we managed to use some clever optimizations to expand the battery life from days to months, and we hope we&apos;ll be able to do the same with this one.<br>- We need to make it smaller, prettier and easier to assemble. Our current prototypes are hand soldered, and still we managed to make it small enough to be wearable. But we can do better if we manage to integrate the components into a PCB.</p>
                                
                            </p>
                        </div>

                            <h2><a href="/project/7244/log/23034-watchduino-now-tells-you-when-your-bus-is-coming">WatchDuino now tells you when your bus is coming</a></h2>
                            <small class="authorship">2 days ago  &bull;
                            <a class="gray-link" href="/project/7244/log/23034#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>We live in London, which is a huge city. Moving around can be a challenge with 11 different underground lines and over 600 bus lines (no joking! <a href="https://en.wikipedia.org/wiki/London_Buses">https://en.wikipedia.org/wiki/London_Buses</a>).</p><p>Luckily, the TFL (organization managing all things Transport For London) does a wonderful job at having real time data for the status of the transport, and not only that, it also exposes that data though an API. Then other wonderful apps such as Google Maps or Citymapper can make use of it to help you find your way around the city.</p><p>When you are waiting on a bus stop in the middle of winter in London you can&apos;t help but to wonder when your bus is due. And you want to do without taking your gloves off also! Wouldn&apos;t it be great to just know just by looking at your wristwatch?</p><p>Now you can, with WatchDuino2! We&apos;ve built an app that will show a list of the next busses, with times, that will arrive at the bus stop where you are staing. When entering the watch app, WatchDuino will contact your Android phone to request for the bus info. Then the Android app with grab your geoposition and query the TFL API to compute your nearest bus stop and the times of the next buses arriving, and send that info back to the watch.<br></p><p><img src="https://cdn.hackaday.io/images/8140051439724707932.jpeg"></p>5 minutes till bus 19? Uhm, might still have time to grab some chips at McDonals!<br>
                                
                            </p>
                        </li>
                    
                        <li>
                            <h2><a href="/project/7244/log/23032-migrated-android-companion-app-to-cordova">Migrated Android companion app to Cordova</a></h2>
                            <small class="authorship">2 days ago  &bull;
                            <a class="gray-link" href="/project/7244/log/23032#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p> As you know, much of the heavy lifting of the WatchDuino apps is actually not done in the watch itself, but on the phone. We made a framework for serializing and handling the bluetooth communications from and to the watch. </p><p>For WatchDuino to work, a companion app needs to be running on Android as a service. This companion app contains the messaging framework, plus the bit of logic for every phone app, that usually is in charge of querying some web service and producing a response with data to pass to the watch.</p><p>For the sake of debugging and testing, we made a simple interface to the companion app so that we can easily pair the devices, inspect all the messages going through, plus triggering notifications from each of the apps, or simply ping the watch to test the connection.<br></p><p><img style="width: 407px; height: 726px;" src="https://cdn.hackaday.io/images/4314651439722967671.png" height="726" width="407"></p><p>One of our biggest sources of frustration when developing WatchDuino2 has been working on the Android companion app. Android apps need to be programmed in Java, which is not a great language for rapid prototyping.</p><p>So we decided to switch to Apache Cordova (<a href="https://cordova.apache.org/">https://cordova.apache.org/</a>) a couple months ago, which is a framework for writing Android apps as if they were web applications, by providing a set of libraries to access the mobile APIs from HTML5 and Javascript. </p><p>Being web developers we are much more comfortable with this kind of stack, so we migrated all of the functionality of the companion app to a new one written with Cordova.</p><p><img style="width: 390px; height: 694px;" src="https://cdn.hackaday.io/images/1516851439723104156.png" height="694" width="390"></p><p>Now we are able to make progress faster, with better code, and less bugs. Much win!</p>
                                
                            </p>
                        </li>
