                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">This project was done as a Master's degree semester project at EPFL in the Robotics and Autonomous Systems orientation. 5 teams of 3 people had to build a bottle recycling robot, this project won. Every single bit of information from the conception and design to mechanical parts and source code is available, published on GitHub.<br /><br />We used a Raspberry Pi with an Arduino (clone) to control the mobile platform. The bottle recognition software is based on the Haar feature-based cascade classifier for object detection via OpenCV. The code on the Raspberry Pi is written in Python, it offered faster development time than C/C++, but with more overhead. This turned out to be useful as this robot won the competition just in the nick of time.<br /><br />The documentation is super extensive as it was a requirement and every single detail from initial ideas to final parts and simulation files are all available.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>The robot does obstacle avoidance until the camera sees and recognises a bottle, once a bottle is detected with the Python script running on the Raspberry Pi its position is forwarded to the Arduino board that hosts the main program in a state machine which passes to the next state: grab the bottle.</p><p>Once the bottle is grabbed the homing system is quite simple: it follows the direction towards the recycling area (corner of the map) using a compass and once it detects a wall (and not an obstacle) it follows it until it reaches the corner and releases the bottle. Repeat.</p><p><img src="https://cdn.hackaday.io/images/6734601403267908011.png"></p><p>The initial plan was to use 2 IR sensors and the camera for obstacle detection, but the camera was too slow and ultimately the simulation showed that more IR sensors would be a simpler solution and work just as well, so we decided to use 4 IR sensors and use the camera only for bottle detection.</p><p>The compass sensor had a lot of trouble in its original position behind the camera, it turns out the metal bars in the floor of the building structure were influencing the compass values, mounting it higher up on a mast fixed that problem.</p><p>The full project report can be found here in pdf format:&nbsp;<a href="https://github.com/Nurgak/STI-competition/blob/master/Documents/Project%20report.pdf?raw=true">https://github.com/Nurgak/STI-competition/blob/master/Documents/Project%20report.pdf?raw=true</a></p>
                                
                            </p>
                        </div>
