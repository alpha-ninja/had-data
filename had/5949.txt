                            <h2><a href="/project/5949/log/19460-software-control">Software Control</a></h2>
                            <small class="authorship">2 months ago  &bull;
                            <a class="gray-link" href="/project/5949/log/19460#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>I didn't want to go too long without an update on this.  I've done some significant work on the software (PC side) to configure the instrument.  The way this will work is:</p><ol><li>Define all the play settings with this application.</li><li>Connect MIDI-OUT from your PC to MIDI-IN on MIDI Hero.</li><li>Software sends MIDI SysEx message containing new settings to MIDI Hero, which then saves them to EEPROM in a bank of presets.</li><li>Presets loaded from EEPROM when the instrument is actually used to perform - swap from acoustic to electric on the fly, or between chord sets, etc.</li></ol><p>Here's a screenshot of the program.</p><p><img src="https://cdn.hackaday.io/images/4713751434251669289.png"></p><p>Still a work-in-progress, but much of the functionality is there.  Don't be alarmed by the Windows 7 UI - the whole thing is being written in... wait for it... FreePascal using the Lazarus IDE.  So that makes it easily cross-platform when I release the code later.</p><p>This is my first time using Pascal / Delphi / etc.  Can definitely see why it was the beginner language back in the day, but I'm treating it more like a cross-platform VB6 and it seems like a good fit.<br></p>
                                
                            </p>

                            <h2><a href="/project/5949/log/18448-just-add-arduino">Just Add Arduino</a></h2>
                            <small class="authorship">3 months ago  &bull;
                            <a class="gray-link" href="/project/5949/log/18448#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>For my first prototype, let's take my Arduino and solder wires all over the insides of this.</p><p><img src="https://cdn.hackaday.io/images/7184581432536116735.jpg"></p><p>Well that was easy.  Digital inputs on practically everything, except the potentiometer which is wired to Analog Input 0.</p><p>Now for the fun part: using the ever comfortable, highly productivity enhancing Arduino IDE.  Because I'm a pleb engineer, I didn't roll my own highly efficient MIDI implementation, and instead just dumped in the <a href="https://github.com/FortySevenEffects/arduino_midi_library/">Arduino MIDI library</a>.</p><p>The way this works is:</p><ul><li>buttons (start, select, tilt) fire "General Purpose" MIDI CC messages: on when held, off when released.</li><li>Whammy bar sends Pitch Bend messages to drop the pitch of all playing notes.</li><li>When upstroke or downstroke strum buttons are pressed, the Frets are read, and one of 2^5 (=32) sets of 6 notes are selected.  These are played one at a time with some microsecond delay between, to simulate actual guitar strumming.</li></ul><p>At least, that's the theory, but there are some bugs in the prototype.  One of the frets seems permanently stuck "on", the whammy bar sends pitch bend nonstop all over the place, and so on.</p>
                                
                            </p>
                        </li>
