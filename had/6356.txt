                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">The number of hardware PWM channels is usually limited. The alternative, software PWMs, leads to slow refresh rates and low precision for the PWM duty cycle. Delta-Sigma is much better then PWM, in both the duty cycle accuracy and the number of instructions.<br /><br />This is a demonstration of 10 LEDs intensity modulators (as many as the MCU's available pins) running on a MSP430G2211 without flickering.<br /><br />Sigma-Delta modulation algorithm is implemented using "synthetic division", and is calculated in just 5 clock cycles (5 instructions) per each LED.<br /><br />The project was inspired by a Jason Sachs's article (link inside code). Thanks Jason!</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p><iframe src="//www.youtube.com/embed/jwq-51bcTuw" allowfullscreen="" frameborder="0" height="281" width="500"></iframe></p><p>The code that calculates the Sigma-Delta "synthetic division" is made of just these 5 lines:</p><pre class="hljs perl">// Sigma delta modulation algorithm using <span class="hljs-string">"synthetic division"</span>		sum[n] += re<span class="hljs-string">q[n]</span>;		<span class="hljs-regexp">//</span> Update integrator value		<span class="hljs-keyword">if</span> (sum[n] &lt; max[n])			outBits++;		<span class="hljs-regexp">//</span> LSB = <span class="hljs-number">1</span>		<span class="hljs-keyword">else</span>			sum[n] -= max[n];	<span class="hljs-regexp">//</span> LSB = <span class="hljs-number">0</span> (untached) <span class="hljs-keyword">and</span> adjust integrator</pre><p>The complete code can be found in the GitHub repository. The rest of the lines from file <a href="https://github.com/RoGeorge/Delta-Sigma_versus_PWM/blob/master/main.c">https://github.com/RoGeorge/Delta-Sigma_versus_PWM/blob/master/main.c</a> is just waving the colors.</p><ul><li><strong>sum[n]</strong> is an accumulator, similar with the PWM counter</li><li><strong>req[n]</strong> is the duty cycle, equivalent of the PWM compare register</li><li><strong>max[n]</strong> is the overflow value for the accumulator</li><li><strong>n</strong> is the channel number, in this demo there are 10 independent "PWM like" channels</li></ul><p>Apart from the intimidating name, "synthetic division", the implementation is pretty simple:</p><p>At each step, add the requested duty cycle to an accumulator. When the accumulator overflows, you set the output to high. If it doesn't overflow, you reset the output to low. Then, in the next step, continue adding from where you were left.</p><p>The maximum advantage regarding the refresh rate is at 50% duty cycle, where Delta-Sigma becomes 'max/2' times higher then PWM, where 'max' is the overflow value.</p><p>As an example, for an 8 bits PWM at 50% duty cycle, Delta-Sigma has a refresh rate 128 times higher then a PWM. For a 16 bits PWM, the Delta-Sigma would be 32768 times better at refresh rate.</p><p>One more advantage is that Delta-Sigma is more responsive. You can modify the duty factor on the fly, without waiting for the whole cycle to end. Of course, you can also choose a different resolution for each channel by modifying <strong>max[n]</strong>.</p><p><strong></strong></p>
                                
                            </p>
