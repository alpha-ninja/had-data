                    <h5>Description</h5>
                    <div class="description">BowlerStudio is a robot development application that combines scripting and device management with powerful control and processing features.<br /><br />BowlerStudio brings the full power of The OpenCV image processing library, a Configurable kinematics engine based on D-H parameters, A Fully Featured JCSG based Cad and 3d Modeling engine, and a built in WebKit browser capable of viewing documentation and executing code directly from Github Gists.<br /><br />Scripts can be loaded from the Local File System and edited using a editor of your choice or loaded from straight out of the web browser from any compatible Github gists for a truly "backed up in the cloud" coding experience.<br /><br /><br /></div>

                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>BowlerStudio assists you in every step of a robotics project from concept to completion. Tools enable users to:</p><ul><li>Interface with motors, sensors, and other electronics hardware.</li><li>Create 3d models for fabrication, and for simulating the motions of your project.</li><li>Give your robot sight with image processing on camera feeds and Kinect data.</li><li>Operate 3d printers and other CNC machines.</li><li>Create custom graphical user interfaces to control yours robots.</li><li>Create and control animations.</li></ul><h2><a id="user-content-the-nitty-gritty-version" class="anchor" href="https://github.com/NeuronRobotics/BowlerStudio#the-nitty-gritty-version"></a>The Nitty-Gritty Version</h2><p>BowlerStudio Robotics development IDE is based on</p><ul><li><a href="https://github.com/miho/JCSG">JCSG</a></li><li><a href="https://github.com/NeuronRobotics/java-bowler">Java-Bowler</a></li><li><a href="http://opencv.org/">OpenCV</a></li><li><a href="https://github.com/bytedeco/javacv">JavaCV</a> which provides</li><li><a href="http://ffmpeg.org/">FFmpeg</a></li><li><a href="http://damien.douxchamps.net/ieee1394/libdc1394/">libdc1394</a></li><li><a href="http://www.ptgrey.com/products/pgrflycapture/">PGR FlyCapture</a></li><li><a href="http://openkinect.org/">OpenKinect</a></li><li><a href="http://muonics.net/school/spring05/videoInput/">videoInput</a></li><li><a href="http://studierstube.icg.tugraz.at/handheld_ar/artoolkitplus.php">ARToolKitPlus</a></li><li><a href="http://cmp.felk.cvut.cz/~uricamic/flandmark/">flandmark</a></li><li><a href="https://github.com/acamilo/CHDK-PTP-Java">CHDK-PTP-Java</a></li><li><a href="https://github.com/jinput/jinput">Jinput</a></li><li><a href="http://motej.sourceforge.net/">motej</a></li><li><a href="https://github.com/usb4java/usb4java">Usb4Java</a></li><li><a href="https://github.com/NeuronRobotics/nrjavaserial">NrJavaSerial</a></li><li><a href="https://github.com/hcarver/bluecove">BlueCove</a>  </li><li>JavaFX 8 3d engine.</li></ul><p>BowlerStudio is a device manager, scripting engine, CAD package, and simulation tool all in one application. A user can develop the kinematic of an robot arm using the D-H parameters-based automatic kinematics engine. With this kinematics model, the user can then generate the CAD for new unique parts to match the kinematic model. The user can then export the model to an STL, and connect a Bowler 3d printer to BowlerStudio. The printer can print out the part (using the newly generated STL) while the user connects a DyIO and begins testing the servos with the kinematics model. When the print is done, the user can assemble the arm with the tested servos and run the model again to control the arm with Cartesian instructions. Once this is complete, the user can then attach a wiimote to train the robot arm through a set of tasks, recording them with the animation framework built into BowlerStudio. To be sure the arm is moving to the right place, the user can attach a webcam to the end and use OpenCV to verify the arm's position, or use the arm (in conjunction with the webcam with OpenCV enabled) to track and grab objects (IE "eye-in-hand" tracking).</p><p>Every step of this task can be performed from within BowlerStudio!</p><p>Let's go through the main features:</p><h1><a id="user-content-scripting-with-gist" class="anchor" href="https://github.com/NeuronRobotics/BowlerStudio#scripting-with-gist"></a>Scripting With Gist</h1><a id="user-content-about-scripts-and-gist" class="anchor" href="https://github.com/NeuronRobotics/BowlerStudio#about-scripts-and-gist"></a>About scripts and Gist<p>Scripts are bits of code that BowlerStudio can load and run. BowlerStudio allows you to open a local file and run it, but BowlerStudio is most powerful when the code lives on Github Gist (a code snippet hosting service from Github). Simply give BowlerStudio the URL for a Gist you want to load and execute. Gists can be selected and edited using the built in browser, or inline in another script using the Gist ID.</p><a id="user-content-java-and-groovy" class="anchor" href="https://github.com/NeuronRobotics/BowlerStudio#java-and-groovy"></a>Java and Groovy<p>BowlerStudio can load and run scripts written in Java, Groovy, and Python. Which parser is used is determined by the file extension. Files that end in .java or .groovy will be run through the Groovy compiler. These Groovy scripts are compiled fully and run directly in the JVM. This means they will execute at full speed, just like a regular application.</p><a id="user-content-python" class="anchor" href="https://github.com/NeuronRobotics/BowlerStudio#python"></a>Python<p>Python, on the other hand, by virtue of its structure, will generally execute much slower then Java. With the reduction in speed you get lots of flexibility and a clean and easy to understand syntax. The python code can also create and return objects to BowlerStudio (such as CAD CSG objects, or UI Tabs).</p><a id="user-content-return-objects" class="anchor" href="https://github.com/NeuronRobotics/BowlerStudio#return-objects"></a>Return Objects<p>A script can return a few object types that will be handled by BowlerStudio: Objects of type "CSG" and "MeshView" will be added to the 3d display. If a transform is added to either of these and updated by a script the user can move an object in the 3d view. Objects of type "Tab" will be added to the Tabmanager and displayed in BowlerStudio. This is an easy way to make control panels or state displays and monitors. Objects of type "BowlerAbstractDevice" (or any subclass) will be added to the connections manager and made available to all other scripts. These can be external devices or virtual communication bus devices. A bowler Server/Client pair is the preferred mechanism for communication between scripts.</p><a id="user-content-device-access" class="anchor" href="https://github.com/NeuronRobotics/BowlerStudio#device-access"></a>Device Access<p>All scripts are passed all connected devices by name when the script is run.                             <p><p>Neuron Robotics Cooperative is a worker owned enterprise that designs and deploys robot controllers to make the process or building robots easy. While i understand that is a bold statement, we have built software based on top-tier research grade and medically certified robotics control platform, and added an easy to use and learn application interface, BowlerStudio.</p><p>To that end, the DyIO and BowlerStudio should be concidered as one project, seperated out here on Hackaday.io for the purpose of making each easier for members of the site to understand.</p><p>That said, we have already begun using this robotics design platform as a way to teach kids programming, and keep their attention from day one with robots! The Dyio takes that excitement out of the computer and brings to life the printed parts that the students generated. We see this platform solving the large air-gap in the learning process between introduction to the concept of programming and building robots, and the final satisfaction of having actually created a robot.
