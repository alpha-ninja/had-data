                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">This is  a small seven segment LED display. It fits on Atmel's ISP header which is available on most development boards and most AVR-based 'duinos. This header is normally only used for Programming, so after Programming you can plug this Diplay on the header to get an easy to use display. Beside the Hardware this project will also provide Libraries for the Arduino IDE (C++) and Atmel Studio (C). <br /><br />With this you will  get an easy and quick to use display for your projects. Just plug it on the the ISP header and you are ready to start start!</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><h2>Hardware</h2><p>There is nothing special about the Hardware: on the ISP header you have access to three IOs of the micro controller.  These are connected to shift-register, which drives the Leds.</p><p><img src="https://cdn.hackaday.io/images/5309441437425380608.png"></p><p>The PCB should be small. So it can be used quick and easy. So I ended with this, after some hours of routing:</p><p><img style="width: 294px; height: 208px;" src="https://cdn.hackaday.io/images/4970231437314973556.jpg" height="208" width="294"></p><p>The used seven segment displays have a very clever pin out (this pin out is also very conmen, so nothing special): The common ground pin are in the middle. You can mount the display on both sides of the PCB and only the order of the segments changes. This could be reversed in software. </p><p>In the second revision two pin headers on both sides of the PCB got added. With these pin headers you can solder two (or more) PCBs together to get a two digit display. This second revision is still quite new. There will be further Information soon. </p><h2>Software<br></h2>On the software side of this project there is only a early version of an Arduino Library yet ( see Bitbucket). With this Library you can control one display. <p>On the Software side I have to Improve the Arduino libary and stat to create the C library for Atmel Studio. </p><h2>Future / to do</h2><ul><li>improve and expand Arduino Library (multi segment support)</li><li>make C library for Atmel Studio</li><li>add support for common Anode displays (invert output and switch common pins to vcc)</li><li>...</li></ul><h2>Used Tools and Software</h2><p><em>Cadsoft</em>,  <strong>Egale 7.2 Freeware</strong> for circuit an PCB<br><em>Jerome Lamy</em>, <strong><a target="_blank" href="https://eagleup.wordpress.com/">egleUp</a></strong> for the 3D images (freeware)<br><em>ImageMagick Studio LLC</em>,<strong>  ImageMagick<a target="_blank" href="http://www.imagemagick.org"></a> </strong>used by eagleUp (open source, Apache 2.0)<br><em>Trimble Navigation,</em> <strong>SktechUp Make</strong> for the 3D images <br><strong>Arduino IDE<br>Notepad++ v6.4.2<br>GIMP 2.8<br></strong></p><h2>Credits</h2><p>First I would like to thank to everyone who write comment, following this project or gave a skull. Specially  <a data-mention-user-id="39031" href="https://hackaday.io/hacker/39031">@Stefan Lochbrunner</a> who had the idea of dais chain multiple displays. </p>

                            <p><p>So the library (see bitbucket) does now support all three kinds of mounting the display. Just use different argument at the .begin function like: </p><pre class="hljs cpp"> myTiny7.begin(<span class="hljs-number">3</span>);<span class="hljs-comment">//if your display is mounted normal use no argument or 0 else:</span><span class="hljs-comment">// 0: normal 		(default)</span><span class="hljs-comment">// 1: mirror 		mounted on the other side of the PCB over the IC</span><span class="hljs-comment">// 2: rotated		mounted on normal side (over the QR-Code) but upside down</span><span class="hljs-comment">// 3: mirror + rotated 	mounted on other side upside down </span></pre>Also Attiny85/456 based boards like the Trinket should be supported now. Then the default pin out is: <br>1: SDO serial data out<br>2: Clock<br>3: latch<br>Change this as additional parameter in the .begin function, if needed, like this:<pre class="hljs cpp">myTiny7.<span class="hljs-keyword">begin</span>(<span class="hljs-number">3</span>, <span class="hljs-constant">SDO_PIN</span>, <span class="hljs-constant">CLK_PIN</span>, <span class="hljs-constant">LATCH_PIN</span>);</pre>
