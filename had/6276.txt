The SAM D21 line, and its cheaper D20 counterpart, are powerful Cortex M0+ micro controllers, but Atmel&#39;s &#39;Xplained Pro&#39; development boards are rather expensive - as is the new Arduino Zero, which utilises one of these chips. 

This project aims to create simple, breadboard friendly PCBs for the 32-pin and 48-pin variants (possibly the 64-pin variant too, if the need arises), with all of the required component footprints, and some spares (USB, 32K Oscillator, etc) that can be populated as necessary.

                            <a class="gray-link" href="/project/6276/log/19572#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>Arduino LLC have released their latest IDE today - version 1.6.5. Until now, I have been unable to verify whether it could be possible to program the Arduino Zero version of these breakout boards. I have looked through the source code of this new version of the IDE, and found that it should - theoretically - be possible to use the Arduino IDE to program an Arduino Zero bootloader onto the SAM D21G18 processor. This would remove the need for the end user of this ZeroNano board (new title?) to own an expensive programmer, and allow them to use the native USB programming option. </p><p>As a result, I can now confirm that I have plans to create both a full 48-pin breakout board, and in addition, another version of these breakout boards. This third board will (hopefully) be pin compatible with the Arduino Nano 3.0, but use the new Arduino Zero processor!</p><p>If you have any interest in owning a ZeroNano, please do let me know. At present I intend to design the schematic/PCB and open source it to the world, however if there is interest then I shall also look into a small production run (this wouldn't affect open sourcing the design, I promise!).</p><p>I'm very happy with today's news, it means that a ZeroNano is a real possibility! I have almost completed my 32-pin breakout board for the SAM D20/D21 series, and I've bumped designing the ZeroNano to the top of my priority list. It is possible I will use a separate project for the ZeroNano, and keep this for the SAM D20/D21 breakout boards, to differentiate the two. If this happens, I will update this log with a link to the new project!</p><p>Until then, please keep an eye out!</p>
                                
                            </p>
                        </li>
                    
                        <li>
                            <h2><a href="/project/6276/log/19423-entry-1-32-pin-schematic">Entry 1 - 32-Pin Schematic</a></h2>
                            <small class="authorship">2 months ago  &bull;
                            <a class="gray-link" href="/project/6276/log/19423#discussion-list">
                                2 comments
                            </a>
                            </small>
                            <p><h2>Intro</h2><p>The project here on <a href="http://hackaday.io" target="_blank">http://hackaday.io</a> was created after I had finished (ish) the 32-pin variant of the breakout board. It is compatible with both the SAM D20 and D21 lines from Atmel, using the D20E / D21E variants (E corresponds to the 32-pin variants). </p><p>The PCB uses the reference designs in the datasheet for the SAM D21, which was checked against the SAM D20 datasheet to make sure there were no major conflicts. The D21 has full speed USB 2.0 Host and Device, so there is also a USB footprint, with some supporting components (Micro USB Socket, ESD protection diodes, and a filter for the cable shield). </p><p>I've crammed as much as I could onto the PCB, with the intention of only populating the footprints that are necessary for each application. Below is the schematic broken down into its relevant parts, with a brief explanation.</p><h2>Chip Footprint</h2><p><img src="https://cdn.hackaday.io/images/5939221434204530150.png"></p><p>The 32-pin footprint, with all of the pins connected to the two 16-pin breakout headers. These are standard 0.1" pitch headers, with one on each side of the board. All of the nets are named and labelled.</p><h2>Bypass Caps</h2><p><img src="https://cdn.hackaday.io/images/2196531434205089515.png"></p><p>Standard parts for most microcontrollers. These help smooth out the power lines. </p><h2>Programming Header</h2><p><img src="https://cdn.hackaday.io/images/5604211434205609111.png"></p><p>Unlike the ever popular Arduino, these chips do not have a nice bootloader pre-programmed in (The D21 does have an example USB bootloader in Atmel Studio 6, but you'll still need to program it onto the chip!). The pins are 0.05" pitch, as per the standard Cortex Debug Connector. The pin headers for these are a little expensive, so I plan to make a breakout board (gotta love the breakout boards!) so I can connect the programmer to pogo pins, and then just use the pads on the board. </p><h2>Reset Button</h2><p><img src="https://cdn.hackaday.io/images/6902961434207232722.png"></p><p>Into the optional parts! Sometimes, power-cycling a board can be a little tough, especially if you're doing a lot of resets during development. For that need, comes the very popular reset button! The usual 6mmx6mm tactile switch was far too large to fit on my PCB, and my usual distributer (Farnell UK) seem to have a sparse range of thinner switches; as a result, there are two footprints, one for a surface mount DTSM-3, and one for a DTS-3 through hole variant. The footprints are on top of each other on the PCB, to minimise the space usage. </p><h2>LEDs!</h2><p><img src="https://cdn.hackaday.io/images/6737361434208482506.png"></p><p>Two LEDs here; one for power, and one connected to an IO line on the microcontroller. The power LED is directly connected to the VDD line on the microcontroller. The resistor values here are just placeholders, they'll need recalculating depending on the specific LEDs that are used, and the voltage the board is powered at (usually 3.3V, though 1.8V would be common too).</p><h2>RTC Crystal</h2><p><img src="https://cdn.hackaday.io/images/3252661434209552754.png"></p><p>The usual oscillator pins do not have a footprint for the crystal, as the SAM D20/D21 range has a good onboard oscillator that (through clock multiplying magic) can clock the microcontroller at 48MHz. However, there are separate pins for a 32.768kHz oscillator, and I quite like the idea of RTC capability; therefore, there is a lovely footprint for that on the PCB instead. The capacitors don't have a given value, as it will vary dependent on the specific oscillator chosen. </p><h2>USB</h2><p><img src="https://cdn.hackaday.io/images/4760421434210213839.png"></p><p>The SAM D21 has full speed USB 2.0, both device and host! To use this capability, the board has a micro USB connector. The D+ and D- (Data lines) have a TVS (transient voltage suppressor) on it for ESD protection. This is an extra measure, and the diodes inside this package are all in parallel with the signal, so this can be left unpopulated and the USB will still work. The shielding on the USB acts as a large antenna, so R2 and C2 act as a filter before it's connected to ground. The values for this come straight from the SAM D21 datasheet, but these components can either be omitted, or bridged with a 0R resistor if you would like the shield grounded. R3 on the USB ID line is used to limit the current through the 5V line to the 3.3V tolerant pins; again the value for this is straight from the SAM D21 datasheet. </p><p>Whilst the SAM D20 doesn't have USB, that doesn't mean it wouldn't benefit from a USB connector! USB...</p>
