                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">A small Arduino based anti-theft system with motion-sensing, SMS alerts, GPS tracking and RFID keys for use on Bicycles, eBikes, motorcycles, RVs, etc. Designed to be build-able for under $100 and small enough to conceal inside a bike seat, pannier or battery box.<br /><br />The design goal was to make it super-simple to use- To arm, remove the PICC key card from it's pocket. To disarm, replace the PICC in it's pocket. If it senses motion when armed, it sends you a text message, including GPS.<br /><br />It uses the Arduino Pro Mini, the Adafruit FONA 808 GSM/GPS board, the MPU6050 accelerometer/gyro and the MFRC522 SPI Mifare RFID reader/writer.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><h2>[<strong>Update: 2015/08/15] </strong>- </h2><h2>I now consider this project complete and highly successful. I&apos;ve been using it IRL for 2 months and its great!  I&apos;ll likely print out a case for it (and add a project log of course) otherwise it&apos;s done. <br></h2><h2>Note<br></h2><p>This project is not for beginners. It assumes some familiarity with Arduino and the Arduino IDE. Intermediate soldering skills are also required. Also we&apos;re pushing the limits of the Atmega328P RAM. So the we use a separate sketch to provision keys. (Note- I originally wrote it as one app and had the key provisioning data in EEPROM. But it was WAY too big to compile once the MPU and GPS parsing code was added. )</p><p>While this project is ongoing, its also quite stable. My biggest remaining challenges are to see-</p><ul><li> how small can I make the finished hardware?</li><li>how many features can I fit into 32k RAM?</li><li>How low can I get the power consumption? </li></ul><p>The current version uses the Arduino Pro Mini at 3.3V which eliminates the need for the boosting regulator that was in the original version. (The first prototype was built from parts I had lying around.) The alarm sits on the FONA like a shield and is about as small as I can make without busting out Kicad . If you wish to follow along and build one yourself, check out the &apos;Directions&apos; section below.</p><h2></h2><h2>Prerequisites<br></h2><p>In addition to the <a href="https://www.arduino.cc/en/Main/Software">Arduino IDE</a>, this sketches use the following Arduino Libraries and will not compile without them-</p><ul><li>I2CDevLib: <a href="https://github.com/jrowberg/i2cdevlib">https://github.com/jrowberg/i2cdevlib</a></li><li>AdaFruit FONA Library: <a href="https://github.com/adafruit/Adafruit_FONA_Library">https://github.com/adafruit/Adafruit_FONA_Library</a></li><li>RunningAverage Library(included in libraries dir): <a href="http://playground.arduino.cc/Main/RunningAverage">http://playground.arduino.cc/Main/RunningAverage</a></li></ul><h2><a id="user-content-software" class="anchor" href="https://github.com/alphacharlie/nano-bike-alarm/blob/master/README.md#software"></a>Software</h2><p>This project consists of 2 sketches- <strong>Bike-Alarm</strong> and <strong>newKey</strong>. </p><ul><li><strong>Bike-Alarm</strong> is (as you might imagine) the alarm sketch. </li><li><span></span><strong>newKey</strong> provisions a new PICC card. In order for everything to work, the <em>RFID encryption keys</em>, <em>sector number</em> and <em>token data</em> MUST match in both sketches. It can also be used to unprovision or test PICCs.</li></ul><p>     (BTW - <strong><em>Please </em>don&apos;t use the default key or token!</strong> It&apos;ll work. But it&apos;s like making your password &apos;password&apos;.)</p><h2><img src="https://cdn.hackaday.io/images/3093961434865889772.jpg"></h2><h2><a id="user-content-usage" class="anchor" href="https://github.com/alphacharlie/nano-bike-alarm/blob/master/README.md#usage"></a>Usage</h2><p>Once the system is installed and configured, to use you simply place the provisioned PICC (card) in it&apos;s &apos;pocket&apos; within range of the RFID reader. Once the reader authenticates the PICC, the alarm emit beep <strong>three short beeps</strong>, indicating it&apos;s disarmed. Remove the PICC and the alarm arms, emitting<strong> one long beep</strong>.</p><p>When armed, the Arduino turns on the GPS, then scans the output from the MPU6050. If motion is sensed an SMS with the GPS coordinated, heading and speed of the bicycle are sent to the alert phone. SMS are sent by default every 4 minutes while motion continues or every 10 minutes until the alarm is disarmed.  (This behavior can be altered in the config.) The alarm is silent and does <em>not</em> beep when triggered. However, you <em>could also </em> change that in the sketch, should you so desire... (or perhaps even take some other action, like have it voice dial your mobile and yell at the thief via speaker phone? ;-)</p><h2><a id="user-content-troubleshooting" class="anchor" href="https://github.com/alphacharlie/nano-bike-alarm/blob/master/README.md#troubleshooting"></a>Troubleshooting</h2><p>If you run into problems there is quite a bit of disabled debugging code in the sketch that can be used with the serial monitor to figure things out. But even enabling the DEBUG #ifdefs is unlikely to work because the sketch uses almost all of the Atmega328P&apos;s memory <em>without any debugging enabled</em>. So you&apos;ll probably need to enable the debug messages you need &apos;a la carte&apos; by commenting out the <strong>#ifdef </strong>and <strong>#endif </strong>on either side of them. Most errors can be deduced by the beep. If you run into issues you can post your question here on the project page and I&apos;ll try to assist.</p><p>To prevent issues always power up with the PICC next to the reader. If the unit beeps continuously on power up there was an error initializing the hardware. </p><p>If the unit beeps continuously when arming then there is no GSM network available to the FONA - Check your antenna and SIM.</p><p>Bugs can be reported to info@boffinry.org.</p><p>This is a work...</p>

It sucks to find out you've got a bad one after all the soldering is complete... </em><br></p><p>The hardware is fairly easy to assemble here. Each component uses a different bus and different pins. The only pins with multiple connections are the <strong>Vcc </strong>and <strong>Gnd </strong>pins. This makes it easy to wire point-to-point for the smallest device footprint. See the wiring diagram below for details.</p><p><img src="https://cdn.hackaday.io/images/9694681434838717679.jpg"></p><p>The circuit above shows the original 5V prototype. The newest version uses the Arduino Pro Mini at 3.3V and does not have the Boosting 5V regulator. Instead the <strong>Raw </strong>pin of the Arduino Mini is connected to the <strong>Bat </strong>pin of the FONA 808. Otherwise the wiring is pretty much the same.</p><p><strong>            Original prototype (left) next to the <em>much </em>smaller v2 layout(right)</strong></p><p><img src="https://cdn.hackaday.io/images/3983061435348970304.jpg"></p><p>To construct the Nano Bike Alarm First you'll want to get the MPU6050 mount set up. To do that, solder 2 male header pins to the A4 and A5 on the Arduino.</p><p><img src="https://cdn.hackaday.io/images/8167731435349187542.jpg"></p><p>Now solder some short jumper wires to the VCC, Gnd and Int pins on the MPU6050.</p><p><img src="https://cdn.hackaday.io/images/2844971435349252435.jpg"></p><p> Now Solder the wire from the <strong>Int  </strong>pin to pin <strong>D2 </strong>on the Arduino (leave yourself a little extra wire for slack.) After that take 2 lengths of pin header and solder it to all the other pins on the top row of the Arduino (i.e. everything <em>except </em><strong>D2</strong>). Now you can lay the MPU6050 upside-down on the Arduino so that the SDA and SCL pins on the MPU connect to <strong>A4 </strong>and <strong>A5. </strong>(Not as confusing as it sounds, see photo below). After that you can connect the Gnd and Vcc jumpers (don't solder the Vcc jumper yet though!)</p><p><img src="https://cdn.hackaday.io/images/4384631435349759406.jpg"></p><p>Now take another short jumper wire and connect it to the bottom of the <strong>Vcc </strong>pin on the Arduino and solder them <em>both, </em>then solder the Arduino to the proto-board as shown in the photo below.</p><p><img src="https://cdn.hackaday.io/images/6560511435349920971.jpg"></p><p>Next solder some female header on to the other half of the proto-board for the FONA. and connect jumper wires so that-</p><ul><li>Bat on the FONA goes to Raw on the Arduino</li><li>Gnd on the FONA goes to Gnd on the Arduino</li><li>Vio on the FONA goes to Vcc on the Arduino</li><li>Rx on the FONA goes to pin D5 on the Arduino</li><li>Tx on the FONA goes to pin D6 on the Arduino</li><li>RST on the FONA goes to pin D7 on the arduino</li><li>Key on the FONA goes to Gnd on the Arduino</li></ul><p>Next solder the beeper to the proto-board and connect the + pin to D8 on the Arduino and the other pin to Gnd (using jumpers). You can also use a bit of Kaptan tape to insulate the different layers from each other. </p><p><img src="https://cdn.hackaday.io/images/9760571435350343489.jpg"></p><p>Next solder the RFID to the alarm so that- </p><ul><li>3.3V on the RFID goes to Vcc on the Arduino</li><li>Gnd goes to Gnd</li><li>Rst on the RFID goes to pin D9 on the Arduino</li><li>SDA on the RFID goes to Pin D10 on the Arduino</li><li>MOSI on the RFID goes to pin D11 on the Arduino</li><li>MISO on the RFID goes to pin D12 on the Arduino</li><li>SCK on the RFID goes to pin D13 on the Arduino.</li></ul><p><strong>Congratulations! You are done building the hardware!</strong></p><p><strong><br></strong></p><p>                              The photo below shows the completed Nano Bike Alarm.<img src="https://cdn.hackaday.io/images/2173131435350763312.jpg"></p><h2><a id="user-content-install" class="anchor" href="https://github.com/alphacharlie/nano-bike-alarm/blob/master/README.md#install"></a> <span></span></h2></div>

                                    <li>
                                        <span class="instruction-number">3</span>
                                        <div class="instruction-list-item"><h2>Vehicle Mounting</h2><p>Once it's set up, the last thing to do is install it on your bike. It can be hidden in the seat or for eBikes inside the battery box. (If your battery box is metal, you may need to futz with antenna and RFID reader placement to ensure you get a GPRS signal and GPS fix.)</p><p>Wherever you install it, there should be a pocket or sleeve to hold the RFID card which is positioned near and parallel to the MRFC522 card reader. For aluminum battery boxes, you'll also have to think about antenna and RFID reader placement. Test your set-up regardless to <em>verify GSM and GPS reception</em>. If the location you install the alarm is not weatherproof, then you may want a small plastic case to enclose it. (Whereas if it's sewn into the seat of a bicycle then size will be paramount and you may just want to heat-shrink wrap the components. The vehicle will dictate the construction methods.)</p><p>If you are installing into a motorcycle/ATV/snow machine, then you can use a 12VDC-&gt;5VDC converter to and have the vehicle recharge the alarm's battery. To do this simply connect the <strong>5VDC</strong> out from the converter to the <strong>+5V</strong> (and <strong>Gnd</strong>) pin of the FONA. </p><p>Another option if you're going to connect it to the vehicle's battery is to <em>not use a battery </em><strong><em>at all</em></strong> for the FONA. The plus is that it's a simpler system. The minus is that if the thief disconnects the vehicle battery then the GPS quits phoning home. If you wish to power the FONA directly from the vehicle battery you need to use an <strong>adjustable</strong> step-down regulator and set the output voltage to <strong>3.9VDC </strong>(instead of 5)<strong> </strong>then connect this output to the <strong>+Bat</strong> and <strong>Gnd</strong> of the FONA (or to the JST-PH). Do <em>not </em>connect the +5V of the FONA. (to anything)<br></p><p>For eBike, connecting the Nano-Alarm to the bike's battery takes more planning. DC Adapters for vehicle mounting like the one below are available in 12VDC, 36VDC and 48VDC and offer an easy way to recharge the alarm. Output connections are the same as for other vehicles. <strong>But make sure that the converter is designed to handle the <em>actual </em>voltage of your system, not the 'on-paper' voltage of the battery.</strong> As an example, a 36V LiFePO4 battery has 12 cells with a fully charged voltage of 3.65V per cell. So our 36V battery is actually 43.8V when fully charged. A 48V LiFePO4 tops off at 58.4V! </p><p> Also <strong>please test your DC-&gt;DC converter output</strong> to ensure that it is <em>only delivering the desired voltage</em> <strong><em>before</em> </strong>plugging your alarm into it!**</p><p><strong></strong><br></p><p><img src="https://cdn.hackaday.io/images/9082651434864463254.png"></p><p>If you need to reduce the input voltage for your DC-&gt;DC converter just a few volts,  you can connect some diodes in series with the DC-&gt;DC power supply. Below I used 1N5408 diodes which are rated for 3A and drop the voltage about a volt each. This gives the LM2596 board shown an extra 5V of headroom...<br></p><p><img src="https://cdn.hackaday.io/images/31571435791008509.jpg"></p><p>You can use a larger/smaller battery than 2500mAh as well. Battery usage while <strong>armed </strong>is around 50-100mA <em>assuming you have a decent GSM signal</em>. (that's with the GPS on too!) So you can estimate how long a battery will last by dividing the battery capacity by 60. So for example a 5000mAh <em>could </em>stay <em>armed </em>for as long as 100 hours.)  If it's eating a lot of battery, getting a better GPS and/or GSM antenna can help extend battery life dramatically simply by allowing the FONA to work <em>less </em>to amplify the radio signals it receives and transmits. </p><p><strong>**Note</strong> - a <em>large </em>number of the switching power supplies on fleaBay include counterfeit ICs. For <em>any</em> power supply you get online - Test it on your actual working voltage for <strong>several hours</strong> at least and ensure that it delivers only the stated voltage and does not get hot, smoke die, etc... Bad one's tend to short out within a few minutes, delivering the full input voltage to the 5V output. If that's going to happen then you want it to be <em>before </em>you plug your <strong>Nano-Alarm</strong> into it. (You can use a 12V incandescent light bulb or a sufficiently large resistor as a test load.)</p></div>
                                    </li>
                                
                            </ul>
