                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">Mechanical solutions usually need some sort of motor to actuate stuff, with the ever more democratized world of 3D printers custom motors fitting a specific needs could be made on a whim, but no such calculator exists to relieve the engineer from the laborious task of calculating all the properties of such a motor.<br /><br />This project aims to offer not only web-based calculator for a 3D printed parametric motor, but also generate the required files so that it could be printed immediately.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>Imagine if you will</p><ul><li><span></span>You want to motorise your old telescope, you have a 3D printer so form factor is not an issue, ideally the motors could be integrated to the parts attaching themselves to the telescope, you know the power and steps/resolution you need, but not how to make the motors. This calculator will evaluate the required parameters based on available data and also show how to make the motor without having to think about the air permittivity, coil capacitance, motor inductance...</li><li>You want to build a personal <a href="http://www.wikiwand.com/en/Wind_turbine">wind turbine</a> as a project to power some electronics in the middle of nowhere where solar energy is not an option, you can either integrate the first motor you find or you can print an <a href="http://www.wikiwand.com/en/Outrunner">outrunner motor</a> having the blades on the rotor, you enter the power and size requirements and get printable files that come close to what you expect and need.</li></ul><p>All parameters shall be left free or have a default realistic value, the different default parts shall be commercially available, for example: motor coil diameter, magnet parameters...</p><p>Since brushed motors are kind of annoying to make the calculator will first focus on brushless types, synchronous (with magnets on the rotor) and asynchronous (without any magnets).</p>
                                
                            </p>
                        </div>

                        <li>
                            <h2><a href="/project/4593/log/20423-nema-17-standard">NEMA-17 standard</a></h2>
                            <small class="authorship">a month ago  &bull;
                            <a class="gray-link" href="/project/4593/log/20423#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>I think the project was a bit too ambitious at first. Allowing all the aspects of the motor to be modified gives a huge amount of freedom, but then building a tool that evaluates performance and generates 3D files for that becomes really complicated...</p><p>An intermediate step is to restrict the motor form and allow only some parameters such as motor length, coil diameter and magnet strength to be set by the user. Since the motor form is fixed I decided to adopt a standard: NEMA-17 stepper motor.</p><p>NEMA-17 is a very popular motor used on lots of 3D printers, maybe it can be used to actuate an actual 3D printer... probably not, but who knows.</p><p><img src="https://cdn.hackaday.io/images/3085211436006228222.png"></p><p><img src="https://cdn.hackaday.io/images/913481436006396431.png"></p><p>The fixation holes are exactly at the same place than the NEMA-17, all parts should be 3D printable (except for magnets, coil, axis, M3 screws and bearings).<br></p><p><img src="https://cdn.hackaday.io/images/9221121436006382973.png"></p>I don't know how it will perform though, or if it will melt... No math has been done yet or anything printed at this point.
                                
                            </p>
                        </li>
                    
                        <li>
                            <h2><a href="/project/4593/log/19796-rethinking-the-mechanical-part">Rethinking the mechanical part</a></h2>
                            <small class="authorship">2 months ago  &bull;
                            <a class="gray-link" href="/project/4593/log/19796#discussion-list">
                                2 comments
                            </a>
                            </small>
                            <p><p>I spent some time thinking how to make the building process as easy as possible, after all the motor needs to be printed, colied, assembled, mounted and controlled. The first concept wasn't very user friendly in that sense.</p><p><img src="https://cdn.hackaday.io/images/189851434859135152.png">Fig. 1: <span class="redactor-invisible-space">New concept. Looks like a standard stepper motor.</span><br></p><img src="https://cdn.hackaday.io/images/1124791434859036050.png">Fig. 2: Exploded view of all the parts.<img src="https://cdn.hackaday.io/images/3240331434859048836.png">Fig. 3: Rotor.<p>This second concept is all about using off-the-shelf parts. It's controlled like a 2-phase stepper motor when the rotor has magnets on it (synchronous) and when coils are put on the rotor it works as an asynchronous motor.</p><p>This makes it easier to control, stepper controllers are really cheap these days and I happen to have some at hand so I won't have to build custom electronics.</p><p>For the rotor it's probably better to use something sturdier than a printed axle, a metal rod will do. I decided to use the tiny 635 type ball bearings on either side of the rotor as well. The motor can still be made with only printed parts though.</p><p>This is still a concept, I haven't printed anything yet because I don't have access to my printer currently.</p>
                                
                            </p>
                        </li>
                    
                        <li>
                            <h2><a href="/project/4593/log/17592-motor-controller-thoughts">Motor controller thoughts</a></h2>
                            <small class="authorship">3 months ago  &bull;
                            <a class="gray-link" href="/project/4593/log/17592#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>Having a motor is nice and all, but for brushless type motors a controller is needed to make them turn. It doesn't have to be something very fancy, an <a href="http://store.hackaday.com/products/teensy-lc">Teensy</a> and some <a href="http://www.wikiwand.com/en/MOSFET">MOSFETs</a> could easily do the work and then some.</p><p>Ideally no sensors would be needed as they can break, are noisy and a hassle to mount and use (compared to doing nothing).</p><p>In the case of a synchronous motor back-EMF black magic would be possible to turn on the right couple of transistors on the H-bridge (like it's done with hard-drives). The asynchronous motor doesn't need sensors (another advantage for this motor type) as the stator magnetic field has to be non-synchronous relative to the rotor anyway, but the H-bridge transistor commutation still needs to happen...</p>
                                
                            </p>
                        </li>
                    
                    </ul>
