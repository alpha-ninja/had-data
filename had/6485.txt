                    <h5>Description</h5>
                    <div class="description">This project uses old floppy drives, hard drives, and my 3D printer and uses them to do something they were never designed to do: play in one of the weirdest ochestras ever made.<br /><br />While there's currently only 3 types of instruments, the system supports pretty much anything with steppers, or anything that makes a noise on a signal pulse as percussion. An old doorbell, noizy fans, etc. can be used. machines that accept G-code through the standard Serial protocol can be used as well.<br /><br />Who really uses floppy drives anymore, or old mechanical doorbells, or exceptionally loud 10GB IDE hard drives? This is a great way to recycle them. Only the hard drives are permanently damaged; the floppy drives and 3D printer still do their respective non-musical jobs without any modification.<br /><br />This is both a hardware and software project, as the instruments are the stars, but the control software is what makes it all happen and keeps things synchronized.</div>

                            <h2><a href="/project/6485/log/23436-possible-new-instrument-doorbell">Possible new instrument: Doorbell?</a></h2>
                            <small class="authorship">10 hours ago  &bull;
                            <a class="gray-link" href="/project/6485/log/23436#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>I finally found that old doorbell I wanted to perhaps use. It&apos;s a really old fashioned one with a solenoid hitting metal plates for the sound.</p><p><img src="http://i.imgur.com/9cRWiDh.jpg"></p><p>It&apos;s design is simple yet clever. A little double sided hammer sits in the middle with a spring and solenoid around it. When the doorbell is pressed, the solenoid activates and slams it into the shorter metal plate, making a &quot;ding&quot; sound. When the button is released, the solenoid deactivates and the now compressed spring swings it into the longer metal plate on the other side, creating a &quot;dong&quot; sound.</p><p><img src="http://i.imgur.com/QWOf5GJ.jpg"></p><p>An interesting solution to put the delay between the two notes in a &quot;ding-dong!&quot; sound. Now, how to exploit that...</p><p>I could, perhaps, try using PWM to put the hammer in a position where it could slam either of the metal plates just as hard. changing the duty cycle to 100% would play a ding, and putting it to 0% plays a dong. After 50ms or so it would return to whatever PWM value results to it sitting in equilibrium. I&apos;m not sure how well this plan might work, but it&apos;s worth a try. It&apos;s not that far off from how drums currently work, which is a 50ms long LOW-HIGH-LOW pulse.<br></p><p>Getting it to be useful in a song is another problem. It isn&apos;t really percussion, and it can only play 2 notes. I could set it up so whenever those notes play in a certain channel, it would play the note instead of a floppy drive, or alongside a floppy drive. For some reason, I&apos;d imagine the classic Super Mario theme wouldn&apos;t sound too bad if some of the notes were played by a doorbell. I&apos;m not musically talented enough to tell you what notes in the song match, and therefore would be replaced by the doorbell&apos;s notes, but that&apos;s my control software&apos;s job, not mine :D</p>
                                
                            </p>
                        </li>

                            <h2><a href="/project/6485/log/23426-power-issues-solution-and-new-system-design">​Power issues solution and new system design</a></h2>
                            <small class="authorship">11 hours ago  &bull;
                            <a class="gray-link" href="/project/6485/log/23426#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>I&apos;ve finally decided to solve all my power supply problems. Currently, I have seperate power supplies for my 3D printer, my Raspberry Pi, my thermal printer, my various miscellaneous projects and now I need a power supply for my floppy drives. I plan to fix all this with one big old ATX power supply.</p><p>It&apos;s going to be hooked up to my all purpose Raspberry Pi server, which controls the standby/active state of the power supply with both a physical button and software commands. This power supply will also power my 3D printer, and my floppy and disk drives. I&apos;m sure it won&apos;t be a problem, seeing as floppy drives and HDDs are designed to be powered by these things.</p><p>I also plan to finally integrate something I&apos;ve wanted to do for a while now: put the Pi in charge of the orchestra. I&apos;m going to try and make a little server program that my GUI client would connect to, and send over a processed MIDI file for playing, since my Pi is also an Octoprint server. That way, I won&apos;t need to keep changing what my 3D printer is plugged into.</p><p><img src="http://i.imgur.com/Jo6JpDH.jpg"></p><p>I&apos;ve updated the main page to include a &quot;System Design&quot; reflecting how this would all work. I&apos;ll also include it here because those thumbnails are weird:</p><p><img src="http://i.imgur.com/WEIkSEu.png"></p>
