                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">I love launching model rockets, especially in the summer, but the lack of feedback on speed, altitude makes the launch less enjoyable. <br /><br />I am setting out to build a lightweight, battery powered telemetry module for my model rockets.  The goal of this project is to collect as much data from my rocket launches. My inspiration for this project comes from here: <a href='http://www.altduino.de/index.php?all' target="_blank">http://www.altduino.de/index.php?all</a> <br />This project set out to do what I wanted but the only problem is that the code is not open source and the developer does not want to share it. I want to create an open source solution, my code is already online from day one.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>Current Hardware: <br></p><ul><li>Arduino Nano atmega328p board </li><li>BMP108 - Baro Sesnor </li><li>3 Axis Acceleromiter </li><li>2.4GHz UART tx/rx </li></ul><br>I have implemented my own protocol similar to how GPS communicates via serial to ensure that data reliability and readability. Currently I can get the altitude, temperature and max altitude using the live downlink and I have tested the downlink in my M1 launch to a max altitude of 150m. I have also implemented a function that will save the highest recorded altitude value to the internal eeprom of the arduino.<p>My current code is downloadable from here: <a href="https://github.com/vcazan/AltiRocket" target="_blank">https://github.com/vcazan/AltiRocket</a></p>
                                
                            </p>

                            <h2><a href="/project/5686/log/19489-going-higher">Going higher</a></h2>
                            <small class="authorship">2 months ago  &bull;
                            <a class="gray-link" href="/project/5686/log/19489#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>It has been a few weeks since my last project log, so I thought I would share with you some updates to this project.</p><p>Since my initial launch, I launched another 3 rockets and gathered more data using my new protocol. This proved to be successful and has inspired me to create an interface for displaying this data after the launch. I will take my inspiration once again from the AltDuino project that does a wonderful job of displaying all data in one easy to read view:</p><p><img src="https://cdn.hackaday.io/images/594741434289564509.jpg" width="260" height="173" style="width: 260px; height: 173px;"></p><p>Another update is that I purchased a much larger rocket to go higher in altitude. I will try and maintain the weight of the sensor unit the same though to ensure maximum altitude can be achieved the first launch. </p><p><img src="https://cdn.hackaday.io/images/1190511434290735551.jpg" width="233" height="415" style="width: 233px; height: 415px;"></p><p>As for the electronics, I made a mistake when purchasing EEPROMs and bought the 2kb version instead of the 256kb version. These new ones have finally arrived and I will be saving all the data that has been transmitted to the external EEPROM as well to ensure that if data loss occurs during launch I can retrieve it after landing.</p><p>Another feature I am debating is an auto launch command that would ignite the fuse with a serial command sequence. I also wanted to have a way to verify ignition has occur ed so I purchased a tiny mic that will measure sound. It is very loud when the motor launches so I will be able to "confirm" launch with that sensor and my baro. This may or may  not be legal so I will double check to make sure. </p><p><img src="https://cdn.hackaday.io/images/4409981434291272330.jpg" style="width: 347px; height: 196px;" width="347" height="196"></p><p>There would be a ground based launch station with a 12v boost that would take a signal from the arduino to launch, but I will need a way to unteather at least two wires from the rocket and launch station when launch occurs, so I am still working out this. </p><p>I plan on launching the larger rocket within the next week or so and I will post pictures and data when I do. </p>
                                
                            </p>
