                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">IVPORT which is the first Raspberry Pi (also Raspberry Pi A,A+,B+ and Raspberry Pi 2 fully compatible) Camera Module multiplexer is designed to make possible connecting more than one camera module to single CSI camera port on Raspberry Pi. It has 1 CSI connectors for the input and 4 CSI connectors for outputs to the camera modules with using flex cables. Also it has stacking headers for the GPIO connection and self stacking up to 4 Ivport.<br /> Multiplexing can be controlled by 3 pins for 4 camera modules, 5 pins for 8 camera modules and 9 pins for maximum up to 16 camera modules with using GPIO.<br /><br />Updated Notes<br /><br />The closest approach to simultaneous capturing at 640x480 resolution with 7 fps each camera connected to Ivport Camera Multiplexer (4 camera boards were connected.) by utilizing Raspberry Pi's video-capture capabilities with a JPEG encoder. See more details here.<br /><br />More information at <a href='https://github.com/ivmech/ivport' target="_blank">https://github.com/ivmech/ivport</a><br /><br /></div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p><img src="https://cdn.hackaday.io/images/1299921408785974337.png"></p><p>There are some different videos while switching view source between <strong>4 Different raspberry Pi Camera Modules</strong> with using <strong>IVPort</strong> module. Bottom right corner video shows switching camera input while capturing video.</p><p><iframe allowfullscreen="" src="//www.youtube.com/embed/rhP1r1i4D7o" frameborder="0" height="281" width="500"></iframe></p><iframe allowfullscreen="" src="//www.youtube.com/embed/vbtqg0oY-E4" frameborder="0" height="281" width="500"></iframe><p><iframe allowfullscreen="" src="//www.youtube.com/embed/IpjJE06ffgU" frameborder="0" height="281" width="500"></iframe></p><p><strong>There are 4 cameras which are connected to Raspberry Pi via IVPort module.</strong></p><p><img src="https://cdn.hackaday.io/images/4410751408785643140.png"></p><p>Multiplex the number of usable camera module on Raspberry Pi from1 to 4 with using one Ivport. It can multiplex <strong>from 1 to 16 </strong>withusing <strong>4 Ivport with stack.</strong></p><p>Multiplexing can be controlled by <strong>3pins for 4 camera modules</strong>, <strong>5 pins for 8 camera </strong>modules and <strong>9 pins for16 camera modules </strong>with using <strong>GPIO</strong> and external script library thatwritten by Python.</p><p><img src="https://cdn.hackaday.io/images/6933501408785666631.png"></p><p><strong>Connectors: </strong>15 Pin FFC CSI<br><strong>Pinout: </strong>RaspberryPi CSI Connector Camera Pinouts<br><strong>Protocol: </strong>MIPI</p><p><strong>Propagation Delay between input andoutput: </strong>0.5 ns</p><p><strong>Switching Time fromone channel to other channel:</strong>50ns (it can increase withdepending to GPIO delay)</p><p><img src="https://cdn.hackaday.io/images/8040641408785707490.png"></p><p>There are also risks that Raspberry Pi GPU is not open sourced therefore camera input willbe supposed as only one camera is connected. It will be hard toimplement more than one camera support into library.
