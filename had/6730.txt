                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">My professor really wanted a multispectral imager and I really needed a summer project. This turned out to be a really interesting challenge of system engineering, programming, and electrical engineering! I 3D printed the first prototype out of an obnoxious yellow plastic. Right now we're working on upgrading the system to fit a battery system and better enclose the electronics.<br /><br />The applications for this are numerous! Because it's a modular system we can hot swap a lot of components. For example. We're utilizing the camera as a multispectral imager. But since the unit functions as a synthetic aperture, we can also remove the filters and conduct three-dimensional imaging. The thermal camera gives us the opportunity to track hot objects and determine the distance of the hot object. Alternatively, we could apply the same filters to all of the cameras and only look at one spectra.<br /><br /> Thanks to SDSM&T, UB, the NSF and Hoffert fund for support. </div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>The imager utilizes five raspberry pi computers, a TP-Link switch, a FLIR Lepton, raspberry pi noir cameras, a zippy 8000mAh battery, and some custom circuitry to provide power to the unit. We had only two iterations of the mechanical design. All of this can be 3D printed and assembled using 6-32 bolts and a screwdriver. Some other hardware also includes nylon spacers and all-thread to secure the stack of pi&apos;s together.</p>
                                
                            </p>

                        <li>
                            <h2><a href="/project/6730/log/22090-making-of-the-code-a-journey-with-tcp">Making of the Code: A Journey with TCP</a></h2>
                            <small class="authorship">14 days ago  &bull;
                            <a class="gray-link" href="/project/6730/log/22090#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>Welp, better start documenting some of the progress that happened with this camera. To be honest, I didn&apos;t think of entering this into the hackaday prize until after we had finished much of the development. </p><p>The entirety of the (non-gui) code was written in Python. I chose python for two reasons: 1) There&apos;s an addon for everything 2) I&apos;ll spend less time debugging syntax errors. </p><p>On top of the many advantages of an easy language, what I wanted to do was a perfect fit for a scripting language. Eventually I hope to have the code ported into C or C++, however if that doesn&apos;t happen, the machine still works, and has contingencies when it doesn&apos;t work. Not optimal, but it produces the stills that are necessary for research. </p><p>Now, let&apos;s chat about TCP.</p><p style="margin-left: 20px;">Why: Guaranteed, corruption-free delivery</p><p style="margin-left: 20px;">Implementation: Python has a nifty way to create sockets and establish 2 way communication over a network.</p><p style="margin-left: 20px;">Wired Ethernet communication was also going to be the method which files were transferred, so this helped things. </p><p>Program flow on slave:</p><p style="margin-left: 20px;">Keep socket open for connection</p><p style="margin-left: 20px;">Read packet (this allows us to reboot, shut down, or take photo)</p><p style="margin-left: 20px;">Take photo (this was achieved using the os.system feature of python)</p><p style="margin-left: 20px;">Send photo over SCP</p><p style="margin-left: 20px;">Wait for another packet</p><p>Development of this was of course, fairly straightforward. In order to allow for SCP to occur, I shared the fingerprints of all of the slaves with the master. This allowed for password-less interactions between the devices. </p><p>One of the more serious issues we&apos;ve been having was actually taking the pictures. Ideally, the cameras take several seconds to be activated, take a picture, and have a .jpg file produced. This had to be accounted for in the code in the form of a delay. This is not perfect, but it works.</p>
                                
                            </p>
