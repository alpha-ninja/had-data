                            <div style="position:relative">
                                <div id="post-body-4052"><h2>
<iframe allowfullscreen width="500" height="281" src="//www.youtube.com/embed/Y5uNffNiwK4" frameborder="0"></iframe></h2><h2>How is Andy a Open Source/Hardware?</h2><p>Andy is completely open source. Everything but his database is on his&nbsp;<a href="http://github.com/spadgenske/ANDY" target="_blank">repository</a>. As for hardware, the&nbsp;<a href="https://cdn.hackaday.io/images/8894961403706208344.png">schematic</a> is also open. &nbsp;When the new bipedal mechanism is semi-finished, the stl files will be on thingiverse.&nbsp;</p><h2>How is Andy a Connected Device?</h2><p>Andy is connected in many different ways. Andy connects with humans via their language, speech. He also talks with&nbsp;<a href="http://github.com/spadgenske/ANDY-client" target="_blank">client programs</a> over the internet and will soon be able to communicate with Mindstorms NXT robots over bluetooth. </p><h2>Is Andy&nbsp;producible&nbsp;and could the work be extended for&nbsp;other uses?</h2><p>When Andy is complete, he will be able to do many different tasks, from bringing you stuff or just answering questions. One robot can't do everything, so Andy will be able to communicate with other robots that can.&nbsp;</p><h2>Is there an intuitive interface?&nbsp;</h2><p>Speech is Andy's UI, just like with humans. Since Andy is a computer, he can also interface with Computers over the internet.</p><h2>Is it/could it be manufacturable?</h2><p>At the moment, no. Since Andy uses a Robosapien for the Bipedal movement, this would not work. But, I am designing a new bipedal system in SolidWorks. The software is designed so all I would have to change is the Movement() class and functions to get Andy to adapt to his new body.</p><h2>What's Not Done?</h2><p>As you can tell in the demo video, Andy cannot move because I burnt out his motors before I filmed. Instead of purchasing new motors (Or a new Robosapien) I began work on a new bipedal mechanism. The legs may use&nbsp;<a href="http://www.ebay.com/itm/4-DOF-Biped-Robot-Mechanical-Leg-Robot-Servo-Motor-Bracket-NO-Servo-Motor-new-/390867789648?pt=LH_DefaultDomain_0&amp;hash=item5b01892750" target="_blank">this,</a> and I will design a torso, head, and arms in SolidWorks for 3D printing. </p><p><strong>Note: The paragraph above is just a prototype, not final design.</strong></p></div>

                            <h2><a href="/project/1205/log/4466-how-the-software-works">How the Software Works</a></h2>
                            <small class="authorship">a year ago  &bull;
                                <a class="gray-link" href="/project/1205/log/4466#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-4466"><p>Here is a quick overview of how the current software works from startup to shutdown. All the code is open source, so you can browse through it&nbsp;<a href="https://github.com/spadgenske/ANDY" target="_blank">here</a>. Below is a basic flowchart.</p><p><img src="https://cdn.hackaday.io/images/6209691404502939342.png"></p><h2>Step One: Startup Andy</h2><p>To start Andy, flip the switch on the back of him. This allows current to flow from a&nbsp;<a href="http://www.ebay.com/itm/Parkzone-PKZ1033-11-1V-1300mAh-LiPo-Battery-for-Hobbyzone-Super-Cub-DSM-RTF-/261360367135?pt=Radio_Control_Parts_Accessories&amp;hash=item3cda4ab61f" target="_blank">11.1v 1.3 amp lipo battery</a> to the L298N motor drivers and to the Raspberry Pi. Once the Pi is booted up, it automatically runs&nbsp;<a href="https://github.com/spadgenske/ANDY/blob/master/boot/boot.sh" target="_blank">boot.sh</a>, a file that starts Andy's software. Next, Andy checks two&nbsp;<a href="https://github.com/spadgenske/ANDY/tree/master/configure" target="_blank">configuration files</a>. (Note: Not all features in those files are implemented) In the files, it checks whether to start the server, (Check for client program logins) and run some modes. (Modes are a future feature). Finally, Andy sets the silence for the microphone.&nbsp;</p><h2>Step Two: Get Command</h2><p>Now that Andy is up and running, he becomes a state machine. He will listen for a command from both his&nbsp;<a href="https://github.com/spadgenske/ANDY/blob/master/src/mic.py" target="_blank">mic</a> and the<a href="https://github.com/spadgenske/ANDY/blob/master/src/server.py" target="_blank"> server</a> (If started). Once the command is received and converted to text, the command can be executed.</p><h2>Step Three: Execute Command&nbsp;</h2><p>Next, Andy will run the correct function from&nbsp;<a href="https://github.com/spadgenske/ANDY/blob/master/src/cmds.py" target="_blank">cmds.py</a> that corrisponds with the command. If the command is the shutdown command, the software and hardware shutdown and the program ends. But, if it is any other command, the command is executed and the program goes back to step two.</p><h1>Conclusion&nbsp;</h1><p>There are many other aspects to Andy's software, over 1600 lines of code. Hopefully this log will give you an understanding of the basics. Also, if you haven't, watch a&nbsp;<a href="http://youtu.be/Y5uNffNiwK4" target="_blank">Demo video of Prototype #1.</a></p><h2></h2></div>
                            </div>

                            <small class="authorship">a year ago  &bull;
                                <a class="gray-link" href="/project/1205/log/4620#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-4620"><p>Below is a log about the hardware used in Andy's first prototype.&nbsp;</p><p><img src="https://cdn.hackaday.io/images/1126391404762523296.png"></p><h2>The Brain</h2><p>Andy has a Raspberry Pi for a brain. It controls absolutely everything from motor control to speech recognition. The Raspberry Pi runs&nbsp;<a href="http://github.com/spadgenske/ANDY" target="_blank">Andy's software</a>, which is explained&nbsp;<a href="http://hackaday.io/project/1205/log/4466-how-the-software-works">here</a>.</p><h2>Power</h2><p>Andy is powered using a 11.1v 1300mah lithium polymer rechargeable battery pictured below. It is regulated for the Pi with a&nbsp;<a href="http://hackaday.io/project/1205/log/4466-how-the-software-works">UBEC</a>.</p><p><img src="https://cdn.hackaday.io/images/2261421404761779737.JPG"></p><h2>Motor Driving</h2><p>The motors in Robosapien are driven with three L298N motor drivers pictured below with a blown up Raspberry Pi. Each driver can control two motors, so Andy can drive a total of 6 motors.</p><p><img src="https://cdn.hackaday.io/images/1681671404761958559.JPG"></p><h2>GPIO Expansion&nbsp;</h2><p>Since the Raspberry Pi doesn't have enough GPIO pins, I use a MCP23017 chip to add more. Right now all that I hook it up to is a LED.</p><p><img src="https://cdn.hackaday.io/images/4430231404761152417.png"></p><h2>Conclusion&nbsp;</h2><p>Lots of features will be added or removed when Andy migrates to 2.0. Hopefully, a version two prototype will be complete before August 4th. </p></div>

                            <h2><a href="/project/1205/log/5408-schematic-and-features-for-version-2">Schematic and Features for Version 2</a></h2>
                            <small class="authorship">a year ago  &bull;
                                <a class="gray-link" href="/project/1205/log/5408#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-5408"><p>I am starting to work on Andy's new hardware. His new body will have 10 degrees of freedom, unlike the old Robosapiens 6. While I wait for orders of parts to ship, I began drawing and testing a new circuit with many new features.</p><p><a href="https://cdn.hackaday.io/images/9767121406150818958.png" target="_blank"><img src="https://cdn.hackaday.io/images/9767121406150818958.png" alt="Click on Image for Better View"></a></p><p>New circuit features include:</p><ul>
<li>Battery Voltage Monitoring &nbsp;</li><li>Power on LED</li><li>5 Segment LED output</li><li>10 servos (4 for the legs, 6 for the arms)</li></ul><h2>Battery Voltage Monitoring</h2><p>With this new feature, if Andy is left on too long, instead of powering off because of under voltage and potentially corrupting the SD card and then finally destroying the Battery, Andy has a built in circuit to monitor it's own battery voltage and then power itself down if voltage is low. Also, the battery level can be displayed in a&nbsp;<a href="https://www.sparkfun.com/products/9935" target="_blank">10 segment LED bar graph. &nbsp;</a></p><h2>Power On LED</h2><p>On the new Andy, the Raspberry Pi will not be visible through a clear case. So, Andy will have a green LED connected to 3v3 that turns on when the Raspberry Pi turns on.</p><h2>5 Segment LED Output</h2><p>Although I haven't found a five segment version yet, (They are all ten) A LED bar graph will display battery voltage, current decibel level, and blink for errors and other useful output that Andy can not show now.&nbsp;</p><h2>10 Servo Motors</h2><p>10 servos means 10 degrees of freedom! Andy will have 4D0F in his legs instead of 2, and 6 DOF in his arms instead of 4. (3 servos per arm).</p><h1>Conclusion&nbsp;</h1><p>Hopefully this August I will have the parts for Andy's legs so I can build the biped. I also hope to film a demo of him walking and turning. Sometime this fall he should have the other features above, and Maybe this winter he will have arms. But, I cannot predict the future so maybe he will be done in September, or maybe I will wreck a bunch of stuff and not finish him till this winter. :( If you have any ideas/features you would like me to add, feel free to comment.</p></div>

                            <h2><a href="/project/1205/log/13284-rebuilding-andy-again">Rebuilding Andy Again...</a></h2>
                            <small class="authorship">7 months ago  &bull;
                                <a class="gray-link" href="/project/1205/log/13284#discussion-list">
                                    2 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-13284"><p>Well,  I haven't been on Hackaday for awhile so I thought I would write about the progress I am having with Andy. I dismantled him again, because I wasn't liking the DIY look and the resistors I used in between the Raspberry Pi GPIO and the servo control line were messing things up more than they were helping. So, now Andy looks something like this:</p><p><img src="https://cdn.hackaday.io/images/4797811422052613979.JPG"></p><p>The MG996R Servos used to make Andy walk were drawing more current than the 1K resistors could handle. So, I removed them. Then, I made Andy walk faster, and then it happened.</p><p>Andy walked right off the table, and the perfboard cracked. The metal legs and electronics survived, but instead of re soldering everything to a new perfboard, I decided to make a Andy 3.0. But, I stalled and built a <a href="http://hackaday.io/project/3721">Game Show Console</a>. Now, I plan on continuing work on Andy.
