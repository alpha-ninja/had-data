                            <div style="position:relative">
                                <div id="post-body-23269"><p>As we have no prior programming experience (and have rarely used a soldering iron in anger) we decided that rather then jumping straight to the 3D facial scan we should maybe try to develop a simple, but customisable model of a conventional nasal CPAP cannula.  </p><p>The little guy below is a commercially available device - costs about AU$30-40 each by the way!!! (another reason for this project - and our commitment to open source it - can&apos;t hurt disrupting the medical device market now can it??)</p><p><img src="https://cdn.hackaday.io/images/2959011439798140419.JPG"></p></div>

                            <h2><a href="/project/7295/log/23274-digital-vernier-calipers-serial-interface">Digital vernier calipers serial interface</a></h2>
                            <small class="authorship">a day ago  &bull;
                                <a class="gray-link" href="/project/7295/log/23274#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-23274"><p>So to customise the template we could either spend years trying to work out how to extract measurements of the diameter of the baby&apos;s nares (nostrils) and width of the nasal septum (spacing of the nostrils) from a 3D scan, or we could just measure them.</p><p>We figured to get some hacking cred for the project we should send the measurements from some digital caliper direct to the sketch.</p><p>Turns out we didn&apos;t earn any cred because this <a href="https://sites.google.com/site/marthalprojects/home/arduino/arduino-reads-digital-caliper">project</a> worked straight out of the box with our calipers.</p><p>Video below shows it in action...sending a value to a processing sketch template - the code for which will follow.  Got no idea about Github so I might just Dropbox these for now.</p><iframe style="width: 500px; height: 281px;" src="//www.youtube.com/embed/TAxD0ZSqF3M" allowfullscreen="" frameborder="0" height="281" width="500"></iframe></div>
                            </div>
                        </li>
                    
                        <li>
                            <h2><a href="/project/7295/log/23279-upgrade-makerbot-replicator-2-for-ninjaflex">Upgrade Makerbot Replicator 2 for NinjaFlex</a></h2>
                            <small class="authorship">21 hours ago  &bull;
                                <a class="gray-link" href="/project/7295/log/23279#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-23279"><p>Printing with NinjaFlex is tricky.  The filament path needs to be constrained so it doesn&apos;t buckle when being driven into the extruder.</p><p>I found that this <a href="http://www.thingiverse.com/thing:53125">extruder upgrade</a> is necessary for my Makerbot Replicator 2 - but easily printable in PLA.  The only changes I made to the standard Makerbot print settings were to slow the printing speed to 30mm/sec (230C print temp).</p><p>I also have a Printrbot Simple Metal - Ninjaflex not printing as well on this yet but will post details if I have any success.  The extruder is jamming when loading fillament.  I need to play with the extruder temps and modify the upgrade to the hot end I am using.<br></p></div>
                            </div>
                        </li>
                    
                        <li>
                            <h2><a href="/project/7295/log/23283-video">Video</a></h2>

                            <div style="position:relative">
                                <div id="post-body-23283"><p>So here&apos;s a little more video showing what we currently have working.  I will publish all code happily - just got to work out Github.  For now video shows the vernier sending measurements to the Processing sketch that define the diameter of each nares and the width of the septum.</p><p>If I hit the &apos;r&apos; key - the processing sketch then writes a file CPAP.obj of that current object.</p><p>This file can be opened in the Makerbot software (or could be imported to Cura etc) and after scaling 10% (the Processing sketch works in pixels not mm&apos;s)</p><p>And prints in ninjaflex...</p><p>The final images of the printed cannula on the bed show the orientation that I print the cannula.  Printing the cylinder in this orientation is not ideal but in fact - because it is so small iy actually prints ok without supports. <br></p><iframe style="width: 500px; height: 281px;" src="//www.youtube.com/embed/lxBez_l5SUM" allowfullscreen="" frameborder="0" height="281" width="500"></iframe></div>
                            </div>

                        <li>
                            <h2><a href="/project/7295/log/23289-what-works-what-doesnt">What Works - What doesn&#39;t...</a></h2>
                            <small class="authorship">20 hours ago  &bull;
                                <a class="gray-link" href="/project/7295/log/23289#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-23289"><p>The basic template with customisable dimensions works from a software perspective and prints (just).  You may notice when the object rotates on the screen I need to write the code to cut out the internal surface of the main cylinder where the two nares tubes intersect it.  Found the math - just not written the code yet.</p><p>I am currently optimising the printing.  So trying different profiles and wall thicknesses for the cylinder tube.  I am finding an eliptical (in profile) cylinder prints well as does a cylinder with a circular outer surface and eliptical inner surface. Bridging the top of the cylinder is obviously the difficult part of the print.</p><p>Next step will be to import a 3D facial scan into a Processing sketch as a PShape object and work out how to extract the data I need to match the basic template to the profile of the infants face.  Got some ideas on how to write a loop to run through the points that define each face in the PShape and extract a plane (through y axis).  Given this is the first programme I have ever written it is a little daunting.</p><p>Currently the Processing sketch draws the cylinder by drawing the circles each end of the cylinder (inner and outer) and then essentially joining them up with a series of triangular surfaces.  I figure if I can get a profile of the face, directly under the nose, and then slice that line into a number of segments I can then draw a whole bunch of connecting cylinders that follow the profile of the face.</p><p>That&apos;s the plan anyways.  At the very least I am pretty happy with my progress for a complete novice.</p><p>I plan that this will be the first prototype - design elements may change once the proof of concept is complete.  A few interesting ideas have cropped up while printing these test versions.  We do think it might be interesting to look at the internal structure of the main cylinder.  Creating turbulent air flows within this form may improve ventilation.  This could well be an interesting application of the rapid prototyping concept.  Particularly as the glaring obvious issue this product will have is that there is currently no FDA (or TAG in Australia etc) approved flexible 3D printing filament.  Partly the point of this Hackaday Prize entry is to demonstrate the potential uses of such a filament.<br></p></div>
