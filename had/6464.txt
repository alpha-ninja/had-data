                    <h5>Description</h5>
                    <div class="description">Having used KVM switches at work for a few years I have been looking for a cheap and expandable KVM that can grow as my collection of machines grow; every time I get a KVM that can do the job another computer comes my way!<br />At present most of my machines have their own monitors so video switching is not a priority just yet; I will be therefore focusing on the Keyboard and Mouse switching for now.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>This is the plan. I say plan... See it more as a brainfart that requires ALOT of actual thought:</p><p>An Arduino Uno will receive input from the Keyboard/Mouse (either using Host Shield or using PS/2 Devices and the PS/2 library.</p><p>The Uno will also act as an i2c master.</p><p>An Arduino Pro Micro clone (based on the Atmega32u4 chip) will be added as a slave for each Machine that needs controlling.</p><p>Each slave will carry the code to provide key strokes for all platforms (Win, Mac, Android etc plus custom). A switch added to the slave can determine mode.</p><p>The master will monitor the PS2 inputs and redirect the keystokes and mouse data to the relative slave when a key combo is entered; The Pro Micros will be recognised as HID devices and pass the received input to the machine.</p><p>In order to make this modular I am keen to design a case that features a backplane of sorts that carries the i2c lines, and then the Pro Micros can be connected by simply plugging into this backplane. This will all the switch to be easily expandable.</p>
                                
                            </p>
                        </div>

                            <a class="gray-link" href="/project/6464/log/20349#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>So...</p><p>I have finally got my hands on a Pro Micro to start playing with (I got impatient so pulled this one out of an old project (a keyboard emulating foot controller for music recording), and will start coding the slave and some of the master tomorrow  :-)</p><p>For the master Uno I have opted for a clone. Partly due to a love for it's colour (ha ha), but mostly because it has a micro USB port rather than the Printer-style USB. This way the end user is able to program the master using the same cable as the slave is connected to the target with.</p><p>I have had an idea for a further use for the master - I often use a small netbook for coding and suchlike, and it would be great to use it's keyboard as an input to other devices when quickly prototyping.</p><p>My idea is to use the serial input on the master as an alternative to the keyboard/mouse inputs on the board.</p><p>Up until now in my heads the master's USB port will grant the user with a serial menu to configure the KM switch in a variety of ways. I now plan on adding a menu option to allow 'through-typing' - anything typed into the serial window while in that mode will be passed to the selected slave as in normal use. a pre-defined keyboard shortcut would end this mode.</p>
                                
                            </p>
                        </li>
                    
                        <li>
                            <h2><a href="/project/6464/log/20087-research">Research</a></h2>
                            <small class="authorship">2 months ago  &bull;
                            <a class="gray-link" href="/project/6464/log/20087#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>I have been looking at various sources looking for ideas to incorporate into my project.</p><p>I am breaking my research into 4 areas; getting keystrokes/mouse movements into the master arduino, getting the keystrokes/mouse movements Out to the slave, receiving the output on the slave, and sending that out to the target machine.</p><p>I have briefly used i2c communication before and have used an Atmega32u4 for keyboard output in the past so I am focusing on how to read PS2 input  I have attached a link to a fantastic project which creates a DIY shield for an Uno.</p><p>It also states that this shield should still work if USB&gt;PS2 adaptors are used, so i may build them into my final design. USB keyboards are much more common these days.</p>
                                
                            </p>
