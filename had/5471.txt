                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">The landscape is vanishing around us.  From the Amazon jungle to the vacant lot across the street, the natural habitat is replaced by man-made structures.  Pyball ("Raspberry Pi Eyeball") is a tool for keeping track of what's happening to the world around us.  Recording the changing environment in time-lapse form lets you play back what happen, making it easy for people to understand how your environment is changing right out from under you.<br /><br />The current Pyball hardware is off-the-shelf from Adafruit.  A Ras-Pi 2, Ras-Pi camera, power supply, WiFi dongle, & weatherproof case.  The hardware is inexpensive enough (about $100 USD) that it's cost-effective to deploy.  <br /><br />Most of the work will eventually be in the software; scripts for time-lapse recording, uploading results, and ultimately detecting and focusing in on "interesting" events the Pyball is witness to.</div>
                </div>

                            <p><h2>Hardware to buy</h2><p>Almost all of the hardware for the Pyball is available off the shelf at <a href="http://www.adafruit.com/">Adafruit</a>. Feel free to shop around though; these are generic items and you might find better prices at Amazon or one of the big-time suppliers like Mouser, Digi-Key, etc.</p><p>Here&apos;s the list:</p><ul><li><strong>Raspberry Pi 2 Model B</strong><ul><li>It&apos;s worth spending the extra $5-10 to get the upgraded Pi 2, Model B. It has substantially more compute horsepower, which will come into play for analyzing the images on the fly or sending video.</li></ul></li><li><strong>Raspberry Pi Camera</strong></li><li><strong>USB WiFi</strong><ul><li>If your mounting location has acccess to hard-wired Ethernet, this isn&apos;t necessary. If your Pyball is mounted some distance from a WiFi station, getting a WiFi adapter with a long antenna makes a substantial improvement in the connection.</li></ul></li><li><strong>5V 2A Micro-USB power supply.</strong><ul><li>You&apos;ll want the 2A version to make sure the Pi has enough juice to run the peripherals (camera &amp; WiFi). Adafruit specs the power supplys up a bit (5.1V) to account for power loss in the cord. Not a bad idea.</li></ul></li><li><strong>Case</strong><ul><li>I used the <a href="https://www.adafruit.com/products/905">Adafruit weatherproof case</a>, but really, look at where you&apos;ll be mounting it. If you can see what you want from an office window, you might not need a case at all.</li></ul></li><li><strong>Camera mount</strong><ul><li>I found the <a href="https://www.adafruit.com/product/1434">Pimoroni Camera Mount</a> useful for mounting the camera in the case.</li></ul></li><li><strong>16G Micro-SD card</strong><ul><li>These are available for cheap from <a href="http://www.amazon.com/SanDisk-Class-microSDHC-Flash-Memory/dp/B001F6YRNO">Amazon</a>. You&apos;ll want at least 16G to have working space for your captured data. Even 16G is still less than $10.</li></ul></li></ul><p>All tolled, the hardware runs about USD$100.</p><h2>Software Setup</h2><p>The Raspberry Pi comes without any software or boot media. <a href="https://www.raspberrypi.org/documentation/installation/installing-images/">Installing Raspbian</a> is just the first step of many for getting the system put together. I installed Raspbian (vs. NOOBS) because I&apos;m not experimenting with any other OS configuration.</p><p>For the first Raspbian boot-up, plug the Pi into a screen and keyboard. You&apos;ll be presented with a dialog to install various system components. You&apos;ll want to add <code>ssh</code> support, for remote access, and definitely enable the camera drivers and configuration.</p><p>My router requires the hardware MAC (media access control) address for authentication. To get these, use the command:</p><pre><code>$ ifconfig | grep HWaddr eth0 Link encap:Ethernet HWaddr b8:27:eb:ea:7c:b9 wlan0 Link encap:Ethernet HWaddr ac:a2:13:39:ce:f1 $ </code></pre><p>The hex numbers give the hardware addresses; <code>eth0</code> is the wired address and <code>wlan0</code> is for the WiFi. <strong>Note:</strong> I learned the hard way the Rasp Pi can&apos;t easily support two network connections. If you&apos;re using WiFi, unplug the Ethernet cable, and vise-versa.</p><p>You&apos;ll need to <a href="https://www.raspberrypi.org/documentation/linux/usage/users.md">set up new users</a>. You should add whatever user you choose to the &quot;sudoers&quot; list, so the adminstrative command <code>sudo</code> is available:</p><pre><code>$ sudo bash # cat &gt;&gt; /etc/sudoers eyeball ALL=(ALL) NOPASSWD: ALL ^D </code></pre><p>To make sure you&apos;re running the latest software:</p><pre><code>$ sudo apt-get update $ sudo apt-get upgrade </code></pre><p>To set your time zone (important for this project):</p><pre><code>$ sudo cp /usr/share/zoneinfo/America/Los_Angeles /etc/localtime </code></pre><p>(Substitute your favorite time-zone city for <code>Los_Angeles</code>, use <code>ls /usr/share/zoneinfo/*/*</code> to see the full list)</p><p>To make sure enough USB power is available to drive the Wifi, add the line</p><pre><code>max_usb_current=1 </code></pre><p>to the file <code>/boot/config.txt</code> and reboot the Pi.</p><p>The camera fails if the user taking the picture isn&apos;t a member of the <code>video</code> group. Fix this with</p><pre><code>$ sudo usermod -G video &lt;username&gt;  $ sudo usermod -G video www-data </code></pre><p>The second line is to make sure the web server can access the video hardware. <strong>Note:</strong> May need to do the second step <em>after</em> installing the apache2 web server; see below.</p><p>Finally, I removed some software I had no use for (Scratch, a toy programming language &amp; Supercollider, an audio tool):</p><pre><code>$ sudo apt-get remove scratch $ sudo apt-get remove supercollider $ sudo apt-get autoremove </code></pre><h2>Hardware Setup</h2><p>Once you&apos;ve collected the items from the shopping list (under &quot;Components&quot;) the configuration of the hardware largely depends on where it&apos;s deployed.
