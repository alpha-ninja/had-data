                            <h2><a href="/project/4675/log/16462-getting-started">Getting started</a></h2>
                            <small class="authorship">4 months ago  &bull;
                                <a class="gray-link" href="/project/4675/log/16462#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-16462"><p>Moving this from the comments section to the build log section where it should have been.</p><p>Started looking into how exactly I'm going to connect the Pi to the load sensors. Since the Pi doesn't have any real built in analog ports available will probably go with a simple A-&gt;D setup. Will test different options, but high on the list is a simple RC setup to measure the charge discharge times to get a rough value back. Should be good enough for the simple game approach I'm planning.</p><p>Also haven't decided if I'm going to go with a scale and steal the load sensors or just one of the inexpensive load sensors available all over the place. Will depend on if I can find a good sale on scales :) Either way the generals are the same, hook to the load sensor, record the resistance, compare it to a known value and judge the weight.</p><p>Did start to work out some of the physical needs like "buttons" to "tare" the containers, lock a weight for things like emptying a bin before weeks end, and reset at end of week to "submit the score"</p><p>Of course, may just make it a touch screen interface. Still have all the stuff from the baby phone that proved to actually be a small display with a built in resistive touch screen. Should be easy enough to hijack that in.</p></div>
                            </div>
                        </li>
                    
                        <li>
                            <h2><a href="/project/4675/log/16463-code-repo-and-ideas">Code Repo and Ideas.</a></h2>
                            <small class="authorship">4 months ago  &bull;
                                <a class="gray-link" href="/project/4675/log/16463#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-16463"><p>Started setting up the code repo on Github (<a href="https://github.com/jdarling/lesswaste" target="_blank">https://github.com/jdarling/lesswaste</a>) today.  A lot of work to do, but after about a month of thinking about things I think I have the initial plan worked out in my head.</p><p>I think I'm going to model the game part of this from traditional resource collection type games where you will produce resources for each week that you recycle stuff.  Trash weight is taken as consumption of resources along with some other general information.  Still a lot of grey to work out, but my initial though is something like the carbon footprint as the scale itself.</p><p>More as I work it out, on the good news side of things, the Pi works perfectly and I've got it all setup to actually run the software.</p></div>
                            </div>

                        <li>
                            <h2><a href="/project/4675/log/17472-slow-going">Slow going</a></h2>
                            <small class="authorship">3 months ago  &bull;
                                <a class="gray-link" href="/project/4675/log/17472#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-17472"><p>I know I haven't put up any updates in a while so I thought I'd at least put up an update saying I'm still working on this project.  I've been devoting most of my time to getting the RPi working with the Arduino to communicate and read the ADC.  This has proven to be a bit more of an undertaking than I initially thought, but it has been good learning experience.  Once I have code working I'll post the followup with code and details.</p><p>The ADC I settled on is based on the HX711 and can be found on Amazon (<a href="http://smile.amazon.com/SMAKN%C2%AE-Weighing-Dual-channel-Conversion-Shieding/dp/B00M6POINC/" target="_blank">http://smile.amazon.com/SMAKN%C2%AE-Weighing-Dual-channel-Conversion-Shieding/dp/B00M6POINC/</a>) for ~$7USD.  It can handle two analog channels and will work well for the trash to recycle bin differential.  Granted at the larger end I'd like to add another one of these to allow for different types of recyclables to be measured (Glass, Plastic, Paper) but for now I'm working with just one.</p><p>As for load sensors, I'm playing with several different ones I got in different bathroom scales.  Most are the typical 3 wire variety and look very similar.  I've also ordered a set of 4 wire ones from Amazon (<a href="http://smile.amazon.com/Electronic-Weighting-Sensor-pressure-transducer/dp/B00VV4RN3Q" target="_blank">http://smile.amazon.com/Electronic-Weighting-Sensor-pressure-transducer/dp/B00VV4RN3Q</a>) as well due to their low cost and the fact I want to see how different they are.  The only concern I have is they are 2 hole setups and I have no clue where the deflection point is, but that should be part of the fun :)</p><p>Anyways, more details as I make progress.</p></div>
                            </div>

Unfortunately I didn't read well enough and the 4 wire load cells I thought I was ordering were actually only 1 single 4 wire load cell.  So, that isn't going to work at all.</p><p>Fortunately, what is going to work is the other 3 wire load cells I ordered.  They came in a nice pack of 4 and I think I can make them do exactly what I want.</p><p>I also ran across some really great articles on using the HX711 directly with the Raspberry PI, so no more need for an Arduino in the mix (yea, less parts)!</p><p>Just in case its useful for others, here is what I have found so far:</p><p><a href="http://hivetool.org/w/index.php?title=Interface_the_HX711_to_Pi">http://hivetool.org/w/index.php?title=Interface_the_HX711_to_Pi</a> (HX711 with the Pi)<br></p><p><a href="https://github.com/ggurov/hx711">https://github.com/ggurov/hx711</a> (HX711 library for the Pi)<br></p><p><img src="http://www.scottcutler.net/catpoop/load_cell_wiring.png"><br></p><p>Hopefully over the next couple days I can figure out how all this is going to go together and then I can get on to devising how exactly I plan on using it :)</p></div>
