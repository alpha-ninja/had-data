                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">The thermocouple is generating a temperature dependent voltage, roughly 10 mV at 250 °C. This voltage is measured by a 16 bits ADC. The ADC is included in the Freescale MKL25Z128VLK4 MCU. This MCU also have a nice feature of hardware averaging of up to 32 ADC samples. After reading the temperature, an on/off PID control loop is implemented inside the same MCU, in order to follow the temperature profile specified by the solder paste manufacturer. A temperature chart is displayed on the touch screen LCD, and data can be send/received over Wi-Fi, using an ESP8266 module. The oven AC power is switched on/off by a BT139 triac, that is driven by a MOC3063 optotriac with zero crossing detection.</div>
                </div>

                            <p><p>Installing the Freescale programming tools is not trivial.</p><ul><li>the IDE is based on Eclipse and GCC, and is called Kinetis Design Studio</li><li>to start a FRDM project, you also need to install the Freescale SDK as an Eclipse update</li><li>the programmer is based on OpenSDA, a software tool which, by the way, is not open, and is from another company</li></ul><p>How to start a new FRDM-KL25Z project:</p><ul><li>Install KDS (Freescale Kinetis Design Studio)</li><li>Install KSDK (Freescale Kinetis SDK), otherwise FRDM-KL25Z will not be present in the New -&gt; Project wizard<ul><li>- In the installation folder of the KSDK go to tools (Default is 'C:\Freescale\KSDK_1.2.0\tools\eclipse_update')</li><li>- Unzip 'KSDK_1.2.0_Eclipse_Update.zip' into 'C:\Freescale\KSDK_1.2.0\tools\eclipse_update\KSDK_1.2.0_Eclipse_Update'</li></ul></li><li>Open KDS and go to Help -&gt; Install New Software...<ul><li>- Press 'Add...', then 'Local', then browse to 'C:/Freescale/KSDK_1.2.0/tools/eclipse_update/KSDK_1.2.0_Eclipse_Update'</li><li>- Install 'KSDK 1.2.0 Eclipse Update'</li></ul></li><li>Restart KDS</li><li>Create a new project<ul><li>- File -&gt; New -&gt; Kinetis Project</li><li>- Type a name in the 'Project name:' field, then press 'Next'</li><li>- Type 'kl25z' in the 'Processor to be used:' field</li><li>- Click on FRDM-KL25Z, then press 'Next'</li><li>- Check the 'Processor Expert' checkbox, then press 'Next'</li><li>- Press 'Finish'</li></ul></li></ul><ol></ol>
                                
                            </p>
                        </li>
                    
                        <li>
                            <h2><a href="/project/6474/log/20207-why-freescale">Why Freescale?</a></h2>
                            <small class="authorship">2 months ago  &bull;
                            <a class="gray-link" href="/project/6474/log/20207#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>First try to read the temperature was with a Texas Instrument 16 bits ADC, ADS1118, and an Arduino.</p><p>It was a failure, because ADS1118 proves to be very unreliable when used with the schematics from it's own datasheet. I fried 3 chips. Still don't know how, but after I looked at the TI evaluation board schematic for ADS1118, I saw insane electrostatic protections for this chip. I guess it was an electrostatic failure, and I swear I never burnt a chip before in this way. </p><p>This is King ADS1118 Failure, the 3rd:</p><p><img src="https://cdn.hackaday.io/images/7611041435599805544.jpg"></p><p>So, I looked for another 16 bits ADC, possible with a temperature sensor on it, for the cold junction compensation of the thermocouple. A K type thermocouple gives about 10 mV at 250 °<span class="redactor-invisible-space">C (482 °<span class="redactor-invisible-space">F), so a </span></span>16 bits ADC was a request in order to avoid a low voltage DC analog amplifier circuit and it's offset and temperature drift problems.<br><br>It proves out that the Freescale dev board FRDM-KL25Z was just the right thing:</p><ul><li>it has a 16 bits ADC</li><li>ADC have a nice feature of hardware averaging up to 32 ADC samples, very useful to filter the voltage noises</li><li>temperature sensor on the chip, good for thermocouple cold junction compensation</li><li>much faster and with more memory and processing power than Arduino </li><li>pinout of the FRDM-KL25Z dev board is compatible with Arduino shields, so the same LCD with touchscreen can be used</li><li>native USB</li></ul><p>With all these hardware goodies, the only drawback is that I am not so familiar with Freescale software development tools, but I really like working with something new.</p><span class="redactor-invisible-space"></span>
                                
                            </p>
                        </li>
                    
                        <li>
                            <h2><a href="/project/6474/log/20204-the-220v-ac-power-onoff">The 220V AC Power On/Off</a></h2>
                            <small class="authorship">2 months ago  &bull;
                            <a class="gray-link" href="/project/6474/log/20204#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>To leave the oven unchanged, a controlled on/off power outlet is necessary.</p><p><img src="https://cdn.hackaday.io/images/211151435596434844.jpg"></p><p>A case from a dead PC power supply is used as an enclosure for the high voltage side of the controller. This is very convenient because it already has power sockets, fan and ventilation holes. The electronics from the old PC supply was removed and replaced with a smaller Mean Well PT-45B power source, a 220V AC to +5, +12 and -12V DC. This will power all the electronics.</p><p> The AC line is switched on/off by a BT139-600E triac, placed on a small radiator. The radiator size is big enough for a case with a running fan. For a galvanic isolation between the 220 AC line and the rest of the electronics parts, the power triac is driven by a small optotriac, MOC 3063. This particular model of optotriac have a zero-cross detection circuit included, so it will switch on the power only when the AC line sinusoid is close to zero. This is very good for the AC line power factor. The schematic for the power side is classic was extracted from the optotriac's datasheet.</p><p><img src="https://cdn.hackaday.io/images/9478011435598096018.jpg"></p>It was tested by applying 12V DC to the input, and it was able to switch the 1400W oven and keep it on with no problem.
                                
                            </p>
                        </li>
                    
                    </ul>
