                    <div class="description">Smart Homes is an awesome concept. However, most of the products on the market today are not really "Smart", and they're not really "automatic". Instead of automating anything, all they give you is the power of remote control. These systems also have little to no intelligence, and mostly rely on the users to set up every minuscule detail about how the system should operate. As a result, a tremendous amount of work is added to the user (which completely defeats the purpose of a smart home system), and sealing off regular, non-tech users from joining the fray.<br /><br />There are two basic building blocks for building a smart home:<br />1. A sensor network<br />2. an A.I. based control system<br /><br />A smart home can only be smart when a complete set of information is sent to the A.I., which then figures out what to do autonomously. Otherwise, a "smart home" just becomes another thing to point a remote control at.<br /><br />This project details the design of the sensors, which also double as light switches.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>In order for a home to be smart, a complete set of data must be provided to an A.I. unit, which will then learn the data and figure out what to do. The basic set of data we want to provide at this time are: lighting conditions, temperature/humidity, and human presence. These would be enough for lighting and climate control, as well as trigger presence based events.</p><p>While there are sensors available for purchase on the market right now, there are a few fundamental problems with them:</p><p>1. They are battery powered. No matter how power efficient the chips are, the batteries will run out one day. Whether we remember to change the batteries is another story.</p><p>2. No matter where you put them, they are conspicuous. I need something that lives in the background, constantly working, but forgotten.</p><p>Light switches fit the bill perfectly. They are plugged into the mains, which means they never run out of power. They are present in every room, which means the data set will be complete. They are inconspicuous, because they're everywhere.</p><p><strong>Current Features:</strong></p><p><strong>Lighting Control:</strong></p><p><strong></strong>- Lighting usage pattern learning A. I.</p><p>- Dynamic timeout and shutoff for more precise energy control</p><p>- Automatic Smart Bulb discovery and pairing</p><p>- Automatic, imperceptible dimming control for better energy savings</p><p><strong>Energy Management:</strong></p><p>- More precise climate control through Nest</p><p>- Whole House presence detection gives more precise control over Nest's away mode features</p><p>- Area temperature detection / presence detection gives better climate control where people are present (instead of where the thermostat is).</p><p><strong>Security:</strong></p><p>- Presence / Away learning can send messages to your smart phone when people is detected in/near the house when no one is normally present</p><p>- Lighting Replay: After obtaining usual lighting usage patterns, the lights can be controlled to simulate normal usage when no one is present. Enabled for Vacation Mode.</p><p><strong>Others:</strong></p><p>- iBeacon in each light switch, along with learned usage patterns, puts the most relevant lighting control right at your finger tip.</p><p>- Third party API will be available</p><p>- Feature request will be available soon</p>
                                
                            </p>
                        </div>

                            <p><p>Today, I released the firmware files for the light switch on my <a href="https://github.com/Squirco/Sensor-Node-LS">Github</a> repository. The code has been completely re-written and cleaned up. Features that were deemed not ready for public release have been removed for now, and I will update the code as those features are re-written to a higher working standard.</p><p>Release notes for this version are as follows:</p><p>v0.5b </p><p>Initial commit after re-write from v0.4c. Some features have been re-written to work more efficiently. Some features have been removed for further re-write.</p><p>Working features:</p><ul><li>Light switch control, both hardware and from remote software</li><li>Temperature measurement</li><li>Ambient light measurement</li><li>Networking and System Join Chain</li></ul><p>Features to be re-implemented:</p><ul><li>on-device sync mode</li><li>LED control</li><li>Presence Detection</li></ul><p>Features under experimentation</p><ul><li>OTA update</li></ul><p>v0.5b </p><p>Initial commit after re-write from v0.4c. Some features have been removed for further re-write.</p><p>Working features:</p><ul><li>Light switch control, both hardware and from remote software</li><li>Temperature measurement</li><li>Ambient light measurement</li><li>Networking and System Join Chain</li></ul><p>Features to be re-implemented:</p><ul><li>on-device sync mode</li><li>LED control</li><li>Presence Detection</li></ul><p>Features under experimentation</p><ul><li>OTA update</li></ul><p>I&apos;m still playing with the CC2541 chip. I decided to see if I can move the microwave tomography features to that chip since it&apos;s constantly broadcasting anyway. This allows for better resource allocation between the two chips.<br></p><p>In other news, I&apos;ve revised the hardware slightly and upgraded the NTC to a BME280, which was intended for a multi-switch version. This will give temperature, humidity and pressure data across every sensor node, which will improve the accuracy and functionality for many things to come.<br></p><p>If you&apos;re one of the brave ones that are willing to try, or have tried (!) playing the hardware/firmware, please let me know how it goes. Feedback is very important.</p><p>Now that these sensor nodes are nearly working as designed. it&apos;s time to spend more time on the device that makes all the data work: <a href="https://hackaday.io/project/4560-squirco-smart-home-system-hub-thermostat">Squirco Smart Home System - Hub + Thermostat</a>.
