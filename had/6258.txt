                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">I created this project because I wanted to learn about PIC microcontrollers and the USB protocol.  I also wanted to see how simple I could make a USB device.  I've gotten it down to two components: a PIC16F1455 microcontroller and the USB connector.  The microcontroller acts as a USB serial device and will send the temperature as an ASCII string once per second.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>The PIC16F1455 is a relatively new microcontroller that can do USB without an external crystal.  It also has internal pull-up resistors for the USB data lines.  These things mean no extra hardware is necessary for USB communication.</p><p>I needed something useful to send over USB and I noticed that the PIC16F1455 has a Temperature Indicator Module.  This peripheral will let you read the operating temperature of the silicon die over the ADC.  It's not terribly accurate, but the silicon die temperature will be about equal to the outside temperature.<span class="redactor-invisible-space"><br></span></p><p><span class="redactor-invisible-space">I carefully soldered a male USB Type A connector to the PIC16F1455's Vdd, Vss, D+, and D- pins.   Then, using the M-Stack USB Stack from Signal 11, I wrote a program to enumerate the PIC16F1455 as a USB CDC serial device and send the temperature in ADC counts as an ASCII string at 1 Hz.<br></span></p><p><span class="redactor-invisible-space">From the computer side, it's easy to connect to the USB temperature data logger like any other serial device, parse the incoming strings, and save the data to a file with a timestamp.</span></p><p><span class="redactor-invisible-space">The last step is to map the ADC counts to a temperature scale.  Microchip has an application note AN1333 with some equations.  However, for a constant Vdd, the scale is pretty linear so it's easiest to record the ADC count at two known temperatures and interpolate between them.</span></p>
                                
                            </p>
                        </div>
