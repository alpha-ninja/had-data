                    <h5>Description</h5>
                    <div class="description">The project aimed at building a open source (hardware + software), low cost development platform for creating interactive devices for deafblind people.<br /><br />This platform is composed of two main components: A main board for reading capacitive sensors and managing the tactile feedback (vibration motors), and a Java software to setup the device gestures language.<br /><br />I set as a goal of the project the construction of a wired glove (<a href='http://en.wikipedia.org/wiki/Wired_glove)' target="_blank">http://en.wikipedia.org/wiki/Wired_glove)</a> to help a deafblind friend to interact with a Android smartphone, this way she will be able to send and receive text messages.</div>

                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><h1>Concept Video</h1><p><iframe width="500" height="281" src="//www.youtube.com/embed/4-4JwWg4oSc" frameborder="0" allowfullscreen=""></iframe></p><h1>The Problem</h1><p>There are roughly <a href="http://www.deafblindinfo.org/faq.html" target="_blank">3,600,000</a> deafblind people in the world, and probably the main issue that they face is the loneliness. As they can have a conversation only if the person is right beside and knows how to communicate with them, this make even harder for people with multiple sensory<span class="redactor-invisible-space"> disabilities to have a minimum of social interaction on daily basis. </span>Other big problem is how they interact with technology, the primary way that they exchange information is by the sense of touch, our devices (e.g. radio, television, computers, tablets and smartphones) do not have haptic feedback, and sometimes buying a refreshable braille display is expensive (<a href="https://www.afb.org/ProdBrowseCatResults.asp?CatID=43" target="_blank">from $3,500 to $15,000</a><span class="redactor-invisible-space"><a href="https://www.afb.org/ProdBrowseCatResults.asp?CatID=43"></a>).</span></p><h1>The Tact-Tiles Approach </h1><p>What I want to do is to create a API and a scalable hardware platform that can make viable (easy and cheap) the creation of gadgets for people with multiple sensory disabilities. Making the interaction with our current devices possible.</p><p><img src="https://cdn.hackaday.io/images/6467671439831910461.png"></p><p>The haptic interaction is made by little pads which are touch sensitive and have a vibration motor for feedback, these pads can be arranged in every possible way, making it possible to create various types of gadgets. All the input and output gestures are processed by an easy to use API.</p><p><img src="https://cdn.hackaday.io/images/9644431439831958195.png"></p><p>A Tact-Tiles device can utilize up to 32 pads if the developer chooses to use the Touch-X board. </p><p><img src="https://cdn.hackaday.io/images/7866021439832033875.jpg"></p><p><img src="https://cdn.hackaday.io/images/9279321439832064672.jpg"></p><p>When this project is completed all the information will be available for the construction of custom pads and boards compatible with the API.</p><p>Here is the complete schematic for the Touch-X v2.0 board:</p><p><img src="https://cdn.hackaday.io/images/6261331439832093332.png"></p><h1>Current Status</h1><p>The breadboard version worked as <a href="https://instagram.com/p/zTt4ioncfz/" target="_blank">expected</a>. The first version of the Touch-X board was tested and it worked very well, but it has a footprint error, so I had to design a second version. I already ordered the v2.0 board (it will be delivered in a week).</p><p>I&apos;m from Brazil but now I&apos;m doing a under graduate exchange program in Ireland for a year, this delayed a little bit of the project but I&apos;m confident that I can finish this project on time.</p><p>The software is not finished yet, I need to solve a simple problem with saving/opening files also I need to improve the gesture detection algorithm, I&apos;ll post a log about that in some weeks. But it is ready to generate the patterns for recognition and update the board with the <a href="http://hackaday.com/2011/09/09/program-an-arduino-using-your-sound-card/">audio output port</a>.</p><h2>Hope you like the concept and help to spread the idea! :D</h2><p>All the software used in this project is licensed by the General Public License 3.0</p><p>All the hardware used in this project is licensed by Creative Commons CC-BY-SA 4.0 and Apache 2.0</p><p>All the other materials (e.g. texts and images) are also licensed by <span class="redactor-invisible-space">Creative Commons CC-BY-SA 4.0</span></p>
