                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">”Walk with Me” is an interactive installation that focuses on the interaction between two people in the present moment. Using EEG headsets, 14 lights in the installation works as a graph that shows how concentrated the two participants are in the moment.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>"Walk with Me" was conceived by Danish designer <a href="http://ditteblohm.com/WALK-WITH-ME" target="_blank">Ditte Blohm</a> in collaboration with Kolding School of Design and Ecco and was exhibited at Salone del Mobile in Milan. I had the pleasure of helping with the technical aspects of the project. </p><p>While Ditte's site and the <a href="http://www.hirschandmann.com/walking-in-comfort-at-milan-furniture-fair/" target="_blank">Hirsch &amp; Mann</a> blog cover more of the concept, this post will focus on the technology. The two <a href="http://store.neurosky.com/products/mindwave-mobile" target="_blank">Mindwave Mobile headsets</a> in the installation were paired with two Blue SMIRF (RN-42 powered) Bluetooth modules. The Mindwave Mobile headsets transmit data using Bluetooth Serial Port Profile (SPP), so raw bytes can be read through the Bluetooth modules as if the Arduino was connected to a Serial port. I found <a href="http://mix-engineering.com/" target="_blank">Sophi Kravitz</a>'s <a href="https://learn.sparkfun.com/tutorials/hackers-in-residence---hacking-mindwave-mobile" target="_blank">Mindwave Mobile teardown on Sparkfun</a> extremely useful. </p><p><br>I did run into a few gotchas, mainly in terms of parsing the data. I couldn't seem to get communication working using Arduino's SoftwareSerial library. Hardware Serial worked just fine. Later I got a chance to try <a href="https://www.pjrc.com/teensy/td_libs_AltSoftSerial.html" target="_blank">AltSoftSerial</a> which also worked. I've cleanup up the code a bit and encapsulated as my first Arduino library: <a href="http://github.com/orgicus/Mindwave" target="_blank">Mindwave</a>. I can't guarantee it's the most efficient code as I'm not primarily a c++/embedded developer, but for fetching simple data from the Mindwave Mobile headset, it should work.<br></p><p><br>The headset provides a few simple values for attention, relaxation or voluntary blinks. Through practice we found attention to have a nice balance being both interesting to explore in terms of interactions (not as simple <br>as voluntary blinks) and reliable enough. Relaxation, which in some of the documentation is labeled as meditation felt harder to pin down. Personally I didn't trust this value, because I seem to be pretty Zen most of the time according to it, even when my attention reaches a peak. This would be a good point to remind you that I do not have any studies in neurology and I wouldn't be qualified to mention more than what the Mindwave Mobile documentation mentions already. That said, the headset also provides brainwave data for alpha, beta, gamma and delta waves. Within the constraints of the project, after a brief exploration, we've chosen not to pursue this path further for the time being. In the future, a bit of pattern recognition might be interesting to explore using this brainwave data. </p><p>For the current iteration we've mapped the attention data belonging to one headset to half of the 1m diameter LED rings. These were controlled using the DMX protocol. In terms of hardware, we used <a href="http://learn.plankman.com/superDmxShield/" target="_blank">Daniel's SuperDMX Shield</a> and soldered the BlueSMIRF-Silver module on top.</p><p><img src="https://cdn.hackaday.io/images/1718121432604763894.jpg"></p>This setup was duplicated for the other headset, mostly. The 2nd shield was dealing Mindwave data only and was connected to the 1st shield using Software Serial. I guess this would be a hacky version of multi-theading: each headset, has it's own microcontroller to deal with parsing, however, one of the headsets also sent DMX data to the 48 CH DMX Dimmer.<p><br><a href="http://whitewing.co.uk/rgbdim48.html" target="_blank">Mike Harrison's 48 CH DMX Dimmer</a> was used on a lot of cool projects and does a lot of neat things, including gamma correction and internal smoothing. This was very cool, because we didn't have to waste cycles interpolating brightness values on the already busy Arduino. All we had to do is feed it different values and the dimmer would nicely sooth between the values.</p><p>Mike might frown at my decision to connect the lights, because even though the DMX dimmer has 48 channels, I've chosen to leave a lot of 'breathing space' let's call it. (Maybe also for reversing brown/blue)</p><p><img src="https://cdn.hackaday.io/images/4868481432605407291.jpg"></p>Ditte's done a great job at soldering the connections and keeping things neat and tidy. I thought I'd return the favour by keeping the DMX Dimmer setup simple, taking advantage of the numbers listed on the board (notice the 1 to 16 white labels). This would avoid confusion when assembling/disassembling the installation.<p>A bit of arithmetic in software was the only thing needed to keep this simple layout.</p><p><img src="https://cdn.hackaday.io/images/6880571432605722738.png"></p>The project wouldn't have been possible without Daniel's Hirschmann's support. We are grateful for all the resources he made...
