                            <h2><a href="/project/7229/log/22997-kicad-library-import">KiCAD Library Import</a></h2>
                            <small class="authorship">2 days ago  &bull;
                                <a class="gray-link" href="/project/7229/log/22997#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-22997"><h1>KiCAD Library Import</h1><p>You can import KiCAD libraries to MeowCAD for use in your projects.  These imports are local to your account.  Navigating to your &apos;Portfolio&apos; page and hitting the &apos;Import&apos; page, you can then select which KiCAD library files you want to import.</p><p><img src="https://cdn.hackaday.io/images/4905171439708212728.png"></p><p>The interface is still a bit rough but functional.</p><h2>Behind The Scenes</h2><p>The import process is a bit convoluted but works as follows:</p><p>* There is a Python CGI script that stores the uploaded file(s) into a staging directory</p><p>* Add an a message to the &apos;importq&apos; Redis array and create a new Redis object with the appropriate library information.</p><p>* The &quot;background&quot; process &apos;libmod_import_d.py&apos; scans the &apos;importq&apos; for new entries and when it finds them, processes them.</p><p>* After &apos;libmod_import_d.py&apos; processes the imports, it creates a file in a queue directory.</p><p>* The &quot;background&quot; process &quot;libmod_snap_d.py&apos; scans the queue directory for new files and when it finds them, starts processing them and creating picture snapshots of each.</p><p>The idea is to decouple each step as much as possible.  The upload should just dump the files into a directory without blocking the client or web server.  Once the files are there, the conversion from KiCAD to JSON can happen.  Once the JSON conversion has happened, then the library element .PNG snapshots can be created.</p><p>All code can be seen on the <a href="https://github.com/abetusk/www.meowcad.com">www.meowcad.com</a> GitHub repo.</p></div>
                            </div>
                        </li>

                            <h2><a href="/project/7229/log/23001-eagle-to-kicad-library-conversion">Eagle to KiCAD Library Conversion</a></h2>
                            <small class="authorship">2 days ago  &bull;
                                <a class="gray-link" href="/project/7229/log/23001#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-23001"><h1>Eagle to KiCAD Library Conversion</h1><p>MeowCAD uses an older version of KiCAD&apos;s board format and it&apos;s useful for MeowCAD to have a conversion from Eagle the older KiCAD version.  In addition, it&apos;s useful to add some &apos;default&apos; components and footprints from some hobby friendly (and open source hardware friendly) companies like Adafruit, SparkFun and Dangerous Prototypes.</p><p>The main repo that was used was a fork of <a href="https://github.com/DanChianucci/Eagle2Kicad">DanChianucci&apos;s</a> <a href="https://github.com/abetusk/Eagle2Kicad">Eagle2Kicad</a> tool.  I modified this slightly to be more command line friendly.  The source is available at my <a href="https://github.com/abetusk/Eagle2Kicad">Eagle2Kicad</a> GtiHub.</p><p>Converted are available in the <a href="https://github.com/abetusk/www.meowcad.com">MeowCAD</a> website source but can also be found in the <a href="https://github.com/abetusk/Open-Hardware-Libraries">Open-Hardware-Libraries</a> source.</p><p><img src="https://cdn.hackaday.io/images/4751531439710241345.png"></p></div>
                            </div>

                            <h2><a href="/project/7229/log/23003-gcode-export">GCode Export</a></h2>
                            <small class="authorship">2 days ago  &bull;
                                <a class="gray-link" href="/project/7229/log/23003#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-23003"><h1>GCode Export</h1><p>MeowCAD has the ability to export to GCode as well.  The feature is still experimental and the GCode itself might be a little &apos;quirky&apos; but the basics are there.</p><p>The main program that does the conversion is the <a href="https://github.com/abetusk/gbl2ngc">gbl2ngc</a> which takes in a Gerber file and produces a GCode file.  The heavy lifting is done by the excellent <a href="http://www.angusj.com/delphi/clipper.php">clipperlib</a> library (by Angus Johnson).  MeowCAD uses this during it&apos;s export process via the <a href="https://github.com/abetusk/www.meowcad.com/blob/release/cgi/downloadProject.py">downloadProject.py</a> file.  downloadProject.py calles gbl2ngc externally and collects the result into a final ZIP file to give back to the user.<br></p><p>I have a <a href="http://mechaelephant.com/ngc_view?id=5491563e-5925-4f72-8209-855b578acd8e">GCode viewer</a> running on one of my other servers.  Below is a screenshot of the front copper layer (there are others):</p><p><img src="https://cdn.hackaday.io/images/7185971439710758517.png"></p></div>
                            </div>

                        <li>
                            <h2><a href="/project/7229/log/23008-board-fabrication">Board Fabrication</a></h2>
                            <small class="authorship">2 days ago  &bull;
                                <a class="gray-link" href="/project/7229/log/23008#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-23008"><h1>PCB Fabrication</h1><p>Ultimately <a href="https://meowcad.com/">MeowCAD</a> is meant to create boards.  The real test is whether you can go from idea to full board.</p><p>I&apos;ve created two functioning boards with MeowCAD.  Both have been designed completely in MeowCAD, schematics and PCBs, sent to OSHPark for creation and reflowed at home with my own reflow oven (a converted convection toaster oven).  You can read a bit about it in a <a href="https://meowcad.com/blog/2015-06-16">blog post I did</a>.</p><p>Both circuits are very simple and are more of a &apos;proof of concept&apos; than anything else.  Remember, I&apos;m still learning electronics.  I built MeowCAD to help me with my electronics education.  Both circuits were my first  time designing and reflowing surface mount components (using 1208 and 0603 parts).</p><h2><a href="https://meowcad.com/project?projectId=1bd24ce3-91d9-4846-b30d-1a8cfacd72d0">8-Bit Heart</a></h2><p>The first board I successfully created through MeowCAD was the &apos;<a href="https://meowcad.com/project?projectId=1bd24ce3-91d9-4846-b30d-1a8cfacd72d0">8-Bit Heart</a>&apos;.  Below is the board in action.  The board itself is about 2 inches square with an ATMega328 directly driving a 4x4 1208 (red) LED matrix.  The battery is a CR2032 pill battery (the kind you find in watches sometimes).  One of the reasons I wanted AdaFruit and Dangerous Prototypes library import was because they both had a nice CR2032 surface mount holder that I wanted to use that wasn&apos;t available through the standard KiCAD libraries.</p><p>The board scrolls a message across the LED display in &apos;ticker tape&apos; fashion.  Below is a video of it in action:</p><iframe style="width: 500px; height: 281px;" width="500" height="281" src="//www.youtube.com/embed/0BI4rqI0vt0" frameborder="0" allowfullscreen=""></iframe><p>Source is available on my <a href="https://github.com/abetusk/8bitheart">GitHub page</a>.</p><h2><a href="https://meowcad.com/project?projectId=12b1cb79-9d14-4319-a8cc-878b5ee93019">h8ARt</a></h2><p>The second board was a smaller one.  I originally wanted to create an earring but I think the board, while small, is still too large.</p><p>The &apos;<a href="https://meowcad.com/project?projectId=12b1cb79-9d14-4319-a8cc-878b5ee93019">h8ARt</a>&apos; board uses an attiny13 driving a 3x3 0603 (red) LED matrix via <a href="https://en.wikipedia.org/wiki/Charlieplexing">Charlieplexing</a>.  The battery is mounted on the back and is a CR1220.  It also displays a message in &apos;ticker tape&apos; style though the font is extremely hard to read as 3x3 is not that legible.  I originally had the source in C but quickly ran into the 1k memory limit of the attiny13 and had to go to assembly to fit all the functionality I wanted in it.  Below is a video of it in action.</p><iframe style="width: 500px; height: 281px;" width="500" height="281" src="//www.youtube.com/embed/Gg_DkGkVkSI" frameborder="0" allowfullscreen=""></iframe><p>Source is also available on my <a href="https://github.com/abetusk/h8ARt">GitHub page</a>.</p><h2>Success!</h2><p>The circuits are simple but I&apos;m very happy with how they turned out.  MeowCAD can be used to actually fabricate boards!</p></div>
