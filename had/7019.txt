                    <h5>Description</h5>
                    <div class="description">The main goal of this project is to make managing medications easier and safer than ever before. This is essentially a "medication vending machine" on steroids (pun intended). The machine has the capability to easily store single pills and dispense them when required. There have been many projects that simply dispense pills after a timer etc. This project is so much more. Here are some of the key features:<br /><br />Automatic pill organization - Each pill is stored individually and can be independently accessed automatically.<br /><br />Wifi / Cloud connected - The machine, it's inventory, and its use statistics can be connected to a doctor or pharmacy so care-givers can monitor their patients use of medication.<br /> <br />Smart Medication Control - By being connected to the cloud the machine can check if medications are safe to be used together and warn patients. It can also prevent over doses by limiting the frequency of dispenses.<br /><br />The machine is powered by a Raspberry Pi and 3D printed Mechanics.<br /></div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><h1>Project Introduction</h1><p>This project is my solution to the problem of medication non-compliance. Medication non-compliance basically refers to when a drug is not taken properly. This could be because someone forgets to take an important medication, someone takes too much of a medication, or someone takes two or more dedications that should not be taken together. This problem ended up costing Americans over $300 Billion last year in totally avoidable medical bills and ER visits. Among the aging population 30% of all early deaths were from this problem.</p><p>My project tries to solve this problem my making it easily to manage medications and prevent medication non-compliance. This is done by storing all your medications in a machine that can dispense individual pill on-demand. By allowing the machine to store the pills:</p><ul><li>You don&apos;t have to worry about mis-reading a pill bottle label and consuming the wrong medication. </li><li>You are always reminded to take the correct medication at the correct time.</li><li>The machine knows what medications you are taking and can make sure they don&apos;t conflict.</li></ul><p>This project takes this concept much further by adding WiFi and allowing it to connect to the cloud. This enables the following functionality:</p><ul><li>Connecting to pharmacy APIs to get prescription information. Big pharmacies like Walgreens and CVS have APIs where you can get a prescription&apos;s info based in its barcode. In theory the machine could even order a refill when it is running low.</li><li>Connecting drug usage information to doctors and family. Doctors will be able to see a patients drug consumption history and improve diagnosis and future prescriptions.</li><li>The machine can connect to a drug interaction database that could improve the prevention of incorrect (over) prescription and dangerous drug interactions.</li></ul><p>Another key feature that differentiates this project form other medicine reminder projects is the rich UI. Most projects allow the user to set a simple timer, daily, hourly, etc. but this doesn&apos;t allow for more dynamic problems such as pain medications. For example, some pain medications have a maximum limit, without a required schedule. The UI for this machine is designed to easily manage this type of use case by locking the dispensing of a drug until the max rate timer has expired. This can be very hard to manage without a robust software UI.</p><h1>Technical Details</h1><p>The machine is primarily controlled by a RaspberryPI. The PI runs a web server that serves a web-page to kweb running in kiosk mode with a PiTFT resistive touch LCD interface. The web interface sends CGI commands that control servos within the machine and read an encoder. The PI also has a WiFi dongle that allows for a smart phone to connect to the internal web server and manage the machine remotely. All inventory info and scheduling is stored in a local SQLite database. A custom power management and servo control board was designed for this project as well. Below is a block diagram of the machine.</p><p><img src="https://cdn.hackaday.io/images/4599671438743093361.png"></p> <p>Here are the software libraries used in this project:</p><ul><li><a href="https://github.com/adafruit/Adafruit-Raspberry-Pi-Python-Code">https://github.com/adafruit/Adafruit-Raspberry-Pi-Python-Code</a></li><li><a href="https://github.com/jquery/jquery-mobile">https://github.com/jquery/jquery-mobile</a></li><li><a href="https://github.com/adafruit/adafruit-rpi-fbtft/">https://github.com/adafruit/adafruit-rpi-fbtft/</a></li></ul><h1>More Details</h1><p>Here&apos;s the extended video about the project.                             <h2><a href="/project/7019/log/22274-custom-servo-and-power-board">Custom Servo and Power Board</a></h2>
                            <small class="authorship">11 days ago  &bull;
                            <a class="gray-link" href="/project/7019/log/22274#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>In the original design I wanted to use a custom made optical encoder. I decided to spin my own board for this and figured I might as well integrate regulators and a PWM chip to drive the servos. Below is the board layout. On the side back side in blue are the infrared photodiodes that I originally planned to use as an optical encoder through the rotating barrel. There are 8 Servo connection at the top that drive the motors. There are two regulators: one that powers the servos and one that powers the Raspberry Pi. The regulators can be enabled/disabled to shutdown the Pi and save power. They are separated to avoid brownout on the Pi&apos;s supply when multiple motors are consuming a lot of power.</p><p><img src="https://cdn.hackaday.io/images/423441438901718801.png"></p><p>Here&apos;s the parts list:</p><ul><li>5 RR1206 IR photo diodes</li><li>2 MCP1826 5V regulators</li><li>1 PCA9685 PWM IC</li><li>2 4.7 uF caps</li><li>1 10uF cap</li><li>2 1uF caps</li><li>5 10k pots</li><li>8 220-ohm resistors</li><li>3 10k resistors</li><li>Male headers</li></ul><p>Here&apos;s the associated schematic:</p><p><img src="https://cdn.hackaday.io/images/8267711438901644176.png"></p>
