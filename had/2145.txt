                    <h5>Description</h5>
                    <div class="description">The main goal is to promote the use of switch mode power supplies as efficient and/or more flexible substitutions.<br /><br />Most users simply pick the 7805 because they don't know the more efficient alternatives. Some of these modules have no simple linear alternatives e.g. Boost, isolated or Buck-Boost converters.  One would otherwise require the use of multiple transformers windings or operate multiple supplies from a much higher input voltage.</div>

                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>The original goal of this project was to document the modules used in my <a href="https://hackaday.io/project/5741-emergency-backup-power-and-wallwarts-eliminator">UPS project.</a>  As my collect grow due to the different requirement of the UPS peripheral, so does the type of module. </p><p> Most users simply pick the 7805 because they don&apos;t know the more efficient alternatives.  This is surprising as there are a lot of affordable DC/DC supply modules or PCB.  I am hoping that this project would bring a bit more awareness to the matter.</p><h2>Buck Mode Power Supply</h2><p>The traditional 78xx linear regulator are simple and convenient to use, but they are highly inefficient especially if you are powering it from a high voltage.  </p><p>This power supply is pretty much straight from TI/National&apos;s LM3485 datasheet with what I have in my parts bin.  Just a plug-in regulator module for one of my old projects.  Just need to add a bulk electrolytic/tantalum etc. cap for the power input.  You should already have that one in your project.</p><p>This was developed to be used 24/7 in my DC UPS since 2008 and has proven itself to be reliable.</p><p><a href="http://www.ti.com/product/LM3485?keyMatch=lm3485&amp;tisearch=Search-EN">http://www.ti.com/product/LM3485?keyMatch=lm3485&amp;tisearch=Search-EN</a></p><p><img src="https://cdn.hackaday.io/images/1577121431640228427.png"></p><p>From the datasheet - Typical Performance Characteristic  (i.e. your mileage may vary)</p><p><img src="https://cdn.hackaday.io/images/5831591406475182540.png"></p><p>There are all kinds of designs out there.  I happen to have samples for this part. This is designed as pluggable modules e.g. 5V or 12V module for my UPS project. </p><p>A number of power supply chips have an upper limit on the duty cycles at 90-95%. This basically means that your output voltage is at most 90-95% of the input. They are sort of like the older generation of linear regulator with a higher drop out. For battery operations, such as running a 5V module from 4 AA batteries, don&apos;t leave you with a lot of head room before the 5V output is outside of the +/- 10% range needed.<br></p><p>When the input voltage is below the set point for the output,  the <span class="redactor-invisible-space">MOSFET in this design is allowed to run at 100% duty cycle.  It connects the input to the output and acts like a straight pass through.  </span>  This drop out mode makes this design behave like a <a href="https://en.wikipedia.org/wiki/Low-dropout_regulator">LDO</a>.  My routers runs off a 12V module and when running off the backup battery, the drop out mode allows my router to run all the way down to the low battery cut off voltage of 9.5V.</p><h2>Buck Module - KIS-3R33S<br></h2><p>These are pulled modules that was popular from Chinese site a few years ago.  There was some reverse engineering, but most of them require modifications inside the module to use them other than 3.3V modules.  I did my homework and show you how to use these without modifications the way the original designer intended.  Why else would they put a voltage adjustment pin?</p><p>See project log here:<br><a href="https://hackaday.io/project/2145-smps-replacement-for-7805/log/17939-kis-3r33s-modules">https://hackaday.io/project/2145-smps-replacement-for-7805/log/17939-kis-3r33s-modules</a></p><h2>Boost Module</h2><p>These are modules that has a higher output than the input voltage.  This particular module was used in my UPS as the DSL modem uses a 26V supply.</p><p><a href="https://hackaday.io/project/2145-smps-replacement-for-7805/log/17938-26v-boost-module">https://hackaday.io/project/2145-smps-replacement-for-7805/log/17938-26v-boost-module</a><span class="redactor-invisible-space"><br></span></p><h2>Isolated Module</h2><p>Both my VoIP (<a href="https://en.wikipedia.org/wiki/ATA">ATA</a>) and cord less phone decided to cut cost and eliminated their isolation circuits as they assume that the other device would have it.  The end result is that these devices cannot coexist from power supplies that shared a common ground.  This isolated supply module allow these two devices to work together in my UPS.</p><p><span class="redactor-invisible-space"><a href="https://hackaday.io/project/2145-smps-replacement-for-7805/log/17942-isolated-module">https://hackaday.io/project/2145-smps-replacement-for-7805/log/17942-isolated-module</a><span class="redactor-invisible-space"><br></span></span></p><h2>SEPIC Converter<br></h2><p>Unlike the previous modules, <a href="https://en.wikipedia.org/wiki/Single-ended_primary-inductor_converter">SEPIC</a> is a buck boost that can operate when its input voltage is lower or higher than its output voltage.                         <li>
                            <h2><a href="/project/2145/log/21303-sepic-converter">SEPIC Converter</a></h2>
                            <small class="authorship">a month ago  &bull;
                            <a class="gray-link" href="/project/2145/log/21303#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>aven&apos;t forgotten about this project.  Pretty much everything I build has its own power supplies, so when the power supply circuits from those project works, they&apos;ll get pulled into this project.</p><p>I have a working prototype of a 5V/100mA <a href="https://en.wikipedia.org/wiki/Single-ended_primary-inductor_converter">SEPIC converter</a>.   It is designed to work from 3V to 15V. I did some quick test <a href="https://hackaday.io/project/4993-dual-channel-battery-chargeranalyzer/log/21302-we-need-bigger-inductors">here</a>.  </p><p><img src="https://cdn.hackaday.io/images/9446771437534856141.png"></p><p>It is basically a buck-boost converter that can regulate an input that is lower or higher than its output.  It sounds too good to be true. There are some draw back as it has a lower efficiency - power going through a few more parts means more losses.  When your input voltage can be higher or lower, then this is an interesting and useful building block.  This is taken from my Dual charger design which has such a requirement.  You are otherwise better off from an efficiency point of view to use either a buck or a boost converter. </p><p><span></span></p><p>Basically, you can turn a boost converter into a buck-boost by adding a cap (C11) and an inductor (L2). The output is taken from L2 with rectifier diode (D4) and output cap (C14).  The cap C11  serves as energy transfer between the two inductors.  This particular version is built with 2 separate inductors. There are also version of the design that have L1 and L2 as a pair of coupled inductors.  I use a couple of those in my charger project.</p><p>When I get around, I&apos;ll clean up the layout and release the tested block.</p><p>Reference: <br><a href="http://www.simonbramble.co.uk/dc_dc_converter_design/buck_boost_converter/sepic_buck_boost_converter_design.htm">http://www.simonbramble.co.uk/dc_dc_converter_design/buck_boost_converter/sepic_buck_boost_converter_design.htm<br>http://www.coilcraft.com/prod_smpwrcoupled.cfm</a><br></p><p>This is the current layout I have for it. The switching frequency is 1.5MHz and it deserves a bit of care.   It is mostly a single layer design with a full ground plane on the secondary side optimized for home made PCB.  I have minimize the high frequency/high current loop area. Use shielded inductors if you want to minimize ECM issues.</p><p><img src="https://cdn.hackaday.io/images/9416801438713394360.png"></p><p>I have ordered the cheapest inductors from China, so it might take a few week to arrive.  This is what it would look like.</p><p><img src="https://cdn.hackaday.io/images/3853211438714086947.png"></p>I notice that AP3019AKTR-G1 (boost converter for LED) are being sold on Aliexpress at $3.09 for 10 pieces.  It is probably a variant of the Diodes Inc AP3012. The feedback voltage is lower, but it is a matter of changing the voltage divider ratios.  May be I should use the SOT23-6 package so that it would take either chips.<hr><p>Here is the home made PCB.</p><p><img src="https://cdn.hackaday.io/images/1513071439641075540.jpg"></p>
                                
                            </p>
                        </li>
                    
                        <li>
                            <h2><a href="/project/2145/log/17942-isolated-module">Isolated Module</a></h2>
                            <small class="authorship">3 months ago  &bull;
                            <a class="gray-link" href="/project/2145/log/17942#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>Both my VoIP ATA box and my cordless phone cheated on the isolation to cut cost.  I had to make this isolated flyback module so that the can coexist in my <a href="https://hackaday.io/project/5741-dc-ups-and-wallwarts-eliminator">UPS</a>.</p><p><img src="https://cdn.hackaday.io/images/282791431642835538.png">I used a similar circuit to the boost design.  L1B provides a feedback for the output voltage. The isolated output at winding L1C is poorly regulated as it is regulated by proxy via transformer ratio. That&apos;s all I need for the phone as it has its own regulator.  </p><p>Adding a LDO to the output is an easy way to tighten the regulation without adding too much overall complexity.  I supposed I could have used the old optoisolator with a TL431 for <a href="http://powerelectronics.com/site-files/powerelectronics.com/files/archive/powerelectronics.com/mag/Kollman%20and%20Betten%20September%202003.pdf">linear current feedback</a>. The feedback network is a bit too advanced for me to simulate in LTSpice besides it is going to be hard to find a TI part model in LinearTech&apos;s tool. This topology will provided a tight regulation as well as being able to use off the shelf SEPIC couple inductor.<span class="redactor-invisible-space"></span></p><p> I used a <a href="http://www.eetimes.com/author.asp?doc_id=1280601">zener diode clamp</a> to limit the voltage across L1A to protect Q1.</p><p>To make the transformer, I counted the number of turns on a 10uH inductor as I unwind it. To reduce the leak inductance, I rewound it with 3 thinner strands of wires at the same time (aka <a href="http://en.wikipedia.org/wiki/Bifilar_coil">trifilar winding</a>).  L1B is for feedback and requires very little current, so you can get by with very thin wires.   Keep the same number of turns as before and you&apos;ll have the same inductance.</p><p><img src="https://cdn.hackaday.io/images/6312861431751238578.png"></p><p><img src="https://cdn.hackaday.io/images/6793241431751259153.png"></p><p>The output as on the two solder pads - 0V on top and +6V on bottom.<br><img src="https://cdn.hackaday.io/images/8247391431644934137.jpg">Sorry have to be the potato picture as it is in use.</p>
                                
                            </p>

                            <p>I bought 20 of used ones from China a long time ago.  <p>I measured some of the internal component values:</p><p><img src="https://cdn.hackaday.io/images/5656461431640981899.png"></p><p>Note: The current rating is not even close to 2A probably due to the inductor and the cooling problem in the case.</p><p>By connecting an external resistor to the ADJ pin (7) to ground, you can increase the output voltage. By connecting an external resistor between the output and ADJ pin, you can decrease the output voltage.  You do not need to modify them if you want to use them as a fixed output supply unless you want to get higher than 10V output. You also need to use a higher input voltage than 12V as the duty cycle is limited.</p><p><img src="https://cdn.hackaday.io/images/7529251431642321237.png">JP1 selects 5V/3.3V outputs.  JP2 and JP3 are lined up with supply rails on the breadboard.  When S1 is closed, the output is turned off.</p><p><img src="https://cdn.hackaday.io/images/8304701431641955806.jpg"></p>

            <div>
                <div id="comment-24903-content">
                    <p>When I was designing Pi Power I gravitated towards the LM3485 as well. One thing I did differently was that I used a sense resistor rather than the MOSFET for ISENSE. That let me be a little bit freer with MOSFET substitutions without impacting the overcurrent protection. But that design is for 2A, so the parts are beefier as well.<br><br>One thing I've figured out is that since this is a hysteresis driven device, the operating frequency rises with the delta between Vin and Vout. And as the operating frequency rises, the impact of the MOSFET's capacitance starts to become significant. I had to reduce the maximum input voltage of my design because the MOSFETs were getting too hot. Alas, their failure mode is to fail-short, which is a very, very bad thing for a buck converter...<br><br></p>
