                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">This anti-theft alarm is basically an Arduino Mega Shield with fairly complex software.<br /><br />The system is composed by a main circuit (the arduino shield) installed in an hidden place and an i2c connected "remote control panel" near the main house door, offering LCD, capacitive buttons and RFID recognition.<br /><br />The whole system may be managed and configured from the webserver; webpages are published in iphone/android webapp format. When the system is alarmed, it sends mail and SMS to the owner.<br />It also connects to house domotic system (e.g. close rolling shutters when it's activated), control IP cams, send push notification, detect humidity and temperature...</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p><strong><em>FATURES</em></strong></p><p>This shield transform an Arduino Mega 2560 in a burglar alarm ... with some cool extras.</p><p>- Supports NFC and RFIDs activation with authorization control<br>- Full web interface for managing and configuring the system, supported by smartphones as native WebApp<br>- Mail and GSM notifications when the system is alarmed<br>- Domotic integration (it may open/close windows, turn on/off lights, and so on)<br>- Fully configurable timings and options, stored in the arduino eprom<br>- Perimetral and Volumetric sensors may be independently enabled/disabled<br>- Automatically disable open windows sensors, warning you when activated<br>- Batteries to keep the system running on power fault<br>- DHT temperature/humidity sensor and communication to emoncms server or similar (read here for further information: <a href="http://emoncms.org/">http://emoncms.org</a>)<br>- Good looking installation :)</p><hr><p><strong><em>HARDWARE</em></strong></p><p>Components are listed in the proper paragraph below. Basically, the system is made up two parts:</p><p>- </p><p><strong>The main circuit</strong><strong> (Arduino Mega shield)</strong>: this shield is connected with all the sensors (at the moment, 6 magnetic sensors and 2 volumetrics), the external siren, the ethernet and the GSM module. It is powered by a small UPS with a 2,2 Ah battery, enough to make it working for some hours if the power fall off. The siren has an independent 2 Ah battery too. The circuit communicates with the remote control panel with i2c. The whole circuit is completely hidden in a in-wall mounted box.<br>It basically read sensors as "buttons"; magnetic sensors (and tampers, like the siren one) were supplied directly with 5V, while the volumetric sensor output (12V) was converted to 5V with a voltage divider. To avoid arduino heating problems, I supplied energy to the components with independent voltage regulators (ENC28J60 has great consumption). The circuit also supply power to the siren. All power parts are protected with fuse and decoupling capacitors [<em>not shown in the schematics for simplicity</em>].</p><p><img src="https://cdn.hackaday.io/images/6339111433459786442.jpg"></p><p>- </p><p><strong>The remote panel</strong>: I tried to build a good looking panel for enabling/disabling/controlling the alarm. So, I use a in-wall mounted box covered with a white plexyglass frame with a small window for LCD. In the box, I put a small circuit, essentially an i2c hub with proper voltage regulators (the whole panel is supplied with 12v) that I use to communicate with all the i2c components: NFC reader, LCD and MPR121 for capacitive buttons/led. I also add a little speaker to give the user sound feedbacks. The panel is connected to the shield with a long 2+4 wire cable. The result is pretty good and it works perfectly. Since the i2c cable that connects to the main circuit is long, more than i2c specification, I planned to add some kind of serial translator. However, it already works perfectly so I didn't need it; the only trick to make it working fine was to add some capacitors to the magnetic sensors pins near the Arduino, since the rolling shutter electric motors randomly introduced some noise that caused i2c errors.</p><p><img src="https://cdn.hackaday.io/images/7361201408469759414.jpg"></p><hr><p><strong><em>SOFTWARE</em></strong></p><p>Software was obviously written with Arduino IDE and it may be downloaded from the link on the left.</p><p>I used several libraries: <em>UIPEthernet </em>for ethernet (or Ethernet if you want to use W5100 standard chipset), <em>time </em>and <em>timezone </em>for time management, <em>LiquidCrystal_I2C </em>for LCD and many others. For safety purpose, I preferred <em>WSWire </em>library over standard <em>Wire</em>, because it offers non-blocking i2c protocol (so, if i2c fails or has been broken, the system keeps running on). You can download the whole Arduino 1.6.4 IDE with the proper libraries from the link on the left.</p><p>The system configuration (sensor number / type, authorized NFC IDs, etc.) has to be hard-coded in the arduino code in some proper sections; system options (siren duration and other timings, domotic control, etc.) may be runtime configurated through the webserver, and will be stored in the Arduino eprom. Date and time are automatically and periodically acquired with NTP.</p><p>Basically, the system has three user interfaces:<br></p><p>- <strong>Remote Panel</strong>: using the capacitive buttons, user may select...</p>
