                            <h2><a href="/project/2308/log/7281-rockblock-naked">RockBLOCK Naked </a></h2>
                            <small class="authorship">a year ago  &bull;
                                <a class="gray-link" href="/project/2308/log/7281#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-7281"><p>The <a href="https://www.rock7mobile.com/shop-product-detail.php?productId=3">RockBLOCK Naked</a> is used to handle the actual sending and receiving of the Iridium SBD messages. This unit has integrated transceiver, power and antenna and most importantly has been approved to transmit and operate on the Iridium network which allows a working project to be produced without the delay of waiting for FCC approval, etc. In the future I would like to produce my own open source Iridium communication board making use of the available <a href="http://www.iridium.com/products/Iridium9602.aspx?productCategoryID=2">Iridium 9602 transceiver</a>.&nbsp;</p><p>Using the Rock 7 web interface a RockBlock can be configured to deliver message to a specific web application URL using HTTP POSTs as detailed in this&nbsp;<a href="https://www.rock7mobile.com/downloads/RockBLOCK-Web-Services-User-Guide.pdf">guide</a>. For the sat.Send the URL will be the sat.Send Meteor web application. The web app will process the POST parameters, store them for display and also forward the message to the relevant SMS number using the Twilio API. For return messages the process is similar, a HTTP POST can be made to the rock7 URL. This will be achieve by using the web app or by sending an SMS to a Twilio number that is associated with the sat.Send (see future post for details and code).</p></div>
                            </div>

                            <h2><a href="/project/2308/log/7284-microcontroller">Microcontroller</a></h2>
                            <small class="authorship">a year ago  &bull;
                                <a class="gray-link" href="/project/2308/log/7284#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-7284"><p>I've decided to use the <a href="http://arduino.cc/en/Main/ArduinoBoardYun">Arduino Yun</a> as the central hub of the project, from previous experience of Arduino development they are really easy to get started with, there's loads of information available and the community is great for support. The Yun is cool because as well as a Atmel ATmega32U4 processor it has an Atheros AR9331 processor that supports Linux which means it can be used as a webserver and can even run node.js which could be interesting and it has built in wifi. As well as all that I won one in a previous competition and I want to make use of it! A guide to the Yun can be found <a href="http://arduino.cc/en/Guide/ArduinoYun">here</a> and below is a brief overview of the specs taken from the <a href="http://arduino.cc/en/Main/ArduinoBoardYun">product page</a>:</p><p>"The Arduino YÃºn is a microcontroller board based on the ATmega32u4 (<a href="http://www.atmel.com/dyn/resources/prod_documents/7766S.pdf">datasheet</a>) and the Atheros AR9331. The Atheros processor supports a Linux distribution based on OpenWrt named OpenWrt-Yun. The board has built-in Ethernet and WiFi support, a USB-A port, micro-SD card slot, 20 digital input/output pins (of which 7 can be used as PWM outputs and 12 as analog inputs), a 16 MHz crystal oscillator, a micro USB connection, an ICSP header, and a 3 reset buttons."</p></div>
                            </div>

                            <h2><a href="/project/2308/log/20187-esp8266">ESP8266</a></h2>
                            <small class="authorship">2 months ago  &bull;
                                <a class="gray-link" href="/project/2308/log/20187#discussion-list">
                                    0 comments
                                </a>
                                </small>
                            <div style="position:relative">
                                <div id="post-body-20187"><p>Over the last few months I've done a lot of experimenting with the RockBlock Naked at work and it's a pretty cool unit that is easy to use. I have a bunch of documentation and code I need to tidy up and upload but overall I'm happy it will work with no issues. I do still think it would be worth trying out a raw Iridium 9602 transceiver but this will have to wait.</p><p>To make this unit viable and useful in real life it has to be portable so it has to be capable of running of it's own power source. I've had a nagging feeling for some time that my initial component selections are probably pretty power hungry so realistically will need to investigate lower power alternatives. For example the ESP8266 is a <a href="http://makezine.com/2015/04/01/esp8266-5-microcontroller-wi-fi-now-arduino-compatible/">UART to WiFi SoC built around a Tensilica Xtensa LX3 processor</a>. The standard breakout boards usually come with a firmware supporting a simple AT command set to configure and control the wireless module but the SoC is actually fully programmable and can be used as a micro-controller, with Wi-Fi and GPIO. Now the <a href="https://github.com/esp8266/Arduino">Arduino IDE has support for the  ESP8266</a> and there is also a Lua based firmware available called <a href="https://github.com/nodemcu/nodemcu-firmware">NodeMcu</a>. There are even <a href="http://randomnerdtutorials.com/esp8266-web-server/">tutorial</a> detailing how to run a webserver and allow GPIO config via an interface. Could this be used to provide a user interface to the RockBlock?</p></div>
