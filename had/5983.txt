                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">Sitting for long periods has become more common in today’s jobs causing serious health issues. Unfortunately the plethora of activity trackers fail to address the issue: being rather active while sitting does not provide a good picture of the amount of time spent sitting.<br /><br />Sit.Up is a simple device that attaches to the chair and alerts the user by vibration if sitting for too long while also tracking the sitting time. It aims to be easy to install and forget on any chair and work with any person. On top of that, it is very low cost and has months to years of battery life. With an optional WiFi connection, the data can be uploaded and you can track your sitting times.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p>Check out the project video<p><iframe width="640" height="360" src="https://www.youtube.com/embed/QBzrj6lIwqI?feature=oembed" frameborder="0" allowfullscreen=""></iframe></p><p>Let&#x2019;s have a look at why sitting is bad for you, could not explain it better myself:</p><p><iframe width="640" height="360" src="https://www.youtube.com/embed/wUEl8KrMz14?feature=oembed" frameborder="0" allowfullscreen=""></iframe></p>Project status<p>At the moment the core of the project is working: sitting is tracked, the data is uploaded and logged and the user is alerted of the prolonged sitting, while the web interface shows daily sitting time and real-time status. However, there is still a lot of work to be done: a dedicated hardware design is needed while the calibration algorithms could use some improvement. The current web interface is not yet complete and could be more user friendly. The summary is shown below:</p><a href="http://www.electrobob.com/wp-content/uploads/2015/08/system_status_1.jpg"><img alt="system_status_1" src="http://www.electrobob.com/wp-content/uploads/2015/08/system_status_1-600x424.jpg"></a>System description<p>Under each chair there is a small module that monitors the status with the help of a capacitive sensor. When you sit down, a timer is started, counting how long you have been on the chair. After a while, a discreet vibration alerts you of the time spent on the chair as an invitation to take a short walk.</p><p>Ignore it for too long and the notifications get more insistent and even public, with the help of a buzzer. After all, not letting your chair buzz would be a public sign of trying to stay healthy.</p><p>The sensor connects to the internet and sends data to the server by using the local WiFi. The web interface allows the user to configure the alert times and track sitting. The web page contains instant status of the chair as well as statistics of the amount of daily sitting times.<a href="http://www.electrobob.com/wp-content/uploads/2015/08/situp_system_3.jpg"><img alt="situp_system_3" src="http://www.electrobob.com/wp-content/uploads/2015/08/situp_system_3-600x254.jpg"></a></p>Licence:<p>All hardware in this project is Open Hardware.                             <h2><a href="/project/5983/log/22567-10082012-wrapping-things-up">10.08.2012 Wrapping things up</a></h2>
                            <small class="authorship">6 days ago  &bull;
                            <a class="gray-link" href="/project/5983/log/22567#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><a href="http://www.electrobob.com/wp-content/uploads/2015/08/all_complete-6-3.jpg"><img alt="all_complete 6-3" src="http://www.electrobob.com/wp-content/uploads/2015/08/all_complete-6-3-600x600.jpg"></a><p>With the original capacitive sensor proven to be the best option, it was time to wrap up the project and build the final prototype device and code.</p><p>Software wise, a lot of features had to be added, most importantly the ability to alert the user when sitting for too long. The device alerts by vibration every 15 minutes, but once an hour has passed, it becomes more annoying. A web interface is in the works which allow configuration of parameters. Other updated include  better calibration, low power modes, vibrator patterns, LED blinks, buzzer sounds, LCD disable and a lot of clean-up.</p><p>Hardware wise, the module lost it&#x2019;s LCD for debugging purposes and gained a vibrator motor, to discretely alert the user if sitting too much. Along with the batteries attached under the PCB, the whole thing is a lot more compact, practically invisible when mounted under a chair. The device is build around a general purpose board I have designed for the microcontroller, the complete schematic is available in <a href="https://github.com/electrobob/Sit_UP">the project documentation on GitHub</a>.</p><p><a href="http://www.electrobob.com/wp-content/uploads/2015/05/DSC_4849.jpg"><img alt="DSC_4849" src="http://www.electrobob.com/wp-content/uploads/2015/05/DSC_4849-600x390.jpg"></a><a href="http://www.electrobob.com/simple-activestandby-current-measurement/">Current measuremen</a>t shows up at around 25&#x3BC;A while sleeping, mostly eaten by the EPS8266 module, the micro itself needs only 0.2&#x3BC;A. Make sure to <em>cut the trace of the red power LED on the module</em>, otherwise it will waste most of the power. In active measurement the current drawn is 4mA, going for an average of about 100&#x3BC;A,  should provide 2 years of stand by time. Of course, frequent sitting for long times drains the battery because of the vibrator motor and WiFi. Nevertheless, with typical usage I expect about 6 months from a pair of AAs.</p><p><a href="http://www.electrobob.com/wp-content/uploads/2015/05/DSC_4853.jpg"><img alt="DSC_4853" src="http://www.electrobob.com/wp-content/uploads/2015/05/DSC_4853-600x398.jpg"></a></p><p>Schematics for the device are simple, with plenty of pins to spare on the micro, there are no major issues. The buzzer and LED get a pin each, while the vibrator gets 3, to increase the current capability an elude a transistor. Optionally, there is an ESP8266 for data uploading and logging. The capacitive sensor itself only requires to connections to the outside: GND and Touch, no other components.</p><p><a href="http://www.electrobob.com/wp-content/uploads/2015/08/sit_up.png"><img alt="sit_up" src="http://www.electrobob.com/wp-content/uploads/2015/08/sit_up-600x437.png"></a>The device, now installed looks much cleaner than before</p><p><a href="http://www.electrobob.com/wp-content/uploads/2015/08/DSC_4921.jpg"><img alt="DSC_4921" src="http://www.electrobob.com/wp-content/uploads/2015/08/DSC_4921-600x398.jpg"></a></p><p>Looking at some data from June, I can say I probably spent too many hours on my home chair, even after a day of office work</p><p><a href="http://www.electrobob.com/wp-content/uploads/2015/08/situp_june_data-2.jpg"><img alt="situp_june_data 2" src="http://www.electrobob.com/wp-content/uploads/2015/08/situp_june_data-2-600x157.jpg"></a></p>
                                
                            </p>

                        <li>
                            <h2><a href="/project/5983/log/22546-12072015-e-filed-sensor">12.07.2015 E-filed sensor</a></h2>
                            <small class="authorship">6 days ago  &bull;
                            <a class="gray-link" href="/project/5983/log/22546#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>As my initial capacitive sensor using aluminium tape as electrodes proved successful, I looked into shrinking the size. Compared to measuring the capacitance, another method of detecting a conductive object like the human body around a sensor is by using the <a href="http://www.microchip.com/pagehandler/en-us/technology/gestic/technology/home.html">electric field method: </a>2 electrodes form a transmitter and receiver. When a conductive body comes between them, the amount of signal received changes. I have attempted to use a 75x100mm double sided PCB for this method, at first. In open air the detection distance can be even 20cm, but things change once this plate is placed under a chair: the sensor is very directional and there are ways to sit on the chair without being detected. Swapping the PCB for the original aluminium tape electrodes works better, but<em> there is no point of implementing this method, as it requires extra parts</em>. The system in this case consists of the microcontroller outputting a 1MHz signal on a pin serving connected to the transmitter electrode while the other electrode, the receiver, gets read out by the microcontroller&#x2019;s ADC after passing through a <a href="https://en.wikipedia.org/wiki/Detector_(radio)">simple detector</a>.</p><p><a href="http://www.electrobob.com/wp-content/uploads/2015/05/DSC_4772.jpg"><img alt="DSC_4772" src="http://www.electrobob.com/wp-content/uploads/2015/05/DSC_4772-600x373.jpg"></a></p>

First, it is designed for momentarily proximity, so after a while it will compensate the person and think there is actually nothing there. Second, it seems to get stuck in &#x201C;detected&#x201D; mode quite often, even the datasheet mentions a &#x201C;Stuck Release Mechanism&#x201D; as a feature! So, no good.</p><p><a href="http://www.electrobob.com/wp-content/uploads/2015/05/DSC_4775.jpg"><img alt="DSC_4775" src="http://www.electrobob.com/wp-content/uploads/2015/05/DSC_4775-600x398.jpg"></a></p>
