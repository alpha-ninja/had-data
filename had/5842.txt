                    <h5>Description</h5>
                    <div class="description">Hackaday 2015<br />Develop robust lightning sensors with GPS based on earlier Arduino design. Develop PCB and move to Atmega328 with AS3935 Franklin Lightning Sensor and u-blox LEA-M8T GPS. <br /><br />For details on the Quarter Finals requirement, please see the log entry below: <a href='https://hackaday.io/project/5842/log/22470-quarter-final-requirements.' target="_blank">https://hackaday.io/project/5842/log/22470-quarter-final-requirements.</a><br /><br />General<br />Food production in Africa can be improved through better weather forecasts, early warnings, and climate observations.                 <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p><strong></strong></p><pre></pre><strong><p>For Quarter Final requirements, see: <br><a href="https://hackaday.io/project/5842/log/22470-quarter-final-requirements">https://hackaday.io/project/5842/log/22470-quarter-final-requirements</a></p><p>For technical details, see: </p><p><a href="https://github.com/nvandegiesen/Team_TAHMO/wiki">https://github.com/nvandegiesen/Team_TAHMO/wiki</a></p>Lightning sensors</strong><p>The AS3935 Franklin Lightning sensor has been designed for inclusion in consumer devices such as watches, fitness trackers, etc. The sensor provides warnings about approaching lightning storms by providing the distance to nearby strikes. When deployed in a network, one can map the actual lightning position though trilateration. Gilbert Mwangi built a proof of concept that mapped lightning around Nairobi. An article about this can be found here: <a href="http://eserver.kabarak.ac.ke/ojs/index.php/journal1/article/view/28/9">http://eserver.kabarak.ac.ke/ojs/index.php/journal1/article/view/28/9</a>. In the present project, we will develop this concept further, move to a low power ATmel MCU, and design the PCB.</p><p><strong><br></strong></p><p><strong>GPS Atmospheric Moisture</strong></p><p>The standard use of a GPS is providing users with their geographical position. The GPS signal is affected by moisture in the atmosphere. Because our stations are (mainly) stationary, we can analyze the signal to extract information about atmospheric moisture. Such information is important for improved weather forecasts as it gives the &quot;total precipitable water column&quot;. See, for example, &quot;<a href="http://dx.doi.org/10.1175/1520-0450(1996)035%3C0830:GMDEOT%3E2.0.CO;2">GPS Meteorology: Direct Estimation of the Absolute Value of Precipitable Water</a>&quot;. Recent improvements in GPS devices allow us to perform this analysis with affordable GPS devices. Because we already need a GPS for the timing of the lightning strikes, this is a &quot;free&quot; add-on.</p><p><strong><br></strong></p><p><strong><span></span>Background &amp; State-of-the-Art</strong></p><p>TAHMO stands for &quot;Trans-African Hydro-Meteorological Observatory&quot;, a program that aims to design, build, deploy, and operate a network 20,000 weather stations in sub-Saharan Africa (<a href="http://www.tahmo.org" target="_blank">www.tahmo.org</a>). Presently, we have pilot stations in Senegal, Chad, Nigeria, Uganda, and South Africa and &quot;pilots at scale&quot; in Ghana and Kenya. </p><p>TAHMO stations have to be be affordable and robust. For example, unlike standard weather stations, there should not be any moving parts as these are quickly affected by dust and insects. Maintenance so be limited to some simple cleaning. The network makes use of the availability of cell-phone coverage that has reached most of Africa over the past decade. </p><p>Presently, we work with stations developed by Decagon that come close to TAHMO specs. These stations cover all standard meteo measurements such as air temperature and relative humidity, wind speed and direction, incoming solar radiation, and barometric pressure. </p><p><strong><em>The aim of the Team TAHMO Hackaday project is to add some very useful advanced sensing capabilities, based on existing consumer electronics. </em></strong></p><p>Presently, we are thinking of:</p><ul><li>Lightning sensors</li><li>GPS derived atmospheric moisture</li><li>Large scale soil moisture</li></ul><p>The team is based on participants of the first TAHMO Sensor Design Competition.</p><p><img src="https://cdn.hackaday.io/images/6710801438116815876.png" width="434" height="449" style="width: 434px; height: 449px;">                                                        First results of lightning multi-lateration around Nairobi, Kenya with early standalone prototypes.</p><p><span></span></p><p><img src="https://cdn.hackaday.io/images/9718521438116937324.png"></p>                                                                                                                   Early prototype used.
                                
                            </p>

                            <h2><a href="/project/5842/log/23075-power-supply-test">Power supply test</a></h2>
                            <small class="authorship">a day ago  &bull;
                            <a class="gray-link" href="/project/5842/log/23075#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>To test whether the small <a href="http://ixapps.ixys.com/DataSheet/SLMD960H12L-Datasheet.pdf">IXYS solar panel</a> would provide sufficient power, we ran the power supply for a week. The load was a simple diode with 330 Ohm resistor, taking 5mA at 3.3V Vcc. The results suggest that the supply is ample, especially because the average power consumption of the detector will be much less (&lt; 1 mA).</p><p><img src="https://cdn.hackaday.io/images/8740511439744179707.png"></p><p><img src="https://cdn.hackaday.io/images/7158531439744044707.png"></p><p>In the final design, we may still want to go for a more advanced solution. This <a href="http://www.linear.com/solutions/1786">solar energy harvester solution</a> has been adopted by Decagon for its next generation weather stations and would work nicely here as well. The voltage regulator, <a href="http://www.ti.com/lit/ds/symlink/lp2950-n.pdf">lp2950</a>, was chosen for its efficiency but sometimes gives off higher voltages than 3.3V, up to 4.5V, while operatiing withing the specified current range. Any suggestions as to why would be very welcome.</p>
                                
                            </p>

                        <li>
                            <h2><a href="/project/5842/log/22469-system-design-document">System design document</a></h2>
                            <small class="authorship">7 days ago  &bull;
                            <a class="gray-link" href="/project/5842/log/22469#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>The system consists of four parts:</p><ul><li>Power supply</li><li>AS3935 Thunder click</li><li>ATMega328</li><li>NEA-M8T GPS Drotek</li></ul><p>The AS3935 Thunder click and the NEA-M8T GPS Drotek are off-the-shelve PCBs. As these involves RF designs, we decided to go ahead with these solution under the motto &#x201C;do not solve a problem that has been solved before&#x201D;. In the final design, we may integrate these PCBs in our own design to save costs. There is an open source design for the AS3935 by Tautic (<a href="http://wiki.tautic.com/Category:AS3935_Lightning_Sensor_Dev_Board"></a><a href="http://wiki.tautic.com/Category:AS3935_Lightning_Sensor_Dev_Board">http://wiki.tautic.com/Category:AS3935_Lightning_Sensor_Dev_Board</a>).</p><p>All parts have been tested but the integrated design, hardware and software, not yet. The ATMega328 provides the intelligence to the system but is asleep most of the time. The AS3935 will be in listening mode most of the time, taking 60 muA at 3.3V. Once lightning is detected, an interrupt wakes up the ATMega328, which starts a timer and wakes up the GPS. The data (energy and time stamp) are sent to the TAHMO station through a SDI12 connection.</p><p><img src="https://cdn.hackaday.io/images/2404361439268055995.png"></p><p><em>Power supply</em></p><p>The power supply is relatively straightforward and is based on a design by Jon Viduchic (<a href="http://tahmo.org/going-solar-with-tahmo-stations/"></a><a href="http://tahmo.org/going-solar-with-tahmo-stations/">http://tahmo.org/going-solar-with-tahmo-stations/</a>). We are running a small duration test with the load being a LED and 330 Ohm resistor (15 mW) running on the board. The solar panel is 4cm x 3.5cm.</p><p><em>AS3935 Thunder click</em></p><p>Communication on the new PCB between the <em>ATMega328</em> and the &apos;<em>AS3935 Thunder click&apos;</em> has been tested. The led needs to be removed to save energy.</p><p><em>ATMega328</em> </p><p>PCB with ATMega has been tested with &apos;<em>AS3935 Thunder click</em>&apos;.</p><p><em>NEA-M8T GPS Drotek</em></p><p>Communication with GPS has been tested before but not yet putting it to sleep and waking the GPS.</p><p><strong>NEXT STEPS</strong></p><ul><li>The software still needs to be integrated according to the flowchart below:</li></ul><p><img src="https://lh4.googleusercontent.com/8TUMtWYsYFEGyHgtyfzweJZUWniiWCcQAfgMhSDQE05Gfc9MOpiPEiw9u_IQ9IfdtxmSDQYzWEA_NCt-CL3LaFtO1SXUz2oGvUuE2vL6L8G5ykp4ZkorQNdI5sUiizs3iYmu02U" alt="system_diagram.png"></p><ul><li>The SDI12 connectivity needs to be implemented and tested. Code is available at <a href="https://github.com/StroudCenter/Arduino-SDI-12/wiki">https://github.com/StroudCenter/Arduino-SDI-12/wiki</a></li><li>A PCB is never perfect so there will be some differences with the next batch. The latest design, and all earlier versions, can be found at the GitHub repository (<a href="https://github.com/nvandegiesen/Team_TAHMO/wiki">https://github.com/nvandegiesen/Team_TAHMO/wiki</a>).</li><li>An important PCB improvement is to have a tx/tr connection instead of only the SDI12 connection. This makes on-board debugging and software updates much simpler. </li><li>Housing! We need to make this innocuous, raintight, and affordable.</li></ul><p><strong>LICENSES</strong></p><p>For the 2015 Hackaday, all licences and permissions have to be documented. In our case, that is relatively simple. </p><p>All software is built as open source software under a GNU Lesser General Public License, by others or us. The code is written as Arduino code and all code built by others is recognized as such. We use Codebender as development environment.</p><p>The designs are freely available through our GitHub repository (see <a href="https://github.com/nvandegiesen/Team_TAHMO/wiki"></a><a href="https://github.com/nvandegiesen/Team_TAHMO/wiki">https://github.com/nvandegiesen/Team_TAHMO/wiki</a>). We used Fritzing for our design.</p>
