                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">The AuthBand is a wearable open source/hardware token which removes the daily hassle of authenticating yourself to the various systems you use (house, car, phone, computer, websites, etc.) in addition to greatly increases the security of these authentication methods.</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>Security has always been a delicate balance between difficulty of bypassing and convenience. Think of all the various authentication &quot;tokens&quot; you use throughout the day. House keys, car keys, &amp; numerous passwords to your gadgets &amp; websites. These all provide little in the means of security while also tending to be inconvenient and cumbersome to use/remember. The AuthBand is a wearable open source/hardware token which aims to solve the aforementioned daily authentication problem.</p><p>It is important to note that this system is very similar to the Bionym&apos;s Nymi product (swear they looked at my project idea notebook!) but has some important differences:</p><ul><li>Does not require a smartphone<ul><li>everything you need for authentication is contained within the AuthBand</li></ul></li><li>Simple &amp; secure asymmetric authentication</li><li>Possible future applications to mobile payments<ul><li>Originally was going to use a Java Smartcard IC so that it would be possible to run apps like Visa PayWave but found them very closed off &amp; hard to get.<br><hr></li></ul></li></ul><p><strong>System Overview</strong></p><p><img src="https://cdn.hackaday.io/images/1978821408453984997.PNG"></p><p><strong>Communication</strong></p><p>A Bluetooth Low Energy module provides low power communication with the authentication systems.</p><p><strong>Security</strong></p><p>A crypto module is a tamper resistant module that securely &amp; efficiently carries out cryptographic operations. They also usually have secure memory for storing sensitive data such as keys. In this project, a crypto module will be used to generate a public/private key pair &amp; when challenged by an authentication system, use the stored private key to generate the valid response.</p><p>A heart-rate monitor module is used to perform the validation of the user upon the AuthBand being placed onto the body. When the user dons the AuthBand for the first time, his/her heart-rate will be analyzed &amp; a biometric template will be created &amp; stored in the secure element. Every time the AuthBand is taken off, the user must be re-validated which is transparent to the user but involves analyzing the user&apos;s heart-rate &amp; validating it against the stored template.</p><p><strong>User Input</strong></p><p>Accelerometers &amp; gryoscopes are used to capture user inputs such as tap gestures or hand waves. These gestures are used to determine if a user wants to do an action (i.e. unlock the door).</p><p><strong>Video</strong></p><p>The video below shows the AuthBand system design followed by its ideal usage case. I haven&apos;t really touched upon the usage case details much in the write up because it is a whole additional project/movement in itself but it is crucial in my opinion.  It is important to not only move to a public key cryptography solution but also to take he security critical authentication processes out of the hands of manufacturers, web designers, etc., &amp; move them to one main authentication entity (depicted as Public Key Server in the video below). This serves two purposes: solves the public key infrastructure issue &amp; it remedies the all too often problem of companies implementing security systems incorrectly. Anyway that is my rant for now, more on that later...</p><p><iframe allowfullscreen="" width="500" height="281" src="//www.youtube.com/embed/5U2nHKnr2Oc" frameborder="0"></iframe></p>
                                
                            </p>

                        <li>
                            <h2><a href="/project/2572/log/21539-crypto-module">Crypto Module</a></h2>
                            <small class="authorship">23 days ago  &bull;
                            <a class="gray-link" href="/project/2572/log/21539#discussion-list">
                                0 comments
                            </a>
                            </small>
                            <p><p>After doing more research into the secure authentication IC marketplace, I have decided to move away from my original plan of a secure element -- essentially a tamper resistant microcontroller that usually runs some kind of JavaCard operating system. The advantage of the secure element implementation was that current JavaCard apps like Visa Paypass could be integrated into the project but I decided that the secure elements were to secretive &amp; hard to get ahold of &amp; more of a headache than I think they are worth. Also they introduce extra layers of complexity that I believe increase the probability of a security issue. </p><p>What am I considering using now you might ask. Good question! Well, I wanted a small footprint, extremely low powered IC with anti-tamper mechanisms that is made to efficiently carry out <a href="https://blog.cloudflare.com/ecdsa-the-digital-signature-algorithm-of-a-better-internet/" target="_blank">Elliptic Curve Digital Signature Algorithm (ECDSA)</a>. Also two other vital features I was looking for were availability (no sense in finding the perfect chip if I can&apos;t get my hands on it) &amp; a good &amp; affordable dev kit. With these features in mind, my searching lead me to the <a href="http://www.atmel.com/Images/Atmel-8895S-CryptoAuth-ATECC108A-Datasheet-Summary.pdf" target="_blank">Atmel ATECC108A</a>. I think it is a perfect fit for the project &amp; ordered the evaluation kit the other day. I should have some preliminary results up in about two weeks &amp; if all goes well, I will get the dev kit (more expensive so wanted to use the much cheaper eval board first) &amp; start building my first prototype!</p>
                                
                            </p>
                        </li>

Now with the 2015 Hackaday Prize in full swing, I have decided to reapply myself to the project &amp; make some good progress towards an idea I truly believe in.</p>
