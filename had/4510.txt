                    <h5>Description</h5>
                    <div class="description">Make a Weloop Tommy an open source sport watch that can be used without a mobile phone. Want to see a heart rate when running, cycling speed and cadence when cycling or number of laps while swimming? Help to make it possible!</div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>The idea is simple. Write a custom firmware that allows to communicate with many BLE enabled sensors and shows live values on a watch screen.</p><p>This project is divided into six phases:</p><ul><li><strong>Phase 1</strong>: find out and document watch pinout so it will be possible to communicate with screen, accelerometer and a flash memory from a custom firmware. <strong>DONE</strong></li><li><strong>Phase 2</strong>: write a custom firmware that will print sample text on a screen and expose a BLE service. Firmware will be flashed using SWD connector. <strong>DONE</strong></li><li><strong>Phase 3</strong>: make it possible to upload a custom firmware using OTA. This will save the waterproof case from destruction. <strong>DONE</strong></li><li><strong>Phase 4: </strong>update softdevice to the newest version (S110 8.0.0). Add possibility to switch to S120 2.0.0 softdevice. <strong>DONE</strong></li><li><strong>Phase 5</strong>: create a &quot;terminal&quot; firmware that allows to control what is displayed on a watch from an android/iOS app. <strong>IN PROGRESS</strong></li><li><strong>Phase 6</strong>: create a &quot;standalone&quot; firmware using S120 softdevice that can connect to external sensors (HRS, CSCS etc) without a phone.<strong></strong> <strong>IN PROGRESS</strong></li></ul><p><strong>Roadmap:</strong></p><ul><li><strong>version </strong><strong>n+1 </strong><br><ul><li>add storage library to add possibility to upload custom fonts and multiple watchsets (in current version only one watchset can be uploaded and there&apos;s 3kB limit). I think that <a href="https://github.com/pellepl/spiffs" target="_blank" rel="nofollow">https://github.com/pellepl/spiffs</a> is the best option but will do some more research.</li><li>static content in watchsets: images, fonts</li><li>support for country specific characters in user fonts</li></ul></li><li><strong>version </strong><strong>n+2 </strong><br><ul><li>email/sms/missed call notifications, need a storage library for this (from n+1 version). I&apos;m not sure how to implement this but I don&apos;t like the way it worked in weloop, I don&apos;t want to remove every notification manually and don&apos;t want to use big amount of flash space for notifications</li></ul></li></ul><p><strong>Instructions:</strong></p><p>Some instructions can be found here: <a href="http://ossw.github.io/">http://ossw.github.io/</a> </p><p><strong>Current version: 0.2.0</strong></p><p><a href="https://github.com/ossw/ossw-firmware-s120/releases/tag/0.2.0">https://github.com/ossw/ossw-firmware-s120/releases/tag/0.2.0</a><span class="redactor-invisible-space"><br></span></p><p><strong>Beta testers group</strong><br></p><a href="https://groups.google.com/group/ossw-beta">https://groups.google.com/group/ossw-beta</a><p>If you like this idea and want to support my work please donate <a href="https://www.paypal.com/cgi-bin/webscr?cmd=_donations&amp;business=V6KL4EH8ZCQX2&amp;lc=US&amp;item_name=Open%20source%20sport%20watch&amp;currency_code=USD&amp;bn=PP%2dDonationsBF%3abtn_donate_SM%2egif%3aNonHosted" target="_blank">HERE</a>. Thanks!<br></p>

                            <h2><a href="/project/4510/log/18892-still-alive">Still alive</a></h2>
                            <small class="authorship">3 months ago  &bull;
                            <a class="gray-link" href="/project/4510/log/18892#discussion-list">
                                25 comments
                            </a>
                            </small>
                            <p><p>Just quick update what I've managed to do:</p><ul><li>it's possible to choose between central (without phone) and peripheral (with phone) mode on watch start</li><li>I've written simple android app that sends sensor data from IpSensorMan to a watch in peripheral mode. Currently it's only HR and CSC but it's extremely easy to add any other sensor type that IpSensorMan supports. IpSensorMan supports both ANT and BLE devices (to use BLE devices you need to install <a href="http://www.iforpowell.com/cms/index.php?page=btle-beta" target="_blank">beta</a> version of this app)</li><li>In central mode watch can connect directly to HR sensor. Other sensors will be added but it requires a little bit more work then adding them in peripheral mode.</li><li>I rewrote gui so it's now more generic and allow to use controls on a screen. Still it requires a bit of work to be able to define what should be displayed on a watch from the phone screen (like cycling speed in this place and this size, heart rate there etc.)</li><li>Handling notifications like missed phone calls and sms is still TODO</li><li>I've managed to create a firmware that reverts watch to original softdevice/bootloader so it will be possible to go back to original firmware if you don't like mine ;)</li><li>I've embedded PAWN abstract machine so it's possible to run scripts on a watch but only from internal flash/ram, have to rewrite it to be able to read them from external flash.</li></ul><p>All of this is still an alpha version so I have to put some more work to make it user friendly and generic. Expect some photos/videos how this works in the next week.<br><br></p>
                                
                            </p>
                        </li>

In central mode watch may use services exposed by other devices.&#xA0;</p><p>Heart rate sensor is always a peripheral device.</p><p><br>So in peripheral mode:<br>- phone is a central device and connects to a watch (peripheral device)<br>- phone may also connect to&#xA0;other peripheral devices like HRS<br>- any data that is sent to watch is collected by the phone<br><br>In central mode:</p><p>- watch is a central device and connects to&#xA0;peripheral devices like HRS</p><p>- watch collects and process data from connected devices</p><p>- phone is not participating in this mode</p>

You have to choose "Distribution packet (ZIP)" when uploading this file.</p><p>ossw-bootloader-s120-0.3.zip - it's a bootloader&nbsp;update pack for those who installed&nbsp;ossw-installer-combined-0.2.bin (old version)</p><p>So if you're really sure you want to install this then you need only the first and the second file. But if you want to test PAWN scripts then you will have to modify this firmware a little bit because I've just test it by embedding compiled scripts into the firmware so there's no way to upload it yet (but it will be very easy).&nbsp;</p><p>The original bootloader restarts every one minute, don't know why, ask weloop guys why they did it that way ;)</p>
