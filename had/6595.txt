                <div class="section section-description">
                    <h5>Description</h5>
                    <div class="description">Troubleshooting USB devices can be troublesome especially if you need to check voltage or current.  It usually involves cutting a USB cable and attaching DMM probes. This project aims to solve that by making it a snap to connect inline with a USB device. This has wide ranging applications to not just real time data but data logging. One application is checking battery capacity. Another might be checking your projects usage during different modes or settings. Even during firmware development while tweaking code. It won't do below 1mA but you would  be able to tell if your device drops into sleep or not. This is one tool every workbench can't be without. <br /><br />With a modular design not only can the Backpack do USB you can swap out the base with the VA Tester so you can check anything up 26V and 3.2A. The limitation is only when using the backpack which houses the INA219B from TI.<br /><br /></div>
                </div>
            

            
                <div class="section section-details">
                    <h5>
                        
                            Details
                        
                    </h5>
                    

                    
                        <div class="post-content details-content">
                            <p><p>The USB Tester consists of two parts. The USB Tester Base which has the USB connectors and test points for use with your own DMM. Then the backpack which has the OLED display, microcontroller and INA219B for sensing voltage/current. The backpack can be connected to either the USB Tester or VA Tester. Then the Bluetooth Backpack can be connected to either base as well. One note is when the backpack is used with the VA Tester it will need its own power since there is no onboard regulator for the 5V supply. <br></p><p>Firmware Features</p><ul><li>Graph of current - 1.28min of history based on default sample speed</li><li>100ms sample time (Adjustable in firmware)</li><li>Peaks</li><li>mWh and mAh tracking</li><li>Big Readout Display with Bar Graphs</li><li>JSON for serial communications</li></ul><p>Java App - Data Logging/View Realtime Data</p><ul><li>Implemented JSON over serial</li><li>Save as CSV</li><li>Uses temp file for long term logging</li><li>Able to resume unsaved logging</li><li>Graph shows max, avg and min for current and voltage</li></ul><p>We&#x2019;d like to thank Ed for all of his help and contributions.<a href="http://www.aerodynes.fr/2013/09/08/fried-circuits-tester-multiscreen-and-energy-measurement/">http://www.aerodynes.fr/2013/09/08/fried-circuits-tester-multiscreen-and-energy-measurement/</a></p><p>Wizkers.io (Created by Ed)</p><p>The USB Tester is also supported by Wizkers.io which supports other devices and provides view live data and logging via a Chrome application which can run offline. </p><p><strong>Stage 2, quarter finals video: Introductions</strong></p><iframe style="width: 500px; height: 281px;" src="//www.youtube.com/embed/C4xSfABtHek" allowfullscreen="" frameborder="0" height="281" width="500"></iframe><p><br><strong></strong></p><p><br><strong></strong></p><p><strong>Stage 3, Semifinals/Best Product video: Improvements</strong></p><iframe style="width: 500px; height: 281px;" src="//www.youtube.com/embed/RFF1sGM4pug" allowfullscreen="" frameborder="0" height="281" width="500"></iframe><p><strong>Model of USB Tester Product Look/Feel</strong></p><p><strong><img src="https://cdn.hackaday.io/images/2999511439794985737.png"></strong></p><p>This design allows the top and bottom to still be easily seperated while still protecting all electronics. By using a transparent material the LED is still visable though the case. </p><p>Licenses</p><p>All the USB Tester firmware and designs are CC-SA-BY-NC. The NC part is actually only there so that if someone wanted to sell their own version I would like to be in the loop. Other than that it is open source and companies are free to use it internally. All sources are on Github for your viewing pleasure.<br></p>
